# Стандарты 1С: Организация хранения данных (расширение)

**Источник:** [v8std.ru — Организация хранения данных](https://v8std.ru) | [ITS 1С](https://its.1c.ru/db/v8std/browse/13/-1/26/27)  
**Связанные инструменты:** `check_standards`, `check_metadata_standards`, `get_object_structure`, `solve_check`, `analyze_bsl`

> Расширенный каталог стандартов 1С по хранению данных. Дополняет `its_data_storage.md` новыми стандартами: имя/синоним/комментарий, подсказка, коды, строковые типы, составные типы, проведение, активность движений, самодостаточность регистров, комментарий документа, удаление устаревших, константы, неактуальные объекты, предопределённые. Стандарты #std474, #std478, #std473, #std432, #std728, #std603, #std633, #std477, #std531, #std534, #std632, #std638, #std697, #std541, #std474.

## #std474: Имя, синоним, комментарий

**ITS:** https://its.1c.ru/db/v8std/content/474/hdoc | **v8std:** https://v8std.ru/std/474/

### Правила
1. Синоним объекта заполняйте обязательно — осмысленный и лаконичный.
2. Не используйте в синонимах сленг, русские слова латиницей; общепринятые сокращения (`НДС`, `МСФО`) допустимы.
3. Стандартным реквизитам и табличным частям задавайте синонимы по прикладному смыслу.

### Пример

```
Объект: Справочник.Контрагенты
Синоним: Контрагенты (НЕ "Контры" или "Klienty")
Реквизит: НаименованиеПолное
Синоним: Полное наименование
```

### Диагностики BSL LS
- `ForbiddenMetadataName` — запрещённое имя
- `MetadataObjectNameLength` — слишком длинное имя
- `SameMetadataObjectAndChildNames` — совпадение имени объекта и дочернего

## #std478: Подсказка и проверка заполнения

**ITS:** https://its.1c.ru/db/v8std/content/478/hdoc | **v8std:** https://v8std.ru/std/478/

### Правила
1. `Подсказка` задавайте только реквизитам, требующим пояснения.
2. Для обязательных реквизитов ставьте `Проверка заполнения = Выдавать ошибку`.
3. Условную обязательность обрабатывайте в `ОбработкаПроверкиЗаполнения` и условным оформлением убирайте автопометку.

## #std473: Использование кодов (номеров) объектов

**ITS:** https://its.1c.ru/db/v8std/content/473/hdoc | **v8std:** https://v8std.ru/std/473/

### Правила
1. Применяйте коды, когда пользователи работают с большими списками.
2. Если поиск по коду удобнее, чем по названию (статьи расходов, единицы измерения).
3. Если код имеет прикладной смысл (ОКЕИ, номер ГТД).

## #std432: Использование реквизитов строкового типа

**ITS:** https://its.1c.ru/db/v8std/content/432/hdoc | **v8std:** https://v8std.ru/std/432/

### Правила
1. Используйте переменную длину (`Допустимая длина = Переменная`) с обязательным максимумом.
2. Длина конкатенированной строки = сумме длин исходных.
3. Нерегламентированные строки — разумный лимит (полное наименование — 250, имя файла — 260).

### Пример

```bsl
// В запросе — явное ограничение длины
ВЫБРАТЬ ВЫРАЗИТЬ(Т.Комментарий КАК СТРОКА(1000)) КАК Комментарий
```

## #std728: Ограничения на использование реквизитов составного типа

**ITS:** https://its.1c.ru/db/v8std/content/728/hdoc | **v8std:** https://v8std.ru/std/728/

### Правила
1. В составном типе для соединений/отборов/сортировки оставляйте только ссылочные типы.
2. Для совмещения ссылки и произвольного ввода используйте отдельный справочник.
3. Не применяйте `ЛюбаяСсылка`, `СправочникСсылка` без явного состава типов (исключение — универсальные объекты).

### Проверки
- АПК: 1329, 1330

## #std603: Требования к проведению документов

**ITS:** https://its.1c.ru/db/v8std/content/603/hdoc | **v8std:** https://v8std.ru/std/603/

### Правила
1. Большинство документов должны проводиться (`Проведение = Разрешить`); непроведённый — черновик.
2. Этапы жизненного цикла — через статусы.
3. Исключения: документы, фиксирующие факт события, но не отражающие учёт.

## #std633: Использование активности движений

**ITS:** https://its.1c.ru/db/v8std/content/633/hdoc | **v8std:** https://v8std.ru/std/633/

### Правила
1. Активность записей регистров учитывайте во всех запросах и отчетах: `ГДЕ Активность = ИСТИНА`.
2. В универсальных отчетах по произвольным регистрам тоже.
3. При отмене проведения с ручной корректировкой отключайте активность, а не удаляйте движения.

### Пример

```bsl
// ПРАВИЛЬНО: учитывать активность
ВЫБРАТЬ
    Рег.Номенклатура,
    Рег.КоличествоОстаток
ИЗ
    РегистрНакопления.ТоварыНаСкладах.Остатки(
        &КонецПериода,
        Склад = &Склад
            И Активность = ИСТИНА) КАК Рег
```

## #std477: Самодостаточность регистров

**ITS:** https://its.1c.ru/db/v8std/content/477/hdoc | **v8std:** https://v8std.ru/std/477/

### Правила
1. Регистр должен быть логически независим от регистраторов.
2. Любая логика и отчеты по регистру не должны обращаться к полям регистратора.
3. Обращение к полям регистратора через точку = неявное соединение с доп. таблицами.

### Пример

```bsl
// ПЛОХО: обращение к полю регистратора через точку
ВЫБРАТЬ
    Рег.Регистратор.Дата КАК Дата,  // неявное соединение
    Рег.Сумма

// ПРАВИЛЬНО: регистратор как измерение или не использовать
ВЫБРАТЬ
    Рег.Период КАК Дата,
    Рег.Сумма
```

## #std531: Реквизит «Комментарий» у документов

**ITS:** https://its.1c.ru/db/v8std/content/531/hdoc | **v8std:** https://v8std.ru/std/531/

### Правила
1. Создавайте реквизит `Комментарий` (строка неограниченной длины) для служебных заметок.
2. Прикладной текст выносите в отдельные реквизиты (`ДополнительнаяИнформация`).
3. Для внешнего редактирования — `ВвестиСтроку` или БСП `ПоказатьФормуРедактированияКомментария`.

## #std534: Удаление устаревших объектов метаданных

**ITS:** https://its.1c.ru/db/v8std/content/534/hdoc | **v8std:** https://v8std.ru/std/534/

### Правила
1. Не удаляйте объекты сразу — пометьте префиксом `Удалить` (`Obsolete`).
2. Обеспечьте перенос данных в новую структуру.
3. Для удаляемого документа-регистратора запретите формирование движений и снимтие пометки удаления.

### Пример

```
Старое имя: Справочник.СтарыеКонтрагенты
Новое имя: Справочник.УдалитьСтарыеКонтрагенты
```

## #std632: Использование констант

**ITS:** https://its.1c.ru/db/v8std/content/632/hdoc | **v8std:** https://v8std.ru/std/632/

### Правила
1. Запись константы выполняйте вне транзакций — может стать узким местом при конкурентной работе.
2. Во время записи константы работа других сеансов приостанавливается при попытке записать ту же константу.

### Пример

```bsl
// НЕПРАВИЛЬНО — запись константы в обработчике проведения
Процедура ОбработкаПроведения(Отказ, РежимПроведения)
    ТекущееЗначение = Константы.СчетчикПроведенныхДокументов.Получить();
    Константы.СчетчикПроведенныхДокументов.Установить(ТекущееЗначение + 1);
КонецПроцедуры

// ПРАВИЛЬНО — константа записывается отдельно, не в транзакции документа
Процедура ОбновитьСчетчик()
    НачатьТранзакцию();
    Попытка
        Константы.Счетчик.Установить(НовоеЗначение);
        ЗафиксироватьТранзакцию();
    Исключение
        ОтменитьТранзакцию();
    КонецПопытки;
КонецПроцедуры
```

## #std638: Работа с неактуальными объектами

**ITS:** https://its.1c.ru/db/v8std/content/638/hdoc | **v8std:** https://v8std.ru/std/638/

### Правила
1. Неактуальные объекты (уволенный сотрудник) не удаляйте — нарушится ссылочная целостность.
2. Запрещайте их выбор через `ОбработкаПолученияДанныхВыбора` и `ОбработкаПолученияФормы`.
3. Ограничение по умолчанию задавайте реквизитом `Недействителен` в отборе.

### Пример

```bsl
Процедура ОбработкаПолученияДанныхВыбора(ДанныеВыбора, Параметры, СтандартнаяОбработка)
    Если Не Параметры.Отбор.Свойство("Недействителен") Тогда
        Параметры.Отбор.Вставить("Недействителен", Ложь);
    КонецЕсли;
КонецПроцедуры
```

## #std697: Использование предопределенных элементов

**ITS:** https://its.1c.ru/db/v8std/content/697/hdoc | **v8std:** https://v8std.ru/std/697/

### Правила
1. В большинстве случаев создавайте предопределённые элементы автоматически.
2. В подчинённом узле РИБ они не создаются и не обновляются автоматически.
3. Для таблиц вне РИБ ставьте `ОбновлениеПредопределенныхДанных = ОбновлятьАвтоматически`.

### Пример

```bsl
Если ПланыОбмена.ГлавныйУзел() = Неопределено Тогда
    // Заполняем предопределенные элементы
    СоздатьПредопределенные();
КонецЕсли;
```

## #std541: Уточнение сущности объекта метаданных

**ITS:** https://its.1c.ru/db/v8std/content/541/hdoc | **v8std:** https://v8std.ru/std/541/

### Правила
1. Разные сущности — разные объекты метаданных (`Организации` и `Подразделения`).
2. Уточняйте сущность явными реквизитами (`Булево`, перечисления), а не косвенными признаками.
3. Не заводите несколько булевых реквизитов для состояний — используйте перечисление статусов.
