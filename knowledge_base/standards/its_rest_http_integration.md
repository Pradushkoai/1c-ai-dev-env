# Стандарты 1С: Интеграции через REST/HTTP

**Источник:** [v8std.ru](https://v8std.ru) | [ITS 1С](https://its.1c.ru/db/v8std)  
**Связанные инструменты:** `audit_security`, `solve_check`, `check_standards`, `analyze_bsl`

> Полный каталог стандартов 1С по интеграциям: HTTP-сервисы, HTTP-соединения, веб-сервисы, REST API (OData), внешние компоненты, COM-интеграции. Стандарты #std748, #std678, #std679, #std770, #std774, #std775, #std669, #std700, #std794, #std542, #std798.

---

## Общие сведения

Интеграции в 1С выполняются через:
- **HTTP-сервисы** (приём входящих REST-запросов) и **HTTP-соединение** (исходящие запросы)
- **веб-сервисы** SOAP (`WSОпределения`, `WSПрокси`)
- **автоматический REST API** (стандартный интерфейс OData)
- **FTP/почту** (`FTPСоединение`, `ИнтернетПочтовыйПрофиль`)
- **COM-интеграцию** и внешние компоненты

Главный сквозной принцип стандартов: **таймауты, безопасный режим, профили безопасности, контроль внешнего кода.**

## #std748: Таймауты при работе с внешними ресурсами

**ITS:** https://its.1c.ru/db/v8std/content/748/hdoc | **v8std:** https://v8std.ru/std/748/

### Правила
1. Для `WSОпределения`, `WSПрокси`, `HTTPСоединение`, `FTPСоединение`, `ИнтернетПочтовыйПрофиль` **обязательно** задавайте таймаут — иначе программа зависает при недоступности удалённой стороны.
2. В общем случае таймаут — не более 3 минут; для больших передач — не более 12 часов.
3. Для веб-сервисов с операциями дольше ~20 с добавляйте контрольную операцию `Ping` (малый таймаут ~7 с), основной прокси — только после успешного `Ping`.
4. Для REST API аналог `Ping` — тестовая команда (код 200 = доступность); для FTP/WebDAV — тестовая загрузка файла-пустышки.
5. Используйте БСП: `ОбщегоНазначения.WSПрокси` (поддерживает Ping) и подсистему «Получение файлов через Интернет» для HTTP(S)/FTP(S).

### Пример (контрольный Ping перед основным вызовом)

```bsl
// Контрольный вызов Ping с малым таймаутом
PingPong = Новый WSПрокси(АдресВебСервиса, , , , , 3);
PingPong.Ping();

// Сервис жив — основной прокси с обычным таймаутом
PingPong = Новый WSПрокси(АдресВебСервиса, , , , , 60);
Результат = PingPong.Pong(НСтр("ru = 'Мяч'"));
```

### Пример (HTTPСоединение с таймаутом)

```bsl
Соединение = Новый HTTPСоединение(
    "api.example.com", 443, , , , 30,
    Новый ЗащищенноеСоединениеOpenSSL
);
Запрос = Новый HTTPЗапрос("/v1/data",
    Новый Соответствие([["Accept","application/json"]])
);
Ответ = Соединение.Получить(Запрос);
```

### Диагностики BSL LS
- `TimeoutsInExternalResources` — проверяет наличие таймаута при создании `HTTPСоединение`/`WSПрокси`/`FTPСоединение`/`ИнтернетПочтовыйПрофиль`

## #std794: Ограничения на использование внешних ресурсов

**ITS:** https://its.1c.ru/db/v8std/content/794/hdoc | **v8std:** https://v8std.ru/std/794/

### Правила
1. Используйте средства платформы вместо внешних компонент, приложений ОС и COM (например, `ВызватьПаузу` вместо средств ОС).
2. Все внешние ресурсы заявите в `РаботаВБезопасномРежимеПереопределяемый` (БСП) и в профилях безопасности кластера.
3. При включении/выключении функциональности, использующей внешние ресурсы, запрашивайте у администратора разрешение (`РаботаВБезопасномРежимеКлиент`).
4. Профиль безопасности принимает **администратор кластера** (не приложения) — модель защиты от недобросовестного приложения/администратора приложения; корректно работает модель сервиса (1cFresh).

## #std678: Безопасность прикладного программного интерфейса сервера

**ITS:** https://its.1c.ru/db/v8std/content/678/hdoc | **v8std:** https://v8std.ru/std/678/

### Правила
1. Злоумышленник может вызывать серверный код с клиента (в т.ч. через HTTP, повторяя поведение клиента платформы). Все серверные функции, доступные с клиента — потенциальная угроза: экспортные методы общих модулей с «Сервер»+«Вызов сервера», и методы форм с `&НаСервере`/`&НаСервереБезКонтекста`.
2. Исключите бизнес-логику из модулей форм — там только клиент-серверное взаимодействие и обработка реквизитов формы.
3. Серверные процедуры должны возвращать в форму **только окончательный результат** — не исходные/промежуточные данные, раскрывающие побочную информацию.
4. Особое внимание — коду с привилегированным режимом и модулям с признаком «Привилегированный».

### Диагностики BSL LS
- `SetPrivilegedMode`

## #std679: Ограничение на установку признака «Вызов сервера» у общих модулей

**ITS:** https://its.1c.ru/db/v8std/content/679/hdoc | **v8std:** https://v8std.ru/std/679/

### Правила
1. Не ставьте «Вызов сервера» всем серверным общим модулям «по умолчанию». В таких модулях — только API, реально вызываемый с клиента и не раскрывающий лишних данных.
2. Серверные методы, не предназначенные для вызова с клиента, — в серверных модулях **без** «Вызов сервера».
3. В управляемом приложении не создавайте/не получайте объекты (`СправочникОбъект`, `ДокументОбъект`) в клиентском коде — только на сервере.

## #std770: Ограничения на использование `Выполнить` и `Вычислить` на сервере

**ITS:** https://its.1c.ru/db/v8std/content/770/hdoc | **v8std:** https://v8std.ru/std/770/

### Правила
1. Опасно не только прямое выполнение пользовательского кода, но и строк, собранных из параметров серверных процедур/функций. Риск выше, если метод в модуле с «Вызов сервера».
2. Перед `Выполнить`/`Вычислить` в серверном коде включайте безопасный режим (`УстановитьБезопасныйРежим(Истина)`); в модели сервиса — и режим разделения данных.
3. Используйте БСП: `ОбщегоНазначения.ВыполнитьВБезопасномРежиме()`, `.ВычислитьВБезопасномРежиме()`, `.ВыполнитьМетодКонфигурации()`.
4. Код, которому нужен доступ к файлам и нельзя пустить в безопасный режим — только после аудита, из контролируемого источника, через централизованную процедуру.

### Пример (безопасный режим перед Выполнить)

```bsl
УстановитьБезопасныйРежим(Истина);
Выполнить Алгоритм;
```

### Диагностики BSL LS
- `DisableSafeMode`, `ExecuteExternalCode`, `ExecuteExternalCodeInCommonModule`
- EDT: `#v8cs:server-execution-safe-mode`

## #std669: Ограничение на выполнение «внешнего» кода

**ITS:** https://its.1c.ru/db/v8std/content/669/hdoc | **v8std:** https://v8std.ru/std/669/

### Правила
1. Запрещено выполнение на сервере в небезопасном режиме любого кода, не являющегося частью программы (внешние отчёты/обработки, расширения, внешние компоненты, фрагменты в `Выполнить`/`Вычислить`, схемы СКД с внешними функциями, загруженные из файлов).
2. Внешний код подключайте только средствами БСП:
   - расширения — базовая функциональность
   - доп. обработки — подсистема «Дополнительные отчёты и обработки»
   - компоненты — «Внешние компоненты»
3. По умолчанию отключите интерактивное открытие внешних отчётов/обработок через «Файл – Открыть» (в т.ч. у администратора); при подключении — предупреждение об опасности; механизм защиты от опасных действий отключать запрещено.
4. Запуск внешних программ — по #std774.

### Диагностики BSL LS
- `UsingExternalCodeTools`

## #std774: Безопасность запуска приложений

**ITS:** https://its.1c.ru/db/v8std/content/774/hdoc | **v8std:** https://v8std.ru/std/774/

### Правила
1. Командную строку внешней программы собирайте только из проверенных частей. Значения из БД, ввода пользователя, хранилища настроек — проверяйте перед запуском.
2. Небезопасны строки с символами: `` $ ` | || ; & && ``.
3. Требование распространяется на `КомандаСистемы()`, `ЗапуститьПриложение()`, `НачатьЗапускПриложения()`, `ПерейтиПоНавигационнойСсылке()`, COM `Wscript.Shell`/`Shell.Application`.
4. Для запуска внешних программ применяйте профильный API БСП.

### Диагностики BSL LS
- `FileSystemAccess`, `ExternalAppStarting`

## #std775: Безопасность ПО, вызываемого через открытые интерфейсы (COM)

**ITS:** https://its.1c.ru/db/v8std/content/775/hdoc | **v8std:** https://v8std.ru/std/775/

### Правила
1. При интеграции через COM отключайте исполнение произвольного кода вызываемого приложения.
2. Перед программным открытием Word/Excel через COM запрещайте макросы.
3. Если автозапуск макросов нужен — реализуйте явные настройки безопасности (отдельно клиентские и серверные; подписанные — рекомендуется).

### Пример (Word — отключение автозапуска макросов)

```bsl
ОбъектWord = Новый COMОбъект("Word.Application");
ОбъектWord.WordBasic.DisableAutoMacros(1);
Документ  = ОбъектWord.Documents.Open(ИмяФайла);
```

### Пример (Excel — проверка подписи макросов)

```bsl
ОбъектExcel = Новый COMОбъект("Excel.Application");
ОбъектExcel.AutomationSecurity = 3; // msoAutomationSecurityForceDisable
Документ = ОбъектExcel.Workbooks.Open(ФайлДокумента);
Если Документ.VBASigned Тогда
    Документ.Close();
    Документ = ОбъектExcel.Workbooks.Open(ФайлДокумента); // с макросами
Иначе
    Документ.Close();
    ВызватьИсключение НСтр("ru = 'Документ не подписан. Открытие отменено.'");
КонецЕсли;
```

### Проверки
- АПК: `#acc:536`, `#acc:537`

## #std542: Доступ к файловой системе из кода конфигурации

**ITS:** https://its.1c.ru/db/v8std/content/542/hdoc | **v8std:** https://v8std.ru/std/542/

### Правила
1. Учитывайте права пользователя ОС (на клиенте — текущий пользователь; на сервере — пользователь сервера 1С/веб-сервера).
2. Запрещено записывать файлы в каталог `КаталогПрограммы()` (только чтение/запуск оттуда).
3. Имена формируемых системой файлов — только английские буквы и цифры; текстовые файлы — в UTF-8 (критично для macOS).
4. Для временных файлов — `ПолучитьИмяВременногоФайла()` (не захардкоженные пути).

### Пример (неправильно/правильно)

```bsl
// Неправильно
ИмяФайла = "C:\Временные файлы 1С\TempFile.xml";

// Правильно
ИмяФайла = ПолучитьИмяВременногоФайла("xml");
Данные.Записать(ИмяФайла);
```

### Диагностики BSL LS
- `FileSystemAccess`, `MissingTemporaryFileDeletion`, `TempFilesDir`

## #std798: Использование ботов системы взаимодействия

**ITS:** https://its.1c.ru/db/v8std/content/798/hdoc | **v8std:** https://v8std.ru/std/798/

### Правила
1. В конфигурации должно быть регламентное задание, вызывающее `СистемаВзаимодействия.ВыполнитьОбработкуБотов()`. Метод работает без учёта разделителей, запускает WebSocket-процесс по одному на область.
2. В модели сервиса (1cFresh): не создавайте предопределённых ботов; включайте/отключайте программно. Регламентное задание — одно и неразделённое.
3. Пользователь системы взаимодействия бота не должен быть связан с пользователем ИБ.
4. Обработчик сообщений должен работать без необработанных исключений — иначе сообщение остаётся в очереди и блокирует следующие.

## #std700: Установка внешних компонент и расширений платформы

**ITS:** https://its.1c.ru/db/v8std/content/700/hdoc | **v8std:** https://v8std.ru/std/700/

### Правила
1. Установка внешних компонент и расширений — интерактивная, решение принимает пользователь; в диалоге объясняйте, зачем нужна компонента и что не будет работать без неё.
2. Предлагайте установку в момент запуска прикладного действия, зависящего от компоненты.
3. Не устанавливайте компоненту автоматически «молча» в коде.

## Обобщённая практика: HTTP-сервисы и REST API

> Платформа 1С предоставляет **HTTP-сервисы** (входящие REST) и **автоматический REST-сервис OData**. Ниже — кросс-стандартные рекомендации, вытекающие из #std678, #std748, #std794 и документации платформы.

### HTTP-сервисы (входящие запросы)
1. Логику выносите из модулей сервисов в общие серверные модули; модуль HTTP-сервиса — только разбор запроса и формирование ответа (аналог #std678 п.1.2 для форм).
2. Обязательно валидируйте входные данные и проверяйте права (`#std737: Проверка прав доступа`).
3. Возвращайте корректные HTTP-коды и заголовки; для ошибок — структурированный JSON с кодом и описанием.
4. Используйте `ISAPI`/веб-сервер с TLS; для аутентификации — OAuth 2.0 (через БСП) или базовую с ограничением по профилям безопасности.

### Пример обработки входящего HTTP-запроса

```bsl
Функция ordersGET(Запрос)
    Если Не ПроверитьПраваНаЧтениеЗаказов() Тогда
        Ответ = Новый HTTPСервисОтвет(403);
        Ответ.Заголовки.Вставить("Content-Type", "application/json");
        Ответ.УстановитьТелоИзСтроки(СериализоватьОшибку("Доступ запрещён"));
        Возврат Ответ;
    КонецЕсли;

    Параметры = Новый Соответствие;
    Параметры.Вставить("Контрагент", Запрос.ПараметрыURL.Получить("id"));

    Данные = ИнтеграцияСервер.ЗаказыКонтрагента(Параметры);
    Ответ = Новый HTTPСервисОтвет(200);
    Ответ.Заголовки.Вставить("Content-Type", "application/json; charset=utf-8");
    Ответ.УстановитьТелоИзСтроки(СериализоватьВJSON(Данные));
    Возврат Ответ;
КонецФункции
```

### Исходящие HTTP-запросы (HTTPСоединение)
- Таймаут обязателен (#std748)
- Используйте `ЗащищенноеСоединениеOpenSSL` для HTTPS
- Обрабатывайте коды ответа и повторы (экспоненциальный backoff для 5xx/429)
- Не хардкодьте URL/токены — храните в регистре сведений/константе с шифрованием (#std740 — безопасное хранение паролей)

### REST API OData
- Стандартный интерфейс `/<публикация>/odata/standard.odata/`. Используйте только для чтения там, где это оправдано; для записи — собственные HTTP-сервисы с контролем.
- Ограничьте доступными объектами через настройки публикации и права.

### JSON (обобщённая практика)
- Чтение: `ЧтениеJSON` + `ПрочитатьJSON()` (для объектов — `СериализаторXDTO.ПрочитатьJSON()` или ручной разбор)
- Запись: `ЗаписьJSON` + `ЗаписатьJSON()`, `НастройкиСериализацииJSON` (`ФорматСериализацииДат.JSON`)
- Кодировка UTF-8 без BOM
