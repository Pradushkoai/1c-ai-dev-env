## #std681: Обработчики событий объектов

**ITS:** https://its.1c.ru/db/v8std/content/681/hdoc

### Суть
Обработчики событий объектов (ПередЗаписью, ПриЗаписи, ОбработкаПроведения и др.) должны содержать только бизнес-логику, специфичную для события.

### Правила
- Обработчик = тонкая обёртка, вызывает общий модуль с бизнес-логикой
- Не дублировать проверки в разных обработчиках
- Не выполнять тяжёлые операции в `ПриЗаписи` — лучше в `ОбработкаПроведения`
- Использовать `Отказ` для блокировки, а не `ВызватьИсключение`

### Пример

```bsl
// ПРАВИЛЬНО: обработчик = обёртка
Процедура ОбработкаПроведения(Отказ, РежимПроведения)
    ПроведениеСервер.ПодготовитьДанныеПроведения(ЭтотОбъект);
    ПроведениеСервер.ОтразитьДвижения(ЭтотОбъект);
КонецПроцедуры

// ПЛОХО: вся логика в обработчике
Процедура ОбработкаПроведения(Отказ, РежимПроведения)
    // 200 строк бизнес-логики
    Движения.Товары.Очистить();
    Для Каждого Стр Из Товары Цикл
        Если Стр.Количество <= 0 Тогда
            Отказ = Истина;
            Сообщить("...");
            Возврат;
        КонецЕсли;
        // ... ещё 190 строк
    КонецЦикла;
КонецПроцедуры
```

## #std682: Обработчики событий форм

**ITS:** https://its.1c.ru/db/v8std/content/682/hdoc

### Суть
Обработчики событий форм (ПриОткрытии, ПриЗакрытии, ПриИзменении) должны содержать только UI-логику.

### Правила
- Бизнес-логика — в общих модулях с `&НаСервере`
- Обработчик `ПриИзменении` — минимальный: вызвать сервер, обновить форму
- Не выполнять запросы в обработчиках формы (кроме `ПриСозданииНаСервере`)
- Не открывать модальные формы в `ПриОткрытии` — использовать `ПоказатьПредупреждение`

### Пример

```bsl
// ПРАВИЛЬНО
&НаКлиенте
Процедура НоменклатураПриИзменении(Элемент)
    ОбновитьЦенуНаСервере();
КонецПроцедуры

&НаСервере
Процедура ОбновитьЦенуНаСервере()
    Объект.Цена = ЦенообразованиеСервер.ПолучитьЦену(Объект.Номенклатура);
КонецПроцедуры

// ПЛОХО: запрос в обработчике формы
&НаКлиенте
Процедура НоменклатураПриИзменении(Элемент)
    Запрос = Новый Запрос("ВЫБРАТЬ ...");  // НА КЛИЕНТЕ!?!
    // ...
КонецПроцедуры
```

## #std683: Использование подписок на события

**ITS:** https://its.1c.ru/db/v8std/content/683/hdoc

### Суть
Подписки на события позволяют обработать одно событие для нескольких объектов без дублирования кода.

### Правила
- Использовать подписки когда одинаковая логика для 2+ объектов
- Обработчик подписки — в общем модуле с признаком "Сервер"
- Имена подписок — осмысленные: `ПередЗаписьюДокументовСерии`
- Не злоупотреблять — каждая подписка замедляет запись

### Пример

```bsl
// Подписка: ПередЗаписьюДокументаПроверитьСерии
// Источник: Документ.Реализация, Документ.Поступление, ...

// Обработчик в общем модуле
Процедура ПередЗаписьюДокументаПроверитьСерии(Источник, Отказ) Экспорт
    СерверныйМодуль.ПроверитьЗаполнениеСерий(Источник, Отказ);
КонецПроцедуры
```

## #std684: События при записи в обмен данными

**ITS:** https://its.1c.ru/db/v8std/content/684/hdoc

### Суть
Обработчики `ПриЗагрузке`, `ПередЗаписью` в обмене данными должны учитывать режим обмена.

### Правила
- В обработчиках проверять `ОбменДанными.Загрузка = Истина`
- При загрузке — не вызывать проверки и проведения
- Не использовать `ЭтотОбъект.Записать()` в обработчике записи

### Пример

```bsl
Процедура ПередЗаписью(Отказ)
    Если ОбменДанными.Загрузка Тогда
        Возврат;  // в режиме обмена — пропускаем проверки
    КонецЕсли;
    // ... обычные проверки
КонецПроцедуры
```

## #std688: События регламентных заданий

**ITS:** https://its.1c.ru/db/v8std/content/688/hdoc

### Суть
Обработчики регламентных заданий должны быть устойчивыми к ошибкам и блокировкам.

### Правила
- Тело — в `Попытка/Исключение` с записью в журнал регистрации
- Не использовать интерактивные вызовы (`ПоказатьПредупреждение`)
- Не рассчитывать на наличие пользователя (сеанс фоновый)
- Учитывать возможность параллельного выполнения (блокировки)

### Пример

```bsl
Процедура РассчитатьСебестоимостьРегламентное() Экспорт
    Попытка
        Блокировка = Новый БлокировкаДанных;
        Блокировка.УстановитьДанные("РегистрНакопления.Себестоимость");
        Если Не Блокировка.Заблокировать() Тогда
            Возврат;  // другой сеанс уже работает
        КонецЕсли;
        
        СебестоимостьСервер.Рассчитать();
    Исключение
        ЗаписьЖурналаРегистрации(
            "РегламентноеЗадание.Себестоимость",
            УровеньЖурналаРегистрации.Ошибка,
            , , ОписаниеОшибки()
        );
    КонецПопытки;
КонецПроцедуры
```

## #std690: Обработчики событий регистра расчета

**ITS:** https://its.1c.ru/db/v8std/content/690/hdoc

### Суть
Регистры расчета имеют специфические события: `ПриНачислении`, `ПриПерерасчете`.

### Правила
- Не выполнять длительные расчёты в `ПриНачислении` — выносить в общий модуль
- Использовать `Перерасчет` для автоматических доначислений
- Не блокировать другие регистры без необходимости

## #std691: Обработчики событий регистра сведений

**ITS:** https://its.1c.ru/db/v8std/content/691/hdoc

### Суть
Регистры сведений: `ПриЗаписи`, `ПередЗаписью`. Особенности — независимая запись.

### Правила
- В `ПередЗаписью` — проверка корректности значений
- В `ПриЗаписи` — отражение в других регистрах (например, пересчёт итогов)
- Не использовать подписки на запись для триггерных расчётов — лучше регламентное задание

## #std692: События формы: список

| Событие | Когда вызывается | Что делать |
|---|---|---|
| `ПриСозданииНаСервере` | Создание формы | Инициализация параметров, заполнение данных |
| `ПриОткрытии` | После создания, перед показом | Проверка прав, фокус на элементе |
| `ПриЗакрытии` | Закрытие формы | Проверка несохранённых изменений |
| `ПередЗаписью` | Кнопка "Записать" | Проверка заполнения |
| `ПриЗаписи` | После записи на сервере | Обновление связанных данных |
| `ПослеЗаписи` | После возврата на клиент | Сообщения пользователю |
| `ПриИзменении` | Изменение реквизита | Зависимые расчёты |
| `ОбработкаВыбора` | Выбор из формы выбора | Обработка выбранного значения |

## #std693: Антипаттерны обработчиков

### 1. Запрос в обработчике на клиенте

```bsl
// КАТЕГОРИЧЕСКИ ПЛОХО
&НаКлиенте
Процедура ПриОткрытии(Отказ)
    Запрос = Новый Запрос("ВЫБРАТЬ ...");  // Запросы НЕ работают на клиенте
КонецПроцедуры
```

### 2. Тяжёлые расчёты в ПриИзменении

```bsl
// ПЛОХО: тормозит UI
&НаКлиенте
Процедура КонтрагентПриИзменении(Элемент)
    // 5 секунд расчётов на сервере
    ДолгиСервер.РассчитатьВсеДолги(Объект.Контрагент);
КонецПроцедуры

// ПРАВИЛЬНО: отложенный расчёт или фон
&НаКлиенте
Процедура КонтрагентПриИзменении(Элемент)
    ПоказатьПредупреждение(, "Долги будут пересчитаны в фоне", 1);
    ДолгиФоновый.ЗапуститьПересчет(Объект.Контрагент);
КонецПроцедуры
```

### 3. Модальные вызовы в ПриОткрытии

```bsl
// ПЛОХО: блокирует UI
&НаКлиенте
Процедура ПриОткрытии(Отказ)
    Если Не ПраваПроверить() Тогда
        ПоказатьВводСтроки(, , "Введите причину");  // модально!
    КонецЕсли;
КонецПроцедуры

// ПРАВИЛЬНО: немодальный вызов
&НаКлиенте
Процедура ПриОткрытии(Отказ)
    Если Не ПраваПроверить() Тогда
        ПоказатьПредупреждение(, "Недостаточно прав", , "Закрыть");
        Отказ = Истина;
    КонецЕсли;
КонецПроцедуры
```

### 4. Возврат без установки Отказ

```bsl
// ПЛОХО: молчаливый отказ
Процедура ОбработкаПроверкиЗаполнения(Отказ, ПроверяемыеРеквизиты)
    Если Не ЗначениеЗаполнено(Реквизит1) Тогда
        Возврат;  // пользователь не узнает, что не так
    КонецЕсли;
КонецПроцедуры

// ПРАВИЛЬНО
Процедура ОбработкаПроверкиЗаполнения(Отказ, ПроверяемыеРеквизиты)
    Если Не ЗначениеЗаполнено(Реквизит1) Тогда
        Отказ = Истина;
        Сообщить("Не заполнен реквизит 1");
    КонецЕсли;
КонецПроцедуры
```
