# Стандарты 1С: Организация хранения данных

**Источник:** [ITS: Организация хранения данных](https://its.1c.ru/db/v8std/browse/13/-1/26/23) | [v8std.ru](https://v8std.ru)

## #std448: Уточнение сущности объекта метаданных

**ITS:** https://its.1c.ru/db/v8std/content/448/hdoc

### Правила
- Справочник — для нормативно-справочной информации (НСИ)
- Документ — для фактов хозяйственной деятельности
- Регистр накопления — для числовых итогов (остатки, обороты)
- Регистр сведений — для периодической или подчинённой информации
- Не использовать справочник для операций (нужен документ)

---

## #std449: Имя, синоним, комментарий

**ITS:** https://its.1c.ru/db/v8std/content/449/hdoc

### Правила
- **Имя** — программное имя (без пробелов, на русском)
- **Синоним** — отображаемое имя (может содержать пробелы)
- **Комментарий** — описание назначения объекта
- Синоним должен быть осмысленным и понятным пользователю
- Не использовать технические термины в синониме

---

## #std450: Подсказка и проверка заполнения

**ITS:** https://its.1c.ru/db/v8std/content/450/hdoc

### Правила
- **Подсказка** — краткое описание поля (tooltip)
- **Проверка заполнения** — `Выводить ошибку` для обязательных полей
- Не использовать `Проверка заполнения = Не проверять` без причины
- Обязательные поля — отметить в подсказке

---

## #std451: Использование кодов (номеров) объектов

**ITS:** https://its.1c.ru/db/v8std/content/451/hdoc

### Правила
- Код справочника — 9-11 символов (достаточно для большинства случаев)
- Номер документа — 11 символов
- Тип кода — строка (не число, для совместимости с префиксами)
- Нумерация — автонумерация, не ручной ввод
- Не использовать код как бизнес-атрибут (это техническое поле)

---

## #std453: Использование реквизитов строкового типа

**ITS:** https://its.1c.ru/db/v8std/content/453/hdoc

### Правила
- Длина строки — минимально необходимая (не 255 «на всякий случай»)
- Для наименований — 100-150 символов
- Для кодов — 9-11 символов
- Для ИНН — 12 (физлица) или 10 (юрлица)
- Не использовать неограниченную длину без необходимости

---

## #std454: Ограничения на использование реквизитов составного типа

**ITS:** https://its.1c.ru/db/v8std/content/454/hdoc

### Правила
- Составной тип — только когда это действительно нужно
- Минимум типов в составном типе (не «все справочники»)
- В запросах — использовать `ВЫРАЗИТЬ()` для составных типов
- Составной тип замедляет запросы (нет точного индекса)

**Связанные стандарты:** #std654 (разыменование составного типа в запросах)

---

## #std455: Требования к проведению документов

**ITS:** https://its.1c.ru/db/v8std/content/455/hdoc

### Правила
- Документ может быть проведён и распроведён
- При проведении — движения в регистрах
- При распроведении — удаление движений
- Не использовать `Пометка удаления` вместо `Не проведён`
- Обработчик `ОбработкаПроведения` — должен создавать движения
- Обработчик `ОбработкаУдаленияПроведения` — должен удалять движения

---

## #std459: Использование активности движений

**ITS:** https://its.1c.ru/db/v8std/content/459/hdoc

### Правила
- `Активность` движения — для временного отключения без удаления
- Не использовать `Активность = Ложь` вместо удаления
- При копировании документа — `Активность = Истина`
- В запросах — учитывать `Активность` (фильтровать)

---

## #std461: Реквизит «Комментарий» у документов

**ITS:** https://its.1c.ru/db/v8std/content/461/hdoc

### Правила
- `Комментарий` — строка неограниченной длины
- Только у документов (не у справочников)
- Для пояснения документа пользователем
- Не использовать для технических данных

---

## #std462: Удаление устаревших объектов метаданных

**ITS:** https://its.1c.ru/db/v8std/content/462/hdoc

### Правила
- Неиспользуемые объекты — удалять из конфигурации
- Перед удалением — проверить отсутствие ссылок
- Использовать `Проверка конфигурации` для поиска устаревших
- Не оставлять «на всякий случай»
