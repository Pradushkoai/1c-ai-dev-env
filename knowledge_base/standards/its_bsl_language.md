# Стандарты 1С: Конструкции встроенного языка (BSL)

**Источник:** [v8std.ru — Соглашения при написании кода](https://v8std.ru) | [ITS 1С](https://its.1c.ru/db/v8std/browse/13/-1/26/24)  
**Связанные инструменты:** `check_standards`, `analyze_bsl`, `solve_check`, `get_code_metrics`, `architecture_analyzer`

> Полный каталог стандартов 1С по конструкциям встроенного языка BSL: каноническое написание, директивы компиляции, определение типа, переменные, инициализация, исключения, Перейти. Стандарты #std441, #std439, #std442, #std445, #std639, #std494, #std790, #std547, #std647, #std640, #std641, #std686, #std693, #std781, #std782, #std411, #std409, #std544.

## #std441: Общие требования к построению конструкций

**ITS:** https://its.1c.ru/db/v8std/content/441/hdoc | **v8std:** https://v8std.ru/std/441/

### Правила
1. Ключевые слова пишите канонически (`КонецЕсли`, не `конецЕсли`).
2. Составные логические выражения переносите по #std444.
3. Не сравнивайте `Булево` с литералами `Истина`/`Ложь`.

### Пример

```bsl
// НЕПРАВИЛЬНО
Если Флаг = Истина Тогда
    // ...
КонецЕсли;

// ПРАВИЛЬНО
Если Флаг Тогда
    // ...
КонецЕсли;
```

### Диагностики BSL LS
- `CanonicalSpellingKeywords` — каноническое написание ключевых слов
- `CompareWithBoolean` — сравнение с `Истина`/`Ложь`

## #std439: Директивы компиляции и инструкции препроцессора

**ITS:** https://its.1c.ru/db/v8std/content/439/hdoc | **v8std:** https://v8std.ru/std/439/

### Правила
1. Директивы `&НаКлиенте`/`&НаСервере` используйте только в модулях форм и команд; в остальных — инструкции препроцессора.
2. Не используйте `#Если Сервер`/`#Если Клиент` внутри клиент-серверных общих модулей.
3. Не разрывайте инструкциициями препроцессора грамматические конструкции и объявления процедур.

### Пример

```bsl
Функция КодОсновногоЯзыка() Экспорт
#Если НЕ ТонкийКлиент И НЕ ВебКлиент Тогда
    Возврат Метаданные.ОсновнойЯзык.КодЯзыка;
#Иначе
    Возврат СтандартныеПодсистемыКлиент.ПараметрКлиента("КодОсновногоЯзыка");
#КонецЕсли
КонецФункции
```

### Диагностики BSL LS
- `CompilationDirectiveNeedLess` — избыточные директивы компиляции
- `ParseError` — нарушение грамматики

## #std442: Определение типа значения переменной

**ITS:** https://its.1c.ru/db/v8std/content/442/hdoc | **v8std:** https://v8std.ru/std/442/

### Правила
1. Определяйте тип сравнением с типом, а не косвенными проверками.
2. Не используйте `Метаданные().Имя` для проверки типа ссылки — применяйте `ТипЗнч` / `НайтиПоТипу`.

### Пример

```bsl
// НЕПРАВИЛЬНО: зависит от строки
Если Ссылка.Метаданные().Имя = "ПоступлениеТоваровУслуг" Тогда

// ПРАВИЛЬНО: типобезопасно
Если ТипЗнч(Ссылка) = Тип("ДокументСсылка.ПоступлениеТоваровУслуг") Тогда
```

## #std445: Получение метаданных объектов

**ITS:** https://its.1c.ru/db/v8std/content/445/hdoc | **v8std:** https://v8std.ru/std/445/

### Правила
1. Если тип известен — получайте метаданные через `Метаданные()` у объекта/ссылки (быстрее глобального `Метаданные`).
2. Если тип неизвестен — используйте `НайтиПоТипу` / `НайтиПоПолномуИмени`.

### Пример

```bsl
// ПРАВИЛЬНО: через ссылку
МетаданныеОбъекта = Ссылка.Метаданные();

// Если тип неизвестен
Метаданные.НайтиПоПолномуИмени("Справочник." + ИмяСправочника)
```

## #std639: Использование переменных в программных модулях

**ITS:** https://its.1c.ru/db/v8std/content/639/hdoc | **v8std:** https://v8std.ru/std/639/

### Правила
1. Вместо переменных модуля используйте подходящие средства платформы (область видимости сложно контролировать).
2. Для передачи параметров между подписками и обработчиками — `ДополнительныеСвойства`.
3. Для кодов возврата — строковые константы.

### Пример

```bsl
// НЕПРАВИЛЬНО: переменная модуля
Перем КонвертацияФайлов Экспорт;

// ПРАВИЛЬНО: ДополнительныеСвойства
Процедура ПередЗаписью(Отказ)
    Если Объект.ДополнительныеСвойства.Свойство("КонвертацияФайлов") Тогда
        // ...
    КонецЕсли;
КонецПроцедуры
```

### Диагностики BSL LS
- `ExportVariables` — экспортные переменные модуля

## #std494: Предварительная инициализация локальных переменных

**ITS:** https://its.1c.ru/db/v8std/content/494/hdoc | **v8std:** https://v8std.ru/std/494/

### Правила
1. В блоках вычисления локальных переменных явно задавайте начальные значения.
2. Снижает риск ошибок, когда переменная остаётся `Неопределено`.
3. Особенно важно для больших `Если/ИначеЕсли/Иначе`.

### Пример

```bsl
// ПРАВИЛЬНО: явная инициализация
МояПеременная = Неопределено;
Если Условие1 Тогда
    МояПеременная = 10;
ИначеЕсли Условие2 Тогда
    МояПеременная = 20;
КонецЕсли;

Если МояПеременная = Неопределено Тогда
    Возврат;  // обработка незаполненного случая
КонецЕсли;
```

## #std790: Вызов исключений в коде

**ITS:** https://its.1c.ru/db/v8std/content/790/hdoc | **v8std:** https://v8std.ru/std/790/

### Правила
1. В общем случае исключение формирует платформа — вручную вызывайте только при необходимости.
2. Для бизнес-ошибок пользователю категорию можно не указывать; для программных — `ОшибкаКонфигурации`.
3. Для проверки прав — `ВыполнитьПроверкуПравДоступа`.

### Пример

```bsl
// Бизнес-ошибка
ВызватьИсключение НСтр("ru = 'Не заполнен реквизит Контрагент.'");

// Программная ошибка
ВызватьИсключение Новый ОписаниеОшибки(
    НСтр("ru = 'Нарушена целостность данных.'"),
    КатегорияОшибки.ОшибкаКонфигурации);
```

## #std547: Ограничение на использование оператора Перейти

**ITS:** https://its.1c.ru/db/v8std/content/547/hdoc | **v8std:** https://v8std.ru/std/547/

### Правила
1. Не используйте оператор `Перейти` — запутывает код.
2. `Перейти` не поддерживается в веб-клиенте.

### Пример

```bsl
// НЕПРАВИЛЬНО
Если Ошибка Тогда
    Перейти ~ОбработкаОшибки;
КонецЕсли;
// ...
~ОбработкаОшибки:
Сообщить("Ошибка");

// ПРАВИЛЬНО: через Если/Иначе или процедуру
Если Ошибка Тогда
    ОбработатьОшибку();
    Возврат;
КонецЕсли;
```

### Проверки
- АПК: 89, 249

## #std647: Имена процедур и функций

**ITS:** https://its.1c.ru/db/v8std/content/647/hdoc | **v8std:** https://v8std.ru/std/647/

### Правила
1. Имена берите от терминов предметной области — самодокументирующиеся.
2. Удаляйте пробелы, каждое слово с большой буквы, предлоги большими буквами.
3. Не используйте типы в названиях (кроме случаев, когда без них непонятно).

### Пример

```bsl
// НЕПРАВИЛЬНО: неинформативные имена
Функция ВыполнитьПроверку(Параметр1, Рекв, ТЗ)

// ПРАВИЛЬНО: имена из предметной области
Функция ПолучитьМассивыРеквизитов(
    ХозяйственнаяОперация,
    МассивВсехРеквизитов,
    МассивРеквизитовОперации)
```

### Диагностики BSL LS
- `FunctionNameStartsWithGet` — функция начинается с «Получить» без необходимости

## #std640: Параметры процедур и функций

**ITS:** https://its.1c.ru/db/v8std/content/640/hdoc | **v8std:** https://v8std.ru/std/640/

### Правила
1. Параметры именуйте как переменные; избегайте неявной передачи через переменные модуля.
2. Порядок — от общего к частному.
3. Необязательные параметры — после обязательных.

### Пример

```bsl
// ПРАВИЛЬНО: обязательные → необязательные
Процедура ПересчитатьСуммуДокумента(
    ИмяПоляСумма,           // обязательный
    ДокументОбъект,          // обязательный
    СуммаВключаетНДС = Истина)  // необязательный
```

### Диагностики BSL LS
- `ExtraCommas`, `MissedRequiredParameter`
- `NumberOfOptionalParams`, `NumberOfParams`
- `OrderOfParams`, `NestedFunctionInParameters`

## #std686: Работа с параметром «Отказ»

**ITS:** https://its.1c.ru/db/v8std/content/686/hdoc | **v8std:** https://v8std.ru/std/686/

### Правила
1. Не присваивайте `Отказ = Ложь` — можно сбросить ранее установленное `Истина` (в т.ч. неявно через булеву функцию).
2. Правило применимо к любым возвращаемым булевым параметрам (`СтандартнаяОбработка`, `Выполнение`).
3. При `Отказ = Истина` объясняйте пользователю причину через `СообщениеПользователю`.

### Пример

```bsl
// НЕПРАВИЛЬНО: может сбросить уже установленный Отказ = Истина
Отказ = ЕстьОшибки();

// ПРАВИЛЬНО: только установка Истина
Если ЕстьОшибки() Тогда
    Отказ = Истина;
КонецЕсли;
```

### Диагностики BSL LS
- `UsingCancelParameter` — неправильное использование `Отказ`

## #std693: Использование объектов типа Структура

**ITS:** https://its.1c.ru/db/v8std/content/693/hdoc | **v8std:** https://v8std.ru/std/693/

### Правила
1. Не передавайте в конструктор структуры более 3 значений — остальное через `Вставить`.
2. Не используйте вложенные конструкторы с параметрами.
3. Создавайте структуру со значениями по умолчанию сразу, а не добавляйте свойства в разных местах.

### Пример

```bsl
// НЕПРАВИЛЬНО: длинный конструктор
Параметры = Новый Структура(
    "НеПомещатьНастройкиВСхему, НеРедактироватьСхему, ...",
    Истина, Истина);

// ПРАВИЛЬНО
Параметры = Новый Структура;
Параметры.Вставить("НеПомещатьНастройкиВСхему", Истина);
Параметры.Вставить("НеРедактироватьСхему", Истина);
```

### Диагностики BSL LS
- `NumberOfValuesInStructureConstructor`

## #std782: Массовая конкатенация строк

**ITS:** https://its.1c.ru/db/v8std/content/782/hdoc | **v8std:** https://v8std.ru/std/782/

### Правила
1. При массовой конкатенации используйте `СтрРазделить()` и `СтрСоединить()` вместо `+` в цикле.

### Пример

```bsl
// НЕПРАВИЛЬНО: конкатенация в цикле
Текст = "";
Для НомерКолонки = 1 По Макет.ШиринаТаблицы Цикл
    Текст = Текст + ТекстОбласти + Символы.ПС;
КонецЦикла;

// ПРАВИЛЬНО: массив + СтрСоединить
МассивСтрок = Новый Массив;
Для НомерКолонки = 1 По Макет.ШиринаТаблицы Цикл
    МассивСтрок.Добавить(ТекстОбласти);
КонецЦикла;
ИзвлеченныйТекст = СтрСоединить(МассивСтрок, Символы.ПС);
```

## #std409: РеквизитФормыВЗначение vs ДанныеФормыВЗначение

**ITS:** https://its.1c.ru/db/v8std/content/409/hdoc | **v8std:** https://v8std.ru/std/409/

### Правила
1. В модулях форм используйте `РеквизитФормыВЗначение()` вместо `ДанныеФормыВЗначение()` — проще синтаксис, меньше ошибок.

### Пример

```bsl
// РЕКОМЕНДУЕТСЯ
ДанныеДляОбработки = РеквизитФормыВЗначение("Объект");

// ВМЕСТО
ДанныеФормы = ДанныеФормыВЗначение(Объект, Тип("ДокументОбъект.Реализация"));
```

### Диагностики BSL LS
- `FormDataToValue` — использование устаревшего `ДанныеФормыВЗначение`

## #std544: Ограничения на экспортные процедуры

**ITS:** https://its.1c.ru/db/v8std/content/544/hdoc | **v8std:** https://v8std.ru/std/544/

### Правила
1. Не размещайте экспортные процедуры/функции в модулях команд и общих командах.
2. К этим модулям нельзя обратиться из внешнего кода — экспорт там не имеет смысла.

### Проверки
- EDT: `export-method-in-command-form-module`
- АПК: 250
