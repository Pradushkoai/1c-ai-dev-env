## #std688: Обработчики регламентных заданий

**ITS:** https://its.1c.ru/db/v8std/content/688/hdoc

### Суть
Обработчики регламентных заданий выполняются в фоновом сеансе без пользователя — должны быть устойчивыми к ошибкам и блокировкам.

### Правила
1. Тело обработчика — обязательно в `Попытка/Исключение`
2. Все ошибки писать в ЖурналРегистрации с уровнем `Ошибка`
3. Не использовать интерактивные вызовы (`ПоказатьПредупреждение`, `Вопрос`)
4. Не рассчитывать на текущего пользователя — сеанс фоновый
5. Учитывать возможность параллельного выполнения
6. Не запускать `ЭтотОбъект.Записать()` безусловно

### Пример правильного обработчика

```bsl
Процедура РассчитатьСебестоимостьРегламентное() Экспорт
    Попытка
        // Проверка блокировки — не запускать параллельно
        Блокировка = Новый БлокировкаДанных;
        Блокировка.УстановитьДанные("РегистрНакопления.Себестоимость");
        Если Не Блокировка.Заблокировать() Тогда
            ЗаписьЖурналаРегистрации(
                "Регламент.Себестоимость",
                УровеньЖурналаРегистрации.Информация,
                , , "Уже выполняется в другом сеансе"
            );
            Возврат;
        КонецЕсли;
        
        // Основная логика в общем модуле
        СебестоимостьСервер.РассчитатьЗаПериод(НачалоМесяца, КонецМесяца);
        
        // Успешное завершение
        ЗаписьЖурналаРегистрации(
            "Регламент.Себестоимость",
            УровеньЖурналаРегистрации.Информация,
            , , "Завершено успешно"
        );
    Исключение
        ЗаписьЖурналаРегистрации(
            "Регламент.Себестоимость",
            УровеньЖурналаРегистрации.Ошибка,
            , , ОписаниеОшибки()
        );
        ВызватьИсключение;  // чтобы 1С показала ошибку в журнале заданий
    КонецПопытки;
КонецПроцедуры
```

## #std689: Расписание регламентных заданий

**ITS:** https://its.1c.ru/db/v8std/content/689/hdoc

### Суть
Расписание должно учитывать пиковые нагрузки и взаимные блокировки.

### Правила
- Не ставить "тяжёлые" задания на начало часа (много конкуренции за ресурсы)
- Использовать смещение: 00:13, 00:27, 00:43 — а не 00:00, 00:15, 00:30
- Разносить тяжёлые задания по времени с запасом
- Ночные регламенты — после 22:00, до 6:00 (если нет ночных пользователей)
- Учитывать часовой пояс сервера (не пользователя)

### Типовое расписание

| Задание | Время | Периодичность |
|---|---|---|
| Рассчёт себестоимости | 01:17 | Ежедневно |
| Обновление индексов | 02:33 | Ежедневно |
| Резервное копирование | 03:00 | Ежедневно |
| Синхронизация с сайтом | каждые 13 минут | Fixed rate |
| Закрытие месяца | последнее число месяца, 23:50 | Ежемесячно |
| Регламент ПДВ | 5 число месяца, 00:00 | Ежемесячно |

## #std690: Фоновые задания и асинхронность

**ITS:** https://its.1c.ru/db/v8std/content/690/hdoc

### Суть
Фоновые задания позволяют выполнять длительные операции без блокировки UI.

### Правила
- Длительные операции (> 5 секунд) — выносить в фоновое задание
- Использовать `ФоновыеЗадания.Выполнить()` для запуска
- Идентификатор задания — сохранять в регистр сведений для отслеживания
- Не запускать больше заданий, чем ядер на сервере
- Проверять статус предыдущего запуска перед новым

### Пример

```bsl
// Запуск фонового задания
Функция ЗапуститьПересчетДолгов(Контрагент) Экспорт
    Параметры = Новый Массив;
    Параметры.Добавить(Контрагент);
    
    // Проверка — не запущен ли уже
    Если ДолгиФоновый.УжеЗапущен(Контрагент) Тогда
        Возврат Неопределено;
    КонецЕсли;
    
    Задание = ФоновыеЗадания.Выполнить(
        "ДолгиСервер.ПересчитатьДолги",
        Параметры,
        ,  // ключ
        "Пересчет долгов: " + Строка(Контрагент)
    );
    
    // Сохранить идентификатор
    ДолгиФоновый.ЗаписатьЗадание(Задание.УникальныйИдентификатор, Контрагент);
    
    Возврат Задание.УникальныйИдентификатор;
КонецФункции

// Проверка результата
Функция ПроверитьЗавершение(Идентификатор) Экспорт
    Задание = ФоновыеЗадания.НайтиПоУникальномуИдентификатору(Идентификатор);
    Если Задание = Неопределено Тогда
        Возврат Неопределено;
    КонецЕсли;
    
    Если Задание.Состояние = СостояниеФоновогоЗадания.Завершено Тогда
        Возврат Истина;
    ИначеЕсли Задание.Состояние = СостояниеФоновогоЗадания.Отменено Тогда
        Возврат Ложь;
    Иначе
        Возврат Неопределено;  // ещё выполняется
    КонецЕсли;
КонецФункции
```

## #std691: Взаимные блокировки регламентных заданий

**ITS:** https://its.1c.ru/db/v8std/content/691/hdoc

### Суть
Параллельные регламентные задания могут создавать взаимные блокировки (deadlock).

### Правила
- Использовать упорядоченный доступ к ресурсам (например, по ИД объектов)
- Не выполнять несколько регламентов на одних и тех же таблицах одновременно
- При взаимной блокировке — одна из сторон должна освободить ресурсы
- Использовать таймауты блокировок

### Пример упорядоченной блокировки

```bsl
// ПЛОХО: два регламента блокируют одни таблицы в разном порядке
// Регламент 1: Сначала Контрагенты → потом Договоры
// Регламент 2: Сначала Договоры → потом Контрагенты
// Результат: взаимная блокировка

// ПРАВИЛЬНО: одинаковое порядочное блокирование
// Регламент 1 и Регламент 2 оба: Контрагенты → Договоры
Процедура Обработать()
    Блокировка = Новый БлокировкаДанных;
    // Сначала Контрагенты
    ЭлементБлокировки = Блокировка.Добавить("Справочник.Контрагенты");
    ЭлементБлокировки.Режим = РежимБлокировкиДанных.Исключительный;
    // Потом Договоры
    ЭлементБлокировки = Блокировка.Добавить("Справочник.ДоговорыКонтрагентов");
    ЭлементБлокировки.Режим = РежимБлокировкиДанных.Исключительный;
    Блокировка.Заблокировать();
    // ...
КонецПроцедуры
```

## #std692: Мониторинг регламентных заданий

**ITS:** https://its.1c.ru/db/v8std/content/692/hdoc

### Суть
Должен быть мониторинг состояния регламентных заданий и оповещение администратора.

### Правила
- Вести журнал выполнения (свои записи в ЖурналРегистрации)
- При ошибке — оповещение администратора (email, и т.д.)
- Автоматический перезапуск при "зависших" заданиях
- Проверка времени выполнения — если стало > 2x обычного — тревога

### Пример мониторинга

```bsl
Процедура ПроверитьРегламентыРегламентное() Экспорт
    // Найти все "зависшие" задания (выполняются > 2 часов)
    Запрос = Новый Запрос;
    Запрос.Текст =
        "ВЫБРАТЬ
        |    Задания.УникальныйИдентификатор КАК УИД,
        |    Задания.Наименование КАК Имя,
        |    Задания.Начало КАК Начало
        |ИЗ
        |    РегистрСведений.РегламентныеЗадания КАК Задания
        |ГДЕ
        |    Задания.Начало < &ДваЧасаНазад
        |    И Задания.Окончание ЕСТЬ NULL";
    Запрос.УстановитьПараметр("ДваЧасаНазад", ТекущаяДатаСеанса() - 7200);
    
    Результат = Запрос.Выполнить();
    Если Не Результат.Пустой() Тогда
        Текст = "Зависшие регламентные задания:" + Символы.ПС;
        Выборка = Результат.Выбрать();
        Пока Выборка.Следующий() Цикл
            Текст = Текст + "- " + Выборка.Имя + " (старт: " + Выборка.Начало + ")" + Символы.ПС;
        КонецЦикла;
        
        // Оповестить администратора
        УведомленияСервер.ОтправитьEmail("admin@company.ru", "Тревога: регламенты", Текст);
        
        // Отменить зависшие
        Выборка.Первый();
        Пока Выборка.Следующий() Цикл
            ФоновыеЗадания.НайтиПоУникальномуИдентификатору(Выборка.УИД).Отменить();
        КонецЦикла;
    КонецЕсли;
КонецПроцедуры
```

## #std693: Использование фоновых заданий в формах

**ITS:** https://its.1c.ru/db/v8std/content/693/hdoc

### Суть
Запуск фоновых заданий из форм — особый случай, требующий обработки результата.

### Правила
- Идентификатор задания — сохранять в реквизит формы
- При закрытии формы — отменять задание (если ещё выполняется)
- Проверять состояние через `ПодключитьОбработчикОжидания` (каждые 3-5 секунд)
- При завершении — обновить данные формы

### Пример

```bsl
&НаКлиенте
Процедура НачатьПересчет()
    Идентификатор = ДолгиФоновый.ЗапуститьПересчет(Объект.Контрагент);
    Если Идентификатор <> Неопределено Тогда
        Элементы.КнопкаПересчет.Видимость = Ложь;
        Элементы.Индикатор.Видимость = Истина;
        ПодключитьОбработчикОжидания("ПроверитьПересчет", 3);
    КонецЕсли;
КонецПроцедуры

&НаКлиенте
Процедура ПроверитьПересчет()
    Состояние = ДолгиФоновый.ПроверитьЗавершение(Идентификатор);
    Если Состояние = Истина Тогда
        ОтключитьОбработчикОжидания("ПроверитьПересчет");
        Элементы.КнопкаПересчет.Видимость = Истина;
        Элементы.Индикатор.Видимость = Ложь;
        ПоказатьПредупреждение(, "Пересчёт завершён");
        ОбновитьОтображениеДанных();
    ИначеЕсли Состояние = Ложь Тогда
        ОтключитьОбработчикОжидания("ПроверитьПересчет");
        Элементы.КнопкаПересчет.Видимость = Истина;
        Элементы.Индикатор.Видимость = Ложь;
        ПоказатьПредупреждение(, "Ошибка пересчёта (см. журнал)");
    КонецЕсли;
КонецПроцедуры
```

## #std694: Антипаттерны регламентных заданий

### 1. Без обработки ошибок

```bsl
// ПЛОХО: при ошибке — задание упадёт молча
Процедура Регламент() Экспорт
    РассчитатьВсё();  // если упадёт — никто не узнает
КонецПроцедуры

// ПРАВИЛЬНО: с Попытка/Исключение и журналом
Процедура Регламент() Экспорт
    Попытка
        РассчитатьВсё();
    Исключение
        ЗаписьЖурналаРегистрации("Регламент", УровеньЖурналаРегистрации.Ошибка,
            , , ОписаниеОшибки());
    КонецПопытки;
КонецПроцедуры
```

### 2. Интерактивный вызов

```bsl
// НЕВОЗМОЖНО в фоновом задании
Процедура Регламент() Экспорт
    Если Вопрос("Продолжить?", РежимДиалогаВопрос.ДаНет) = КодВозвратаДиалога.Нет Тогда
        Возврат;
    КонецЕсли;
КонецПроцедуры
// Ошибка: в фоновом сеансе нет UI
```

### 3. Бесконечный цикл

```bsl
// ПЛОХО: регламент повиснет навсегда
Пока Истина Цикл
    ОбработатьОчередь();
КонецЦикла;

// ПРАВИЛЬНО: ограничение по времени
ДатаНачала = ТекущаяДатаСеанса();
Пока ТекущаяДатаСеанса() - ДатаНачала < 300 Цикл  // не более 5 минут
    Если Не ОбработатьОчередь() Тогда
        Прервать;  // очередь пуста
    КонецЕсли;
КонецЦикла;
```

### 4. Блокировка без таймаута

```bsl
// ПЛОХО: если блокировку не получить — повиснет
Блокировка.Заблокировать();

// ПРАВИЛЬНО: с таймаутом
Попытка
    Блокировка.Заблокировать(10);  // 10 секунд
Исключение
    ЗаписьЖурналаРегистрации("Регламент", УровеньЖурналаРегистрации.Предупреждение,
        , , "Не удалось получить блокировку");
    Возврат;
КонецПопытки;
```
