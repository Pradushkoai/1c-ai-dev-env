# Стандарты 1С: Создание и изменение объектов метаданных

**Источник:** [ITS: Создание и изменение объектов метаданных](https://its.1c.ru/db/v8std/browse/13/-1/26/22) | [v8std.ru](https://v8std.ru)  
**Связанные инструменты:** `get_object_structure`, `inspect`, `check_standards`, `dsl_compile_meta`

---

## #std452: Имена объектов метаданных в конфигурациях

**ITS:** https://its.1c.ru/db/v8std/content/452/hdoc

### Правила
- Имена — на русском, осмысленные
- Не более 80 символов
- Без пробелов, спецсимволов (только буквы, цифры, `_`)
- Не использовать зарезервированные слова
- Имя справочника — существительное во множ. числе (`Товары`, `Контрагенты`)
- Имя документа — отглагольное существительное (`ЗаказКлиента`, `ПоступлениеТоваров`)

**BSL LS:** `ForbiddenMetadataName`, `MetadataObjectNameLength`, `SameMetadataObjectAndChildNames`

## #std460: Использование подсистем

**ITS:** https://its.1c.ru/db/v8std/content/460/hdoc

### Правила
- Подсистемы — для логической группировки объектов
- Иерархия не более 3 уровней
- Имена — существительные во множ. числе
- Каждый объект — минимум в одной подсистеме
- Подсистемы верхнего уровня — по функциональным областям

---

## #std463: Использование общих реквизитов

**ITS:** https://its.1c.ru/db/v8std/content/463/hdoc

### Правила
- Общие реквизиты — только для данных, общих для всех объектов
- Например: `Организация`, `Подразделение`
- Не использовать для редко используемых полей
- Учитывать влияние на производительность (дополнительное поле в каждой таблице)

---

## #std464: Использование определяемых типов

**ITS:** https://its.1c.ru/db/v8std/content/464/hdoc

### Правила
- Определяемый тип — для полей, тип которых зависит от конфигурации
- Например: `СсылкаНаДокументПоступления` — может быть разным в разных конфигурациях
- Имя — существительное, описывающее сущность
- Не использовать для примитивных типов (строка, число)

---

## #std467: Правила создания общих модулей

**ITS:** https://its.1c.ru/db/v8std/content/467/hdoc

### Правила
- Один общий модуль — одна функциональная область
- Именование по суффиксам: `Кэшируемый`, `Клиент`, `КлиентСервер`, `ВызовСервера`, `Глобальный`, `ПолныеПрава`
- Не более 500 строк в модуле (рекомендуется < 300)
- Экспортные методы — только в области ПрограммныйИнтерфейс
- Не использовать переменные уровня модуля (особенно для ПовтИсп)

**BSL LS:** `CommonModuleNameCached`, `CommonModuleNameClient`, `CommonModuleNameClientServer`, `CommonModuleNameFullAccess`, `CommonModuleNameGlobal`, `CommonModuleNameServerCall`, `CommonModuleNameWords`, `CommonModuleMissingAPI`, `CommonModuleVariables`

---

## #std468: Использование функциональных опций

**ITS:** https://its.1c.ru/db/v8std/content/468/hdoc

### Правила
- Функциональные опции — для включения/выключения функциональности
- Имя — существительное, описывающее функциональность
- Не более 50 функциональных опций в конфигурации
- Не использовать для редко меняющихся настроек

---

## #std469: Использование параметров сеанса

**ITS:** https://its.1c.ru/db/v8std/content/469/hdoc

### Правила
- Параметры сеанса — для данных текущего пользователя/сеанса
- Например: `ТекущийПользователь`, `ТекущаяОрганизация`
- Не использовать для редко используемых данных
- Устанавливать при начале сеанса

---

## #std470: Версия платформы для разработки

**ITS:** https://its.1c.ru/db/v8std/content/470/hdoc

### Правила
- Разработка — на минимальной поддерживаемой версии платформы
- Тестирование — на всех поддерживаемых версиях
- Не использовать возможности только последней версии без проверки
