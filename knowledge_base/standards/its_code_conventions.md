# Стандарты 1С: Соглашения при написании кода

**Источник:** [ITS: Соглашения при написании кода](https://its.1c.ru/db/v8std/browse/13/-1/26/24) | [v8std.ru](https://v8std.ru)  
**Связанные инструменты:** `check_standards`, `analyze_bsl`, `solve_check`, `get_code_metrics`

---

## #std456: Оформление модулей

**ITS:** https://its.1c.ru/db/v8std/content/456/hdoc

### Правила
- Каждая процедура/функция — отдельная логическая единица
- Области кода обязательны: `#Область ... #КонецОбласти`
- Порядок областей: ПрограммныйИнтерфейс → СлужебныйПрограммныйИнтерфейс → СлужебныеПроцедурыИФункции → Инициализация
- Не более 200 строк в процедуре (рекомендуется < 50)
- Отступы — табуляция

**BSL LS:** `CodeOutOfRegion`, `DuplicateRegion`, `EmptyRegion`

---

## #std457: Тексты модулей

**ITS:** https://its.1c.ru/db/v8std/content/457/hdoc

### Правила
- Кодировка UTF-8 с BOM
- Перевод строк CRLF (Windows) или LF (Linux)
- Не использовать символы табуляции в строковых литералах
- Комментарии — на русском языке

---

## #std458: Структура модуля

**ITS:** https://its.1c.ru/db/v8std/content/458/hdoc

### Правила
- Заголовок модуля — краткое описание
- Области в строгом порядке
- Экспортные методы — в начале
- Переменные модуля — только в области Инициализация
- Не более 1000 строк в модуле (рекомендуется < 500)

### Структура
```bsl
// Модуль: ИмяМодуля
// Назначение: ...

#Область ПрограммныйИнтерфейс
// Экспортные процедуры и функции
#КонецОбласти

#Область СлужебныйПрограммныйИнтерфейс
// Служебные экспортные
#КонецОбласти

#Область СлужебныеПроцедурыИФункции
// Внутренние методы
#КонецОбласти

#Область Инициализация
// Переменные, инициализация
#КонецОбласти
```
**BSL LS:** `CodeOutOfRegion`, `CodeBlockBeforeSub`, `CommonModuleVariables`

## #std502: Структуры и таблицы значений как параметры

**ITS:** https://its.1c.ru/db/v8std/content/502/hdoc

### Правила
- Использовать `Знач` для параметров-структур/таблиц если не нужно изменять
- Не передавать `ТаблицаЗначений` между клиентом и сервером — использовать массив структур
- Для параметров по умолчанию — `Знач = Неопределено`

## #std503: Работа с параметром "Отказ"

**ITS:** https://its.1c.ru/db/v8std/content/503/hdoc

### Правила
- `Отказ = Истина` — только при ошибке, не сбрасывать
- Не проверять `Отказ` в обработчике — накапливать
- В конце обработчика — проверка `Если Отказ Тогда Возврат;`
- Не использовать `Отказ = Ложь` — может затереть чужую ошибку

### Пример
```bsl
// ПРАВИЛЬНО
Процедура ОбработкаПроверкиЗаполнения(Отказ, ПроверяемыеРеквизиты)
    Если Не ЗначениеЗаполнено(Реквизит1) Тогда
        Отказ = Истина;
        Сообщить("Не заполнен реквизит");
    КонецЕсли;
    // Не проверяем Отказ — накапливаем ошибки
    Если Не ЗначениеЗаполнено(Реквизит2) Тогда
        Отказ = Истина;
    КонецЕсли;
КонецПроцедуры
```

## #std504: Использование дублирующего кода

**ITS:** https://its.1c.ru/db/v8std/content/504/hdoc

### Правила
- Не дублировать код — выносить в общий модуль
- Использовать `ОбщегоНазначения.` методы из БСП
- При 3+ повторениях — обязательный рефакторинг

---

## #std506: Получение метаданных объектов

**ITS:** https://its.1c.ru/db/v8std/content/506/hdoc

### Правила
- Не использовать `Метаданные.Справочники.Имя` в коде — использовать `Метаданные.НайтиПоИмени()`
- Не кэшировать метаданные в переменных модуля
- Использовать `Метаданные.Имя` вместо строковых литералов

---

## #std508: Использование переменных в программных модулях

**ITS:** https://its.1c.ru/db/v8std/content/508/hdoc

### Правила
- Не использовать переменные уровня модуля в общих модулях
- Локальные переменные — объявлять в начале процедуры
- Имена переменных — осмысленные, не `А`, `Б`, `В`
- Префиксы: `мИмя` — переменная модуля, `лИмя` — локальная (опционально)

---

## #std510: Перехват исключений в коде

**ITS:** https://its.1c.ru/db/v8std/content/510/hdoc

### Правила
- `Попытка/Исключение` — только когда есть что перехватывать
- Не использовать пустой `Исключение` — логировать ошибку
- `ОписаниеОшибки()` — для логирования
- `ВызватьИсключение` — для проброса выше
- Не подавлять ошибки без необходимости

### Пример
```bsl
// ПРАВИЛЬНО
Попытка
    Результат = ВыполнитьОперацию();
Исключение
    ЗаписьЖурналаРегистрации("Ошибка", УровеньЖурналаРегистрации.Ошибка, , , ОписаниеОшибки());
    ВызватьИсключение;
КонецПопытки;

// ПЛОХО: пустое исключение
Попытка
    Результат = ВыполнитьОперацию();
Исключение
КонецПопытки;
```

## #std511: Вызов исключений в коде

**ITS:** https://its.1c.ru/db/v8std/content/511/hdoc

### Правила
- `ВызватьИсключение "Сообщение"` — для пользовательских ошибок
- Не использовать `ВызватьИсключение` без сообщения
- Текст исключения — на русском, понятный пользователю

---

## #std559: Использование объектов типа Структура

**ITS:** https://its.1c.ru/db/v8std/content/559/hdoc

### Правила
- Структура — для передачи именованных параметров
- Не использовать для хранения большого количества данных (лучше ТаблицаЗначений)
- Ключи — осмысленные, без пробелов
- `Свойство()` — для проверки существования ключа

---

## #std567: Использование модуля объекта, модуля менеджера и общих модулей

**ITS:** https://its.1c.ru/db/v8std/content/567/hdoc

### Правила
- **Модуль объекта** — бизнес-логика объекта (обработчики, проверка заполнения)
- **Модуль менеджера** — фабричные методы (создание, поиск, итоги)
- **Общий модуль** — переиспользуемая логика (кросс-объектная)
- Не дублировать логику между модулем объекта и менеджера

---

## #std580: Минимизация кода, выполняемого на клиенте

**ITS:** https://its.1c.ru/db/v8std/content/580/hdoc

### Правила
- На клиенте — только UI-логика (открытие форм, сообщения)
- Бизнес-логика — на сервере (`&НаСервере`)
- Тяжёлые операции — только на сервере
- Использовать `&НаСервереБезКонтекста` для уменьшения трафика

---

## #std581: Доступ к файловой системе из кода

**ITS:** https://its.1c.ru/db/v8std/content/581/hdoc

### Правила
- На сервере — ограничить доступ к файловой системе
- На клиенте — только через диалог выбора файла
- Не использовать абсолютные пути — только через `КаталогВременныхФайлов()`
- Временные файлы — удалять после использования
