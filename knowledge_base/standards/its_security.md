# Стандарты 1С: Безопасность

**Источник:** [ITS: Общие вопросы безопасности](https://its.1c.ru/db/v8std/browse/13/-1/26/30) | [v8std.ru](https://v8std.ru)  
**Связанные инструменты:** `audit_security`, `solve_check`, `check_standards`

## #std482: Безопасность прикладного программного интерфейса сервера

**ITS:** https://its.1c.ru/db/v8std/content/482/hdoc

### Правила
- Все экспортные методы серверных общих модулей — проверять параметры
- Не передавать необработанный пользовательский ввод в `Выполнить()`
- Использовать `СтрНайти`, `СтрЗаменить` для санитизации
- Не доверять данным из внешних источников (HTTP-сервисы, файлы)

**BSL LS:** `DisableSafeMode`, `UnsafeSafeModeMethodCall`

## #std483: Ограничение на установку признака «Вызов сервера»

**ITS:** https://its.1c.ru/db/v8std/content/483/hdoc

### Правила
- `ВызовСервера` — только для общих модулей, НЕ для форм
- Минимум экспортных методов в модуле с `ВызовСервера`
- Каждый метод — проверять входные параметры
- Не использовать `ВызовСервера` для тяжёлых операций

**BSL LS:** `CommonModuleNameServerCall`

---

## #std484: Безопасное хранение паролей

**ITS:** https://its.1c.ru/db/v8std/content/484/hdoc

### Правила
- НЕ хранить пароли в коде (`Константа = "пароль"`)
- Использовать `ХранилищеЗначения` с шифрованием
- Пароли в БД — только хэш (не открытый текст)
- Использовать `БезопасноеХранилищеДанных` (если доступно)

**BSL LS:** поиск хардкода паролей в security_auditor (SEC005-SEC008)

---

## #std485: Ограничение на выполнение "внешнего" кода

**ITS:** https://its.1c.ru/db/v8std/content/485/hdoc

### Правила
- `Выполнить()` — ТОЛЬКО с `УстановитьБезопасныйРежим(Истина)`
- `ВыполнитьПроцедуру()` — проверять имя процедуры
- Не выполнять код из непроверенных источников
- Внешние обработки — проверять через `БезопасноеХранилищеДанных`

**BSL LS:** `ExecuteExternalCodeInCommonModule`, `DisableSafeMode`

---

## #std486: Использование безопасного режима

**ITS:** https://its.1c.ru/db/v8std/content/486/hdoc

### Правила
- `УстановитьБезопасныйРежим(Истина)` — перед `Выполнить()`
- Не отключать `УстановитьБезопасныйРежим(Ложь)` без необходимости
- Если отключили — включить обратно сразу после операции
- Безопасный режим запрещает: COM, файловая система, shell

---

## #std487: Использование привилегированного режима

**ITS:** https://its.1c.ru/db/v8std/content/487/hdoc

### Правила
- `УстановитьПривилегированныйРежим(Истина)` — только в общих модулях с `ПолныеПрава`
- Минимум кода в привилегированном режиме
- Не использовать для обхода RLS
- После операции — отключить

**BSL LS:** `SetPrivilegedMode`, `PrivilegedModuleMethodCall`

## Дополнительные правила безопасности (из security_auditor.py)

### SEC001: SQL-инъекция — конкатенация строк в запросе
```bsl
// ПЛОХО
ТекстЗапроса = "ВЫБРАТЬ ... ГДЕ Код = """ + Код + """";
// ПРАВИЛЬНО
ТекстЗапроса = "ВЫБРАТЬ ... ГДЕ Код = &Код";
Запрос.УстановитьПараметр("Код", Код);
```

### SEC002: Выполнить() без безопасного режима
```bsl
// ПЛОХО
Выполнить(ВнешнийКод);
// ПРАВИЛЬНО
УстановитьБезопасныйРежим(Истина);
Выполнить(ВнешнийКод);
УстановитьБезопасныйРежим(Ложь);
```

### SEC003: COM-объекты без необходимости
- COM-объекты — только в безопасном контексте
- Предпочитать нативные методы

### SEC004: Path traversal
- Проверять пути файлов на `..`
- Использовать `СтрНайти(Путь, "..")` для проверки

### SEC005-SEC008: Хардкод паролей/токенов
- Не хранить пароли в коде
- Использовать `ХранилищеЗначения` или `БезопасноеХранилищеДанных`

### SEC009-SEC015: Прочие правила
- `Exec` внешних программ — только с проверкой
- `НайтиПоКоду` — проверять результат
- `ЗаписьЖурналаРегистрации` — не логировать чувствительные данные
- `ВременныеФайлы` — удалять после использования
- `XMLParser` — проверять на XXE
- `HTTPСоединение` — проверять SSL сертификаты
- `ЗапретитьОбменДанными` — не отключать без необходимости
