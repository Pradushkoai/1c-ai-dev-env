**Источник:** [v8std.ru — Разработка пользовательских интерфейсов](https://v8std.ru) | [ITS 1С](https://its.1c.ru/db/v8std/browse/13/-1/26/26)  
**Связанные инструменты:** `check_standards`, `check_form_quality`, `analyze_bsl`, `solve_check`

> Полный каталог стандартов 1С по разработке UI: информирование пользователя, Сообщить vs СообщениеПользователю, длительные операции, печатные формы, командный интерфейс. Стандарты #std400, #std418, #std642, #std755, #std548, #std789, #std711, #std396, #std466.

## #std400: Информирование пользователя

**ITS:** https://its.1c.ru/db/v8std/content/400/hdoc | **v8std:** https://v8std.ru/std/400/

### Правила
1. Ошибки проверки заполнения выводите в панель сообщений формы через `СообщениеПользователю`.
2. Проверку реквизитов — вне транзакции записи; проверку целостности — в транзакции.
3. Используйте асинхронные методы `ПоказатьВопрос`, `ПоказатьПредупреждение`.

### Пример

```bsl
Оповещение = Новый ОписаниеОповещения("ПослеЗакрытияВопроса", ЭтотОбъект);
ПоказатьВопрос(Оповещение,
    НСтр("ru = 'В этом месяце еще не задан курс валюты. Использовать курс за прошлый месяц?'"),
    РежимДиалогаВопрос.ДаНет);
```

## #std418: Ограничение на использование метода Сообщить

**ITS:** https://its.1c.ru/db/v8std/content/418/hdoc | **v8std:** https://v8std.ru/std/418/

### Правила
1. Для вывода сообщений используйте `СообщениеПользователю`, даже когда сообщение не привязано к элементу формы.
2. Метод `Сообщить()` применять не следует.
3. С БСП — `ОбщегоНазначения.СообщитьПользователю()`.

### Пример

```bsl
// НЕПРАВИЛЬНО: Сообщить
Сообщить("Не заполнен реквизит");

// ПРАВИЛЬНО: СообщениеПользователю
Сообщение = Новый СообщениеПользователю;
Сообщение.Текст = НСтр("ru = 'Не заполнен реквизит Контрагент.'");
Сообщение.Поле = "Контрагент";
Сообщение.УстановитьДанные(Объект);
Сообщение.Сообщить();
```

### Диагностики BSL LS
- `DeprecatedMessage` — использование устаревшего `Сообщить`

## #std642: Длительные операции на сервере

**ITS:** https://its.1c.ru/db/v8std/content/642/hdoc | **v8std:** https://v8std.ru/std/642/

### Правила
1. Если серверный вызов может выполняться дольше 8 секунд — выполняйте асинхронно через фоновое задание.
2. Код длительной обработки — в модуле менеджера; результат — во временное хранилище.
3. С БСП — модули `ДлительныеОперации`, `ДлительныеОперацииКлиент`.

### Пример

```bsl
// В модуле менеджера
Функция ОпределитьНастройкиУчетнойЗаписи(АдресЭлектроннойПочты, Пароль) Экспорт
    // длительная обработка — например, проверка соединения с почтовым сервером
    Возврат Настройки;
КонецФункции

// В форме
&НаКлиенте
Процедура НачатьПроверку(Команда)
    ПараметрыВыполнения = ДлительныеОперацииКлиент.ПараметрыВыполнения(УникальныйИдентификатор);
    Оповещение = Новый ОписаниеОповещения("ПослеПроверки", ЭтотОбъект);
    ДлительныеОперацииКлиент.НачатьВыполнениеНаСервере(
        Оповещение, "ДлительныеОперации.ОпределитьНастройкиУчетнойЗаписи", Параметры);
КонецПроцедуры
```

### Диагностики BSL LS
- `MissingTempStorageDeletion` — не удалено временное хранилище

## #std755: Длительные операции на клиенте

**ITS:** https://its.1c.ru/db/v8std/content/755/hdoc | **v8std:** https://v8std.ru/std/755/

### Правила
1. Не выполняйте потенциально длительные клиентские операции в обработчиках `ПриИзменении` и при открытии формы.
2. Получение информации об обновлениях — по кнопке или в регламентном задании на сервере.
3. Проверку доступности сетевого ресурса — в обработчике команды, а не `ПриИзменении`.

## #std548: Формирование печатных форм

**ITS:** https://its.1c.ru/db/v8std/content/548/hdoc | **v8std:** https://v8std.ru/std/548/

### Правила
1. Для печати достаточно прав на чтение данных и выполнение команды печати.
2. Поддерживайте множественную печать; выборку делайте одним запросом.
3. Печатная и экранная формы должны совпадать по составу и порядку строк табличных частей.

### Пример

```bsl
ТабДокумент = Новый ТабличныйДокумент;
ТабДокумент.ПолеСверху = 0;
ТабДокумент.ПолеСлева = 0;
ТабДокумент.ПолеСнизу = 0;
ТабДокумент.ПолеСправа = 0;
ТабДокумент.ОриентацияСтраницы = ОриентацияСтраницы.Портрет;
```

## #std789: Печатные формы с учётом изменений макета

**ITS:** https://its.1c.ru/db/v8std/content/789/hdoc | **v8std:** https://v8std.ru/std/789/

### Правила
1. При печати через БСП (макет СКД) сохраняйте состав полей в макетах `ДанныеПечати`.
2. Можно расширять состав полей, но не переименовывать/удалять.
3. Код вывода должен быть устойчив к изменениям макета.

## #std711: Общие принципы построения командного интерфейса

**ITS:** https://its.1c.ru/db/v8std/content/711/hdoc | **v8std:** https://v8std.ru/std/711/

### Правила
1. Количество элементов в панелях и меню — чтобы помещались без прокрутки при стандартном разрешении.
2. Элементы располагайте по убыванию важности и частоты использования.
3. Не размещайте рядом команды с совпадающими первыми символами в названии.

## #std396: Обработчик ОбработкаЗаполнения

**ITS:** https://its.1c.ru/db/v8std/content/396/hdoc | **v8std:** https://v8std.ru/std/396/

### Правила
1. Ограничения ввода на основании проверяйте в `ОбработкаЗаполнения` модуля объекта.
2. Для отказа — `ВызватьИсключение` с понятным текстом.
3. Сначала специальное заполнение по типу `ДанныеЗаполнения`, затем общее.

### Пример

```bsl
Процедура ОбработкаЗаполнения(ДанныеЗаполнения, ТекстЗаполнения, СтандартнаяОбработка)
    Если ТипЗнч(ДанныеЗаполнения) = Тип("СправочникСсылка.Сотрудники") Тогда
        Если ДанныеЗаполнения.ЭтоГруппа Тогда
            ВызватьИсключение НСтр("ru = 'Ввод приказа на основании группы невозможен.'");
        КонецЕсли;
    КонецЕсли;
КонецПроцедуры
```

## #std466: Обработчик ПриКопировании

**ITS:** https://its.1c.ru/db/v8std/content/466/hdoc | **v8std:** https://v8std.ru/std/466/

### Правила
1. В `ПриКопировании` выполняются действия при создании объекта копированием.
2. Здесь очищают реквизиты, значения которых не должны сохраняться в новом объекте (дата, номер, проведено).

### Пример

```bsl
Процедура ПриКопировании(ОбъектКопирования)
    // Очистить реквизиты, не наследуемые при копировании
    Дата = ТекущаяДатаСеанса();
    Ответственный = ПараметрыСеанса.ТекущийПользователь;
    Проведен = Ложь;
    ПометкаУдаления = Ложь;
КонецПроцедуры
```
