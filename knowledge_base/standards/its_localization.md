# Стандарты 1С: Локализация

**Источник:** [v8std.ru — Требования по локализации](https://v8std.ru) | [ITS 1С](https://its.1c.ru/db/v8std/browse/13/-1/26/26/29)  
**Связанные инструменты:** `check_standards`, `analyze_bsl`, `solve_check`, `audit_security`

> Полный каталог стандартов 1С по локализации: форматирование дат/чисел/Булево, интерфейсные тексты, макеты, регламентные задания, денежные поля, автогенерированные данные, международная версия. Стандарты #std761, #std763, #std764, #std765, #std766, #std767, #std769, #std778, #std784.

## #std761: Интерфейсные тексты в коде

**ITS:** https://its.1c.ru/db/v8std/content/761/hdoc | **v8std:** https://v8std.ru/std/761/

### Правила
1. Все строки для интерфейса (сообщения, надписи, тексты команд, выражения СКД) оборачивайте в `НСтр` вместо прямых литералов.
2. Для составных строк используйте `СтрШаблон` с логически завершёнными фразами (разный порядок слов в языках).
3. В `НСтр` строку ограничивайте одинарными кавычками.

### Пример

```bsl
ПоказатьПредупреждение(,
    НСтр("ru = 'Для выполнения операции необходимо установить расширение работы с файлами.'"));

// Составная строка — через СтрШаблон
Текст = СтрШаблон(НСтр("ru = 'Документ %1 не проведён.'"), НомерДокумента);
```

## #std763: Форматирование даты, числа, Булево

**ITS:** https://its.1c.ru/db/v8std/content/763/hdoc | **v8std:** https://v8std.ru/std/763/

### Правила
1. Форматную строку функции `Формат` оборачивайте в `НСтр`, когда задаются `ДФ`, `ДП`, `ЧН`, `ЧФ`, `ЧРД` или вывод `Булево`.
2. Даты выводите локальным форматом `ДЛФ` (а не явным `ДФ=дд.ММ.гггг`), т.к. порядок и разделители различаются по странам.
3. Для `Булево` всегда применяйте `НСтр` к форматной строке.
4. Не переопределяйте локализацию по умолчанию параметром `L=`.

### Пример

```bsl
// Правильно
Формат(ДатаУтверждения, "ДЛФ=ДД");
Формат(ДатаУтверждения, НСтр("ru='ДФ=''ММММ гггг ""г.""'''"));
Предупреждение(Формат(Истина, НСтр("ru='БЛ=Нет; БИ=Да'")));
```

### Проверки
- АПК: 1367, 1368, 1369, 335

## #std764: Строковые константные выражения в коде

**ITS:** https://its.1c.ru/db/v8std/content/764/hdoc | **v8std:** https://v8std.ru/std/764/

### Правила
1. Не локализуйте внутренние строковые идентификаторы (не выводятся пользователю) — не применяйте к ним `НСтр`; именуйте их по правилам идентификаторов.
2. Не используйте в алгоритмах строковые представления объектов/типов (они зависят от языка интерфейса) — используйте `Метаданные()`, типы, предопределённые значения.

### Пример

```bsl
Возврат "ОперацияВыполненаУспешно"; // внутренний идентификатор — без НСтр
Оповестить("ЗаписьФайл", Новый Структура("Событие", "ВерсияСохранена"), ФайлСсылка);
```

## #std765: Элементы форм — требования по локализации

**ITS:** https://its.1c.ru/db/v8std/content/765/hdoc | **v8std:** https://v8std.ru/std/765/

### Правила
1. Не присваивайте реквизитам форм строковые значения без локализации — используйте списки значений с локализуемым представлением или перечисления.
2. Для всех таблиц и групп задавайте заголовки; если заголовок не нужен пользователю — явно отключайте его показ.
3. Сокращайте количество незначащей информации в локализации (удаляйте бессмысленные подсказки у групп).

### Пример

```bsl
// ПЛОХО: строковые значения без локализации
Если Режим = "Рабочий" Тогда
    Режим = "Демо";
КонецЕсли;

// ПРАВИЛЬНО: перечисление или локализуемые значения
Если Режим = Перечисления.РежимРаботы.Рабочий Тогда
    Режим = Перечисления.РежимРаботы.Демо;
КонецЕсли;
```

## #std766: Макеты — требования по локализации

**ITS:** https://its.1c.ru/db/v8std/content/766/hdoc | **v8std:** https://v8std.ru/std/766/

### Правила
1. Табличные, текстовые и HTML-макеты переводятся стандартно; исключение — двоичные макеты и HTML с картинками (текст в картинках переводится отдельно).
2. Для печати на языке, отличном от текущего, используйте параметры макета с языковыми версиями.
3. Именованные параметры подстановки в макетах подчиняются #std761.

## #std767: Регламентные задания — требования по локализации

**ITS:** https://its.1c.ru/db/v8std/content/767/hdoc | **v8std:** https://v8std.ru/std/767/

### Правила
1. Для предопределённых регламентных заданий наименование в конфигураторе не задавайте — достаточно синонима.
2. Для параметризованных (непредопределённых) заданий наименование задавайте программно через `НСтр`.

### Пример

```bsl
Задание = РегламентныеЗадания.СоздатьРегламентноеЗадание(
    Метаданные.РегламентныеЗадания.РассылкаОтчетов);
Задание.Наименование = СтроковыеФункцииКлиентСервер.ПодставитьПараметрыВСтроку(
    НСтр("ru = 'Рассылка отчетов: %1'"), РассылкаОтчетов);
Задание.Записать();
```

## #std769: Поставка международной версии конфигурации

**ITS:** https://its.1c.ru/db/v8std/content/769/hdoc | **v8std:** https://v8std.ru/std/769/

### Правила
1. Национальную (российскую) специфику выделяйте в отдельные объекты метаданных, отсутствующие в международной версии.
2. Реквизиты нац. специфики можно оставлять, но конфигурация должна работать при их удалении.
3. Национальные алгоритмы (загрузка курсов валют и т.п.) выносите в отдельные объекты.

### Пример

```bsl
Процедура ПриОпределенииСимволовНациональногоАлфавита(
    СимволыНациональногоАлфавита,
    ДополнительныеДопустимыеСимволы) Экспорт
    СимволыНациональногоАлфавита = "абвгдеёжзийклмнопрстуфхцчшщъыьэюя";
КонецПроцедуры
```

## #std778: Денежные поля — требования по локализации

**ITS:** https://its.1c.ru/db/v8std/content/778/hdoc | **v8std:** https://v8std.ru/std/778/

### Правила
1. Увеличивайте разрядность целой части денежных полей (`Сумма`, `Цена`, `Себестоимость`) для стран с низким курсом валюты.
2. В запросах `ВЫРАЗИТЬ` денежных полей приводите к `ЧИСЛО(31,2)` (целая часть до 29 — ограничение DB2).
3. В `Формат` и свойствах СКД не задавайте общую длину числа.

### Пример

```bsl
ОписаниеТиповСумма = Новый ОписаниеТипов("Число", Новый КвалификаторыЧисла(15, 2));

// В запросе
ВЫБРАТЬ ВЫРАЗИТЬ(Т.Сумма КАК ЧИСЛО(31, 2)) КАК Сумма
```

## #std784: Автогенерированные данные в ИБ

**ITS:** https://its.1c.ru/db/v8std/content/784/hdoc | **v8std:** https://v8std.ru/std/784/

### Правила
1. Программно записываемые строки (начальное заполнение, предопределённые элементы) формируйте на языке информационной базы, а не на языке текущего пользователя.
2. То же касается обработчиков начального заполнения строковых реквизитов предопределённых элементов.

### Пример

```bsl
// ПРАВИЛЬНО: язык ИБ, а не текущего пользователя
Комментарий = НСтр("ru = 'Комментарий к проводке'");
```
