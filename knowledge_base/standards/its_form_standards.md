## #std740: Структура управляемой формы

**ITS:** https://its.1c.ru/db/v8std/content/740/hdoc

### Суть
Управляемая форма должна иметь чёткую структуру: реквизиты, команды, обработчики.

### Правила
1. Реквизиты формы — только необходимые для UI
2. Серверные процедуры — с префиксом `&НаСервере`
3. Клиентские процедуры — с префиксом `&НаКлиенте`
4. Использовать `&НаСервереБезКонтекста` когда не нужны данные формы
5. Минимум логики в обработчиках — выносить в общие модули

### Структура модуля формы

```bsl
// 1. Переменные формы
Перем ТекущийПользователь Экспорт;  // только если нужно

// 2. Обработчики формы (стандартные)
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    // Инициализация
КонецПроцедуры

&НаКлиенте
Процедура ПриОткрытии(Отказ)
    // UI-подготовка
КонецПроцедуры

// 3. Обработчики элементов формы
&НаКлиенте
Процедура НоменклатураПриИзменении(Элемент)
    ОбновитьЦенуНаСервере();
КонецПроцедуры

&НаСервере
Процедура ОбновитьЦенуНаСервере()
    Объект.Цена = ЦенообразованиеСервер.ПолучитьЦену(Объект.Номенклатура);
КонецПроцедуры

// 4. Обработчики команд формы
&НаКлиенте
Процедура КомандаРассчитать(Команда)
    РассчитатьНаСервере();
КонецПроцедуры

&НаСервере
Процедура РассчитатьНаСервере()
    // Бизнес-логика через общий модуль
    РасчетСервер.Рассчитать(Объект);
КонецПроцедуры
```

## #std741: Элементы формы

**ITS:** https://its.1c.ru/db/v8std/content/741/hdoc

### Правила
- Имена элементов — осмысленные: `КоличествоТовара`, не `Поле1`
- Группировка — через `ГруппаФормы` для логических блоков
- Видимость элементов — через `Элементы.Имя.Видимость`
- Доступность — через `Элементы.Имя.Доступность`
- Цвет/шрифт — только через условное оформление, не в коде

### Паттерн динамического управления элементами

```bsl
// Динамическое создание элемента
&НаСервере
Процедура СоздатьПолеВвода()
    Элемент = Элементы.Добавить("ПолеВвода", Тип("ПолеФормы"));
    Элемент.Вид = ВидПоляФормы.ПолеВвода;
    Элемент.ПутьКДанным = "Объект.Значение";
    Элемент.Заголовок = "Значение";
    Элемент.Видимость = Истина;
КонецПроцедуры
```

## #std742: Командный интерфейс

**ITS:** https://its.1c.ru/db/v8std/content/742/hdoc

### Правила
1. Глобальный командный интерфейс — через подсистемы
2. Локальные команды — в командной панели формы
3. Не дублировать команды в разных местах
4. Подсказки команд — осмысленные и краткие

### Категории команд

| Категория | Где разместить | Когда использовать |
|---|---|---|
| Создание | Подсистема + форма списка | Новые объекты |
| Печать | Подсистема + форма объекта | Печатные формы |
| Отчёты | Подсистема | Аналитика |
| Обработки | Подсистема | Сервисные функции |
| Стандартные | Встроенные | Проведение, отмена |

## #std743: Условное оформление

**ITS:** https://its.1c.ru/db/v8std/content/743/hdoc

### Суть
Условное оформление — для подсветки ячеек/строк по условию, без программного кода.

### Правила
- Цвет, шрифт, видимость — через условное оформление
- Не использовать `Элемент.ЦветФона` в коде
- Сложные условия — через `ЭлементУсловногоОформления.Отбор`
- Динамическое условное оформление — в `ПриСозданииНаСервере`

### Пример

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    // Подсветка просроченных документов красным
    ЭлементОформления = УсловноеОформление.Элементы.Добавить();
    
    // Что оформляем
    ОформляемоеПоле = ЭлементОформления.Поля.Элементы.Добавить();
    ОформляемоеПоле.Поле = Новый ПолеКомпоновкиДанных("СписокДата");
    
    // Условие
    ЭлементОтбора = ЭлементОформления.Отбор.Элементы.Добавить(Тип("ЭлементОтбораКомпоновкиДанных"));
    ЭлементОтбора.ЛевоеЗначение = Новый ПолеКомпоновкиДанных("СписокПросрочен");
    ЭлементОтбора.ВидСравнения = ВидСравненияКомпоновкиДанных.Равно;
    ЭлементОтбора.ПравоеЗначение = Истина;
    
    // Оформление
    ЭлементОформления.Оформление.УстановитьЗначениеПараметра("ЦветФона", WebЦвета.Розовый);
КонецПроцедуры
```

## #std744: Клиент-серверное взаимодействие

**ITS:** https://its.1c.ru/db/v8std/content/744/hdoc  
**Связано с #std580:** Минимизация кода на клиенте

### Правила
- На клиенте — только UI-логика
- Бизнес-логика — на сервере (`&НаСервере`)
- Тяжёлые операции — только на сервере
- Использовать `&НаСервереБезКонтекста` для уменьшения трафика
- Передавать только необходимые данные (не весь `Объект`)

### Пример: минимальный трафик

```bsl
// ПЛОХО: передаём весь объект (тяжёлый)
&НаКлиенте
Процедура ПроверитьДоступность()
    Если Не ПроверитьНаСервере(Объект) Тогда  // весь объект передаётся
        ПоказатьПредупреждение(, "Недоступно");
    КонецЕсли;
КонецПроцедуры

// ПРАВИЛЬНО: передаём только нужное
&НаКлиенте
Процедура ПроверитьДоступность()
    Если Не ПроверитьНаСервере(Объект.Номенклатура) Тогда
        ПоказатьПредупреждение(, "Недоступно");
    КонецЕсли;
КонецПроцедуры

// ОПТИМАЛЬНО: без контекста
&НаСервереБезКонтекста
Функция ПроверитьНаСервере(Номенклатура)
    Возврат НоменклатураСервер.Доступна(Номенклатура);
КонецФункции
```

## #std745: Таблицы на форме

**ITS:** https://its.1c.ru/db/v8std/content/745/hdoc

### Правила
1. Использовать `ДанныеФормыДерево` или `ДанныеФормыКоллекция` для табличных частей
2. Не передавать таблицы значений между клиентом и сервером (только массивы структур)
3. Поиск строки — через `НайтиСтроки()`, не через цикл
4. Удаление строк — через `НайтиСтроки()` + обратный цикл

### Пример

```bsl
// ПЛОХО: цикл для поиска
Для Инд = 0 По Объект.Товары.Количество() - 1 Цикл
    Если Объект.Товары[Инд].Номенклатура = Искомая Тогда
        // ...
        Прервать;
    КонецЕсли;
КонецЦикла;

// ПРАВИЛЬНО: НайтиСтроки
Отбор = Новый Структура("Номенклатура", Искомая);
НайденныеСтроки = Объект.Товары.НайтиСтроки(Отбор);
Если НайденныеСтроки.Количество() > 0 Тогда
    // ...
КонецЕсли;

// Удаление нескольких строк
Отбор = Новый Структура("ПометкаУдаления", Истина);
Для Инд = Объект.Товары.Количество() - 1 По 0 Цикл
    Если Объект.Товары[Инд].ПометкаУдаления Тогда
        Объект.Товары.Удалить(Инд);
    КонецЕсли;
КонецЦикла;
```

## #std746: Формы выбора и подбора

**ITS:** https://its.1c.ru/db/v8std/content/746/hdoc

### Правила
1. Форма выбора — через `ОткрытьФорму("Справочник.Имя.ФормаВыбора", Параметры)`
2. Обработка выбора — через `Оповещение` (немодально)
3. Форма подбора — отдельная форма с возвратом массива
4. Не использовать модальные `Выбрать()` (устаревшее)

### Пример

```bsl
// Немодальный выбор
&НаКлиенте
Процедура ВыбратьНоменклатуру()
    Параметры = Новый Структура;
    Параметры.Вставить("РежимВыбора", Истина);
    Параметры.Вставить("МножественныйВыбор", Истина);
    
    Оповещение = Новый ОписаниеОповещения("ВыбратьНоменклатуруЗавершение", ЭтотОбъект);
    ОткрытьФорму("Справочник.Номенклатура.ФормаВыбора", Параметры, , , , , Оповещение);
КонецПроцедуры

&НаКлиенте
Процедура ВыбратьНоменклатуруЗавершение(ВыбранноеЗначение, ДопПараметры) Экспорт
    Если ВыбранноеЗначение <> Неопределено Тогда
        // обработка выбранной номенклатуры
    КонецЕсли;
КонецПроцедуры
```

## #std747: Оптимизация форм

### 1. Не загружать все данные в форму

```bsl
// ПЛОХО: загрузили все записи
Список.Параметры.Отбор.Очистить();  // все 1000000 записей

// ПРАВИЛЬНО: динамический список с отбором
Список.Параметры.Отбор.Установить("Период", Новый Диапазон(Начало, Конец));
```

### 2. Не делать запросы в ПриСозданииНаСервере без необходимости

```bsl
// ПЛОХО: запрос в ПриСозданииНаСервере для каждого открытия формы
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    Запрос = Новый Запрос("ВЫБРАТЬ ...");
    // медленно для каждого открытия
КонецПроцедуры

// ПРАВИЛЬНО: кэшировать в параметрах формы или регистре сведений
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    Данные = КэшСервер.ПолучитьДанные(Параметры.КлючКэша);
КонецПроцедуры
```

### 3. Не открывать формы в цикле

```bsl
// ПЛОХО: 1000 открытий формы
Для Каждого Стр Из Таблицы Цикл
    ОткрытьФорму("Документ.Имя.Форма.Объект", Новый Структура("Ключ", Стр.Ссылка));
КонецЦикла;

// ПРАВИЛЬНО: пакетная обработка
Для Каждого Стр Из Таблицы Цикл
    ДокументыСервер.Обработать(Стр.Ссылка);
КонецЦикла;
```

## #std748: Антипаттерны форм

### 1. Использование модальных вызовов

```bsl
// ПЛОХО: блокирует UI
Если Вопрос("Удалить?", РежимДиалогаВопрос.ДаНет) = КодВозвратаДиалога.Да Тогда
    // ...
КонецЕсли;

// ПРАВИЛЬНО: немодальный
ПоказатьВопрос(Новый ОписаниеОповещения("УдалитьЗавершение", ЭтотОбъект), 
    "Удалить?", РежимДиалогаВопрос.ДаНет);
```

### 2. Прямой запрос на клиенте

```bsl
// НЕВОЗМОЖНО: запросы работают только на сервере
&НаКлиенте
Процедура Проверить()
    Запрос = Новый Запрос("ВЫБРАТЬ ...");  // ОШИБКА
КонецПроцедуры

// ПРАВИЛЬНО: через серверный вызов
&НаКлиенте
Процедура Проверить()
    Если ПроверитьНаСервере() Тогда
        // ...
    КонецЕсли;
КонецПроцедуры

&НаСервере
Функция ПроверитьНаСервере()
    Запрос = Новый Запрос("ВЫБРАТЬ ...");
    Возврат Не Запрос.Выполнить().Пустой();
КонецФункции
```

### 3. Слишком много данных в реквизитах

```bsl
// ПЛОХО: 100 реквизитов в форме, все передаются на сервер и обратно
// Если нужно только 5 — остальные не создавать

// ПРАВИЛЬНО: 5 ключевых реквизитов + динамический расчёт остальных
```

### 4. Циклы в обработчиках

```bsl
// ПЛОХО: цикл по 1000 строк в ПриИзменении
&НаКлиенте
Процедура НоменклатураПриИзменении(Элемент)
    Для Каждого Стр Из Объект.Товары Цикл
        ПересчитатьСтроку(Стр);  // 1000 серверных вызовов
    КонецЦикла;
КонецПроцедуры

// ПРАВИЛЬНО: один серверный вызов со всей таблицей
&НаКлиенте
Процедура НоменклатураПриИзменении(Элемент)
    ПересчитатьВсеНаСервере();
КонецПроцедуры

&НаСервере
Процедура ПересчитатьВсеНаСервере()
    Для Каждого Стр Из Объект.Товары Цикл
        // пересчёт без клиент-серверных вызовов
    КонецЦикла;
КонецПроцедуры
```
