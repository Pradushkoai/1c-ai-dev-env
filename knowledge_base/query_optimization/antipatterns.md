# /1c-query-antipatterns — Антипаттерны запросов 1С

Чего НЕ надо делать в запросах 1С, и как правильно.

---

## 1. Подзапрос в SELECT (N+1) {#ap-subquery-in-select}

### Антипаттерн
Подзапрос в SELECT для каждой строки основного запроса.

```bsl
// ПЛОХО: Подзапрос для каждой строки
"ВЫБРАТЬ
|       Заказы.Ссылка,
|       (ВЫБРАТЬ К.ИНН ИЗ Справочник.Контрагенты КАК К
|        ГДЕ К.Ссылка = Заказы.Контрагент) КАК ИНН
|ИЗ Документ.ЗаказКлиента КАК Заказы"
```

### Решение: JOIN

```bsl
"ВЫБРАТЬ
|       Заказы.Ссылка, Контрагенты.ИНН
|ИЗ Документ.ЗаказКлиента КАК Заказы
|       ЛЕВОЕ СОЕДИНЕНИЕ Справочник.Контрагенты КАК Контрагенты
|       ПО Заказы.Контрагент = Контрагенты.Ссылка"
```

## 2. JOIN с подзапросом (с группировкой) {#ap-join-with-subquery}

### Антипаттерн
JOIN с подзапросом, содержащим GROUP BY — крайне медленно.

```bsl
// ПЛОХО: JOIN с подзапросом
"... ЛЕВОЕ СОЕДИНЕНИЕ (ВЫБРАТЬ ... СГРУППИРОВАТЬ ПО ...) КАК Итоги ..."
```

### Решение: Временная таблица + JOIN

```bsl
"ВЫБРАТЬ ... ПОМЕСТИТЬ ВТИтоги ... СГРУППИРОВАТЬ ПО ... ИНДЕКСИРОВАТЬ ПО ...;
ВЫБРАТЬ ... ИЗ ... ЛЕВОЕ СОЕДИНЕНИЕ ВТИтоги ..."
```

## 3. Фильтр после виртуальной таблицы {#ap-filter-after-virtual}

### Антипаттерн
Фильтрация в WHERE после виртуальной таблицы (Остатки, Обороты).

```bsl
// ПЛОХО: Фильтр в WHERE — виртуальная таблица вычисляет ВСЕ записи
"ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки(&Дата) КАК Остатки
|ГДЕ Остатки.Номенклатура В (&Список)"
```

### Решение: Фильтр в параметрах виртуальной таблицы

```bsl
"ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки(
|       &Дата, Номенклатура В (&Список)) КАК Остатки"
```

## 4. Разыменование составного типа без ВЫРАЗИТЬ {#ap-composite-type-deref}

### Антипаттерн
Обращение через точку к составному типу без ВЫРАЗИТЬ.

```bsl
// ПЛОХО: Запрос по ВСЕМ типам составного поля
"ВЫБРАТЬ ТоварыНаСкладах.Регистратор.Дата"
```

### Решение: ВЫРАЗИТЬ к конкретному типу

```bsl
"ВЫБРАТЬ ВЫРАЗИТЬ(ТоварыНаСкладах.Регистратор
|       КАК Документ.ПоступлениеТоваровУслуг).Дата"
```

## 5. Обращение через точку для отображения {#ap-dot-for-presentation}

### Антипаттерн
Получение Наименование через точку — дополнительный JOIN.

```bsl
// ПЛОХО: Доп. JOIN через точку
"ВЫБРАТЬ ТоварыНаСкладах.Склад.Наименование"
```

### Решение: ПРЕДСТАВЛЕНИЕ()

```bsl
"ВЫБРАТЬ ПРЕДСТАВЛЕНИЕ(ТоварыНаСкладах.Склад) КАК Склад"
```

## 6. ИЛИ в WHERE — блокировка индексов {#ap-or-in-where}

### Антипаттерн
ИЛИ в WHERE блокирует использование индексов.

```bsl
// ПЛОХО: ИЛИ — full scan
"ГДЕ Товары.Артикул = &Артикул ИЛИ Товары.Код = &Код"
```

### Решение: ОБЪЕДИНИТЬ ВСЕ

```bsl
"ВЫБРАТЬ ... ГДЕ Товары.Артикул = &Артикул
|ОБЪЕДИНИТЬ ВСЕ
|ВЫБРАТЬ ... ГДЕ Товары.Код = &Код"
```

## 7. Запрос в цикле (N+1) {#ap-query-in-loop}

### Антипаттерн
Выполнение запроса в цикле для каждой строки таблицы.

```bsl
// ПЛОХО: N+1 запросов
Для Каждого Стр Из Таблицы Цикл
    Запрос = Новый Запрос("ВЫБРАТЬ ... ГДЕ Код = &Код");
    Запрос.УстановитьПараметр("Код", Стр.Код);
    Результат = Запрос.Выполнить();
КонецЦикла;
```

### Решение: Один запрос с массивом

```bsl
Запрос = Новый Запрос("ВЫБРАТЬ ... ГДЕ Код В (&МассивКодов)");
Запрос.УстановитьПараметр("МассивКодов", Таблица.ВыгрузитьКолонку("Код"));
```

## 8. ВЫБРАТЬ * {#ap-select-star}

### Антипаттерн
Выборка всех полей через `*`.

```bsl
// ПЛОХО: Все поля, включая ненужные
"ВЫБРАТЬ * ИЗ Справочник.Номенклатура"
```

### Решение: Только нужные поля

```bsl
"ВЫБРАТЬ Ссылка, Код, Наименование ИЗ Справочник.Номенклатура"
```

---

## 9. Конкатенация строк в запросе (SQL-инъекция) {#ap-query-concatenation}

### Антипаттерн
Конкатенация строк для построения запроса — SQL-инъекция.

```bsl
// ПЛОХО: SQL-инъекция
ТекстЗапроса = "ВЫБРАТЬ ... ГДЕ Код = """ + Код + """";
```

### Решение: Параметры

```bsl
ТекстЗапроса = "ВЫБРАТЬ ... ГДЕ Код = &Код";
Запрос.УстановитьПараметр("Код", Код);
```

## 10. ПОДОБНО с % в начале {#ap-like-start-percent}

### Антипаттерн
ПОДОБНО с `%` в начале строки — full scan.

```bsl
// ПЛОХО: % в начале — full scan
"ГДЕ Товары.Наименование ПОДОБНО \"%текст\""
```

### Решение: % только в конце

```bsl
"ГДЕ Товары.Наименование ПОДОБНО \"текст%\""
```
