# Паттерн: Создание внешней обработки

## Когда использовать
Когда нужно создать обработку для массовых операций, выгрузок, загрузок, и т.д.

## Структура

```
DataProcessors/
└── ИмяОбработки/
    ├── ИмяОбработки.xml               # Метаданные
    ├── Ext/
    │   └── Module.bsl                  # Модуль объекта
    └── Forms/
        └── Форма/
            ├── Форма.xml
            └── Ext/Form/
                ├── Form.xml
                └── Module.bsl          # Модуль формы
```

## Модуль объекта (Module.bsl)

```bsl
#Область ПрограммныйИнтерфейс

Процедура ВыполнитьОбработку() Экспорт
    // Основная логика обработки
КонецПроцедуры

#КонецОбласти

#Область СлужебныйПрограммныйИнтерфейс

Функция ПолучитьДанныеДляВыгрузки() Экспорт
    // Получение данных
    Возврат Данные;
КонецФункции

#КонецОбласти
```

## Модуль формы (Form/Module.bsl)

```bsl
#Область ОбработчикиСобытийФормы

&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    // Инициализация формы
КонецПроцедуры

#КонецОбласти

#Область ОбработчикиКомандФормы

&НаКлиенте
Процедура ВыполнитьОбработку(Команда)
    ВыполнитьОбработкуНаСервере();
    Сообщить("Обработка выполнена!");
КонецПроцедуры

&НаСервере
Процедура ВыполнитьОбработкуНаСервере()
    РеквизитФормыВЗначение("Объект").ВыполнитьОбработку();
КонецПроцедуры

#КонецОбласти
```

## Best Practices

1. **Разделение логики**: модуль объекта — бизнес-логика, модуль формы — UI
2. **&НаСервере**: тяжёлые операции на сервере
3. **&НаКлиенте**: UI-операции на клиенте
4. **Сообщить()**: информируйте пользователя о прогрессе
5. **Обработка ошибок**: используйте Попытка/Исключение
6. **Транзакции**: используйте НачатьТранзакцию/ЗафиксироватьТранзакцию

## Антипаттерны

- ❌ Бизнес-логика в модуле формы (должна быть в модуле объекта)
- ❌ Тяжёлые операции на клиенте (используйте &НаСервере)
- ❌ Отсутствие обработки ошибок
- ❌ Прямые запросы в форме (выносите в модуль объекта)
