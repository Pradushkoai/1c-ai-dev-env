# Паттерн: Создание справочника

## Когда использовать
Когда нужно создать новый справочник в конфигурации 1С.

## Структура

```
Catalogs/
└── ИмяСправочника/
    ├── ИмяСправочника.xml          # Метаданные
    ├── Ext/
    │   ├── ObjectModule.bsl         # Модуль объекта
    │   └── ManagerModule.bsl        # Модуль менеджера
    └── Forms/
        ├── ФормаЭлемента/
        │   ├── ФормаЭлемента.xml
        │   └── Ext/Form/
        │       ├── Form.xml         # Описание формы
        │       └── Module.bsl       # Модуль формы
        └── ФормаСписка/
            └── ...
```

## Обязательные элементы

### Метаданные (ИмяСправочника.xml)
```xml
<Catalog uuid="...">
    <Properties>
        <Name>ИмяСправочника</Name>
        <Synonym>...</Synonym>
        <Hierarchical>false</Hierarchical>
        <CodeLength>11</CodeLength>
        <DescriptionLength>100</DescriptionLength>
        <DefaultObjectForm>Catalog.Имя.ФормаЭлемента</DefaultObjectForm>
        <DefaultListForm>Catalog.Имя.ФормаСписка</DefaultListForm>
    </Properties>
    <ChildObjects>
        <Attribute uuid="...">
            <Properties>
                <Name>Реквизит1</Name>
                <Type>
                    <v8:Type>xs:string</v8:Type>
                    <v8:StringQualifiers>
                        <v8:Length>50</v8:Length>
                    </v8:StringQualifiers>
                </Type>
            </Properties>
        </Attribute>
    </ChildObjects>
</Catalog>
```

```bsl

    // Подготовка перед записью

    // Действия после записи
```

```bsl

    Возврат НайтиПоРеквизиту("Код", Код);

#КонецОбласти
```

## Best Practices

1. **Именование**: используйте осмысленные имена без сокращений
2. **Синоним**: обязательно указывайте русский синоним
3. **Длина кода**: 11 символов для новых справочников
4. **Длина наименования**: 100 символов (стандарт)
5. **Иерархия**: используйте HierarchyFoldersAndItems для иерархических справочников
6. **Формы**: всегда создавайте ФормаЭлемента и ФормаСписка
7. **Обработчики**: реализуйте ОбработкаПроверкиЗаполнения для валидации

## Антипаттерны

- ❌ Справочник без синонима
- ❌ Код длиной 0 (без кода) без причины
- ❌ Отсутствие модуля объекта при наличии логики
- ❌ Прямой доступ к реквизитам из модуля формы (используйте серверные процедуры)
