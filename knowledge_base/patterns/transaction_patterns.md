# Паттерн: Работа с транзакциями 1С

**Категория:** transactions  
**Применимо к:** Все объекты с записью (справочники, документы, регистры)

---

## Правильная структура транзакции

```bsl
Процедура ЗаписатьДокументСПроверкой(ДокументОбъект) Экспорт

    НачатьТранзакцию();
    Попытка
        // 1. Проверка заполнения
        Если Не ДокументОбъект.ПроверитьЗаполнение() Тогда
            ВызватьИсключение "Не заполнены обязательные поля";
        КонецЕсли;

        // 2. Бизнес-логика (в модуле объекта)
        ДокументОбъект.Записать();

        // 3. Фиксация
        ЗафиксироватьТранзакцию();

    Исключение
        ОтменитьТранзакцию();
        ЗаписьЖурналаРегистрации(
            "Ошибка.ЗаписьДокумента",
            УровеньЖурналаРегистрации.Ошибка,
            ,
            ,
            ОписаниеОшибки()
        );
        ВызватьИсключение;
    КонецПопытки;

КонецПроцедуры
```

## Правила

1. `НачатьТранзакцию()` — ВСЕГДА перед `Попытка`
2. `ЗафиксироватьТранзакцию()` — ВСЕГДА внутри `Попытка`
3. `ОтменитьТранзакцию()` — ВСЕГДА внутри `Исключение`
4. После `ОтменитьТранзакцию` — `ВызватьИсключение` для проброса
5. Минимум кода внутри транзакции — только запись
6. Не делать интерактивные операции (Предупреждение, ОткрытьФорму)
