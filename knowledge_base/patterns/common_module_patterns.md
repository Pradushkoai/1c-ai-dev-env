# Паттерн: Общие модули 1С

**Категория:** common_modules  
**Применимо к:** CommonModule

---

## Именование общих модулей

| Суффикс | Назначение | Настройки | Пример |
|---------|-----------|-----------|--------|
| `Кэшируемый` | Повторное использование (ПовтИсп) | Сервер, ПовтИсп | `ДокументыОборотКэшируемый` |
| `Клиент` | Только клиент | Клиент | `ФормыКлиент` |
| `КлиентСервер` | Клиент и сервер | Клиент, Сервер | `ОбщегоНазначенияКлиентСервер` |
| `ВызовСервера` | Серверный вызов из клиента | Сервер, ВызовСервера | `ДокументыВызовСервера` |
| `Глобальный` | Глобальный контекст | Сервер, Глобальный | `ДокументыГлобальный` |
| `ПолныеПрава` | Привилегированный режим | Сервер, ПолныеПрава | `ДокументыПолныеПрава` |

## Структура общего модуля

```bsl
// CommonModule: ПримерыКэшируемый
// Настройки: Сервер, ПовтИсп

#Область ПрограммныйИнтерфейс

// Экспортные методы — API модуля
// Доступны из других модулей и через ПовтИсп
Функция ПолучитьДанныеПоКоду(Код) Экспорт
    Возврат ПолучитьИзКэша(Код);
КонецФункции

#КонецОбласти

#Область СлужебныйПрограммныйИнтерфейс

// Экспортные методы — только для внутренних модулей конфигурации
Функция ОчиститьКэш() Экспорт
    // ...
КонецФункции

#КонецОбласти

#Область СлужебныеПроцедурыИФункции

// Внутренние методы — не экспорт
// Не доступны из других модулей
Функция ПолучитьИзКэша(Код)
    // ...
КонецФункции

#КонецОбласти
```

## Правила

1. **Не использовать переменные уровня модуля** — stateless (особенно для ПовтИсп)
2. **Области обязательны**: ПрограммныйИнтерфейс → СлужебныйПрограммныйИнтерфейс → СлужебныеПроцедурыИФункции
3. **Минимум экспортных методов** — только то что нужно внешним модулям
4. **Кэшируемые модули** — не изменяют состояние, только возвращают данные
5. **Глобальные модули** — только утилиты, не бизнес-логика
