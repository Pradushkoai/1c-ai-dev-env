# Паттерны работы с регистрами накопления

**Источник:** [ITS #std651](https://its.1c.ru/db/v8std/content/651/hdoc), [ITS #std648](https://its.1c.ru/db/v8std/content/648/hdoc), [ITS #std649](https://its.1c.ru/db/v8std/content/649/hdoc), [ITS #std650](https://its.1c.ru/db/v8std/content/650/hdoc), [ITS #std733](https://its.1c.ru/db/v8std/content/733/hdoc), [v8std.ru](https://v8std.ru)  
**Связанные инструменты:** `generate_query`, `analyze_queries`, `check_transactions`, `solve_check`

> Паттерны работы с регистрами накопления: остатки, обороты, списание, оприходование, перерасчёт. Возникло из практики — в задаче "Прибыль от продажи Лук" постоянно приходилось работать с регистрами накопления.

---

## 1. Виды регистров накопления

| Вид | Назначение | Примеры |
|---|---|---|
| **Остатки** | Учитывает текущее количество/сумму | ТоварыНаСкладах, ВзаиморасчетыСКонтрагентами |
| **Обороты** | Только движения за период | Продажи, Закупки |
| **ОстаткиИОбороты** | Редко — комбинированный | Не использовать без необходимости |

### Структура регистра

- **Измерения** — ключи (Номенклатура, Склад, Контрагент)
- **Ресурсы** — числовые значения (Количество, Сумма)
- **Реквизиты** — доп. атрибуты (Проект, Подразделение)

## 2. Паттерн R1: Списание с остатка (ФИФО)

**Когда:** списать количество с остатков по принципу "первый пришёл — первый ушёл".

```bsl
// Получить остатки на дату списания, упорядоченные по регистратору (ФИФО)
ВЫБРАТЬ
    Ост.Номенклатура КАК Номенклатура,
    Ост.Склад КАК Склад,
    Ост.КоличествоОстаток КАК Доступно,
    Ост.СтоимостьОстаток КАК СтоимостьДоступно
ПОМЕСТИТЬ ВТ_Остатки
ИЗ
    РегистрНакопления.ТоварыНаСкладах.Остатки(
        &МоментСписания,
        Номенклатура = &Номенклатура
            И Склад = &Склад
    ) КАК Ост
УПОРЯДОЧИТЬ ПО
    Ост.Регистратор  // ФИФО — по документу прихода
ИНДЕКСИРОВАТЬ ПО
    Номенклатура
;

// Списание по партиям
// Внешний цикл: по строкам документа списания
// Внутренний: по строкам ВТ_Остатки пока не спишется всё
```

**В коде 1С:**

```bsl
Процедура СписатьПоФИФО(Номенклатура, Склад, Количество)
    Запрос = Новый Запрос(
        "ВЫБРАТЬ
        |    Ост.Регистратор.Дата КАК ДатаПрихода,
        |    Ост.Регистратор КАК ДокументПрихода,
        |    Ост.КоличествоОстаток КАК Доступно,
        |    Ост.СтоимостьОстаток КАК СтоимостьДоступно
        |ИЗ
        |    РегистрНакопления.ТоварыНаСкладах.Остатки(
        |        &МоментСписания,
        |        Номенклатура = &Номенклатура
        |            И Склад = &Склад
        |    ) КАК Ост
        |УПОРЯДОЧИТЬ ПО
        |    ДатаПрихода");
    Запрос.УстановитьПараметр("МоментСписания", ТекущаяДатаСеанса());
    Запрос.УстановитьПараметр("Номенклатура", Номенклатура);
    Запрос.УстановитьПараметр("Склад", Склад);
    
    ОсталосьСписать = Количество;
    Выборка = Запрос.Выполнить().Выбрать();
    Пока Выборка.Следующий() И ОсталосьСписать > 0 Цикл
        СписатьСПартии = Мин(ОсталосьСписать, Выборка.Доступно);
        // Создать движение в регистр
        Движение = Движения.ТоварыНаСкладах.ДобавитьРасход();
        Движение.Период = Дата;
        Движение.Регистратор = Ссылка;
        Движение.Номенклатура = Номенклатура;
        Движение.Склад = Склад;
        Движение.Количество = СписатьСПартии;
        Движение.ДокументПрихода = Выборка.ДокументПрихода;  // для партионного учёта
        
        ОсталосьСписать = ОсталосьСписать - СписатьСПартии;
    КонецЦикла;
    
    Если ОсталосьСписать > 0 Тогда
        ВызватьИсключение "Недостаточно товара на складе";
    КонецЕсли;
КонецПроцедуры
```

## 3. Паттерн R2: Списание по средней

**Когда:** конфигурация использует среднюю себестоимость.

```bsl
// Средняя себестоимость = (Сумма остатка + Сумма прихода) / (Количество остатка + Количество прихода)
ВЫБРАТЬ
    СУММА(Ост.КоличествоОстаток) КАК КоличествоОстаток,
    СУММА(Ост.СтоимостьОстаток) КАК СтоимостьОстатка,
    ВЫБОР
        КОГДА СУММА(Ост.КоличествоОстаток) > 0
            ТОГДА СУММА(Ост.СтоимостьОстаток) / СУММА(Ост.КоличествоОстаток)
        ИНАЧЕ 0
    КОНЕЦ КАК СредняяСебестоимость
ИЗ
    РегистрНакопления.ТоварыНаСкладах.Остатки(
        &Момент,
        Номенклатура = &Номенклатура И Склад = &Склад
    ) КАК Ост
```

## 4. Паттерн R3: Перерасчёт регистра

**Когда:** нужно пересчитать движения за период (например, после изменения документа).

```bsl
// 1. Очистить старые движения документа
Движения.ТоварыНаСкладах.Записывать = Истина;
Движения.ТоварыНаСкладах.Очистить();

// 2. Сформировать новые движения
Для Каждого Стр Из Товары Цикл
    Движение = Движения.ТоварыНаСкладах.ДобавитьПриход();
    Движение.Период = Дата;
    Движение.Номенклатура = Стр.Номенклатура;
    Движение.Склад = Склад;
    Движение.Количество = Стр.Количество;
    Движение.Сумма = Стр.Сумма;
КонецЦикла;

// 3. Записать движения
Движения.ТоварыНаСкладах.Записать();
```

## 5. Паттерн R4: Движения по нескольким регистрам

**Когда:** документ делает движения в несколько регистров одновременно.

```bsl
Процедура ОбработкаПроведения(Отказ, Режим)
    // Очистить все движения
    Движения.ТоварыНаСкладах.Записывать = Истина;
    Движения.Продажи.Записывать = Истина;
    Движения.Взаиморасчеты.Записывать = Истина;
    
    Движения.ТоварыНаСкладах.Очистить();
    Движения.Продажи.Очистить();
    Движения.Взаиморасчеты.Очистить();
    
    Для Каждого Стр Из Товары Цикл
        // Списание товаров
        ДвТовары = Движения.ТоварыНаСкладах.ДобавитьРасход();
        ДвТовары.Период = Дата;
        ДвТовары.Номенклатура = Стр.Номенклатура;
        ДвТовары.Склад = Склад;
        ДвТовары.Количество = Стр.Количество;
        ДвТовары.Сумма = Стр.Сумма;
        
        // Продажа
        ДвПродажи = Движения.Продажи.Добавить();
        ДвПродажи.Период = Дата;
        ДвПродажи.Номенклатура = Стр.Номенклатура;
        ДвПродажи.Контрагент = Контрагент;
        ДвПродажи.Количество = Стр.Количество;
        ДвПродажи.Сумма = Стр.Сумма;
        
        // Взаиморасчёты
        ДвРасчеты = Движения.Взаиморасчеты.ДобавитьРасход();
        ДвРасчеты.Период = Дата;
        ДвРасчеты.Контрагент = Контрагент;
        ДвРасчеты.Договор = Договор;
        ДвРасчеты.Сумма = Стр.Сумма;
    КонецЦикла;
    
    // Записать все
    Движения.Записать();
КонецПроцедуры
```

## 6. Паттерн R5: Проверка остатков перед списанием

```bsl
Процедура ОбработкаПроведения(Отказ, Режим)
    // Проверить остатки
    Запрос = Новый Запрос(
        "ВЫБРАТЬ
        |    Т.Номенклатура КАК Номенклатура,
        |    Т.Склад КАК Склад,
        |    Т.Количество КАК Нужно,
        |    ЕСТЬNULL(Ост.КоличествоОстаток, 0) КАК Доступно
        |ПОМЕСТИТЬ ВТ_Товары
        |ИЗ
        |    Документ.Реализация.Товары КАК Т
        |ГДЕ
        |    Т.Ссылка = &Ссылка
        |
        |ИНДЕКСИРОВАТЬ ПО Номенклатура, Склад
        |;
        |
        |ВЫБРАТЬ
        |    Т.Номенклатура КАК Номенклатура,
        |    Т.Склад КАК Склад,
        |    Т.Нужно КАК Нужно,
        |    ЕСТЬNULL(Ост.Доступно, 0) КАК Доступно,
        |    Т.Нужно - ЕСТЬNULL(Ост.Доступно, 0) КАК Недостаток
        |ИЗ
        |    ВТ_Товары КАК Т
        |    ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.ТоварыНаСкладах.Остатки(
        |        &Момент,
        |        Номенклатура В (ВЫБРАТЬ Номенклатура ИЗ ВТ_Товары)
        |    ) КАК Ост
        |        ПО Т.Номенклатура = Ост.Номенклатура
        |            И Т.Склад = Ост.Склад
        |ГДЕ
        |    Т.Нужно > ЕСТЬNULL(Ост.Доступно, 0)");
    
    Запрос.УстановитьПараметр("Ссылка", Ссылка);
    Запрос.УстановитьПараметр("Момент", МоментВремени());
    
    Результат = Запрос.Выполнить();
    Если Не Результат.Пустой() Тогда
        Отказ = Истина;
        Выборка = Результат.Выбрать();
        Пока Выборка.Следующий() Цикл
            Сообщить("Недостаточно: " + Строка(Выборка.Номенклатура)
                + " нужно " + Выборка.Нужно
                + " доступно " + Выборка.Доступно);
        КонецЦикла;
    КонецЕсли;
    
    Если Отказ Тогда
        Возврат;
    КонецЕсли;
    
    // ... формирование движений
КонецПроцедуры
```

## 7. Паттерн R6: Свёртка движений

**Когда:** нужно удалить дубли/компенсирующие движения.

```bsl
// Получить движения документа, сгруппировать по измерениям
ВЫБРАТЬ

    Дв.Номенклатура, Дв.Склад
```

## 8. Паттерн R7: Восстановление последовательности

**Когда:** изменён документ в середине периода — нужно перепровести более поздние документы.

```bsl
Процедура ВосстановитьПоследовательность()
    // Найти первое нарушение границы последовательности

        |    Последовательность.ПартионныйУчет КАК Посл");

        Выборка = Результат.Выбрать();
        Выборка.Следующий();
        
        // Провести документы после границы

        КонецЦикла;

КонецПроцедуры
```

## 9. #std648: Чтение данных в транзакции

**ITS:** https://its.1c.ru/db/v8std/content/648/hdoc

### Правила
- Чтение данных в транзакции — через запросы с `ВЫБРАТЬ РАЗРЕШЕННЫЕ`
- Использовать `БлокировкаДанных` для явных блокировок
- Не использовать `Получить()` / `Найти()` в транзакции (неявная блокировка)

### Пример

```bsl
Процедура ОбработкаПроведения(Отказ, Режим)
    // Явная блокировка
    Блокировка = Новый БлокировкаДанных;
    Элемент = Блокировка.Добавить("РегистрНакопления.ТоварыНаСкладах");
    Элемент.УстановитьЗначение("Склад", Склад);
    Элемент.Режим = РежимБлокировкиДанных.Исключительный;
    Блокировка.Заблокировать();
    
    // Чтение с учётом блокировок (РАЗРЕШЕННЫЕ)
    Запрос = Новый Запрос(
        "ВЫБРАТЬ РАЗРЕШЕННЫЕ
        |    Ост.Номенклатура, Ост.КоличествоОстаток
        |ИЗ
        |    РегистрНакопления.ТоварыНаСкладах.Остатки(
        |        &Момент,
        |        Склад = &Склад
        |    ) КАК Ост");
    // ...
КонецПроцедуры
```

## 10. #std649: Запись в транзакции

**ITS:** https://its.1c.ru/db/v8std/content/649/hdoc

### Правила
- Транзакция — `НачатьТранзакцию` → `Попытка` → `ЗафиксироватьТранзакцию` → `Исключение` → `ОтменитьТранзакцию`
- Вне транзакции — только чтение, не запись
- Длительные транзакции блокируют других пользователей

### Пример

```bsl
Процедура ПерепровестиВДокументов(МассивСсылок)
    НачатьТранзакцию();
    Попытка
        Для Каждого Ссылка Из МассивСсылок Цикл
            ДокументОбъект = Ссылка.ПолучитьОбъект();
            ДокументОбъект.Записать(РежимЗаписиДокумента.Проведение);
        КонецЦикла;
        ЗафиксироватьТранзакцию();
    Исключение
        ОтменитьТранзакцию();
        ЗаписьЖурналаРегистрации("Перепроведение", УровеньЖурналаРегистрации.Ошибка,
            , , ОписаниеОшибки());
        ВызватьИсключение;
    КонецПопытки;
КонецПроцедуры
```

## 11. #std650: Чтение остатков в транзакции

**ITS:** https://its.1c.ru/db/v8std/content/650/hdoc

### Правила
- Остатки читать в той же транзакции, где будет списание
- Использовать `РАЗРЕШЕННЫЕ` чтобы не ждать чужие блокировки
- Не читать остатки вне транзакции перед списанием — они устареют

### Пример

```bsl
Процедура СписатьТовары(Отказ)
    НачатьТранзакцию();
    Попытка
        // Чтение остатков ВНУТРИ транзакции
        Запрос = Новый Запрос(
            "ВЫБРАТЬ РАЗРЕШЕННЫЕ
            |    Ост.Номенклатура, Ост.КоличествоОстаток
            |ИЗ
            |    РегистрНакопления.ТоварыНаСкладах.Остатки(
            |        &Момент,
            |        Склад = &Склад
            |    ) КАК Ост");
        
        // Формирование движений
        // ...
        
        // Запись
        Движения.Записать();
        ЗафиксироватьТранзакцию();
    Исключение
        ОтменитьТранзакцию();
        Отказ = Истина;
        ВызватьИсключение;
    КонецПопытки;
КонецПроцедуры
```

## 12. #std651: Блокировки при работе с остатками

**ITS:** https://its.1c.ru/db/v8std/content/651/hdoc

### Правила
1. Блокировать по минимальному набору измерений (Номенклатура + Склад, не вся таблица)
2. Использовать `БлокировкаДанных` ПЕРЕД чтением остатков
3. Не блокировать регистр целиком — это убивает параллелизм
4. Управляемые блокировки лучше автоматических

### Пример явной блокировки

```bsl
Процедура Списать(Отказ)
    // Заблокировать нужные записи перед чтением
    Блокировка = Новый БлокировкаДанных;
    Для Каждого Стр Из Товары Цикл
        Элемент = Блокировка.Добавить("РегистрНакопления.ТоварыНаСкладах");
        Элемент.УстановитьЗначение("Склад", Склад);
        Элемент.УстановитьЗначение("Номенклатура", Стр.Номенклатура);
        Элемент.Режим = РежимБлокировкиДанных.Исключительный;
    КонецЦикла;
    Блокировка.Заблокировать();
    
    // Теперь безопасно читать и списывать
    // ...
КонецПроцедуры
```

## 13. Типичные ошибки

### 13.1. Чтение вне транзакции

```bsl
// ПЛОХО: между чтением и записью остаток изменится
Остаток = ПолучитьОстаток();
Если Остаток >= Количество Тогда
    // ... между этими строками кто-то списал товар
    Списать(Количество);  // ОШИБКА: списываем больше чем есть
КонецЕсли;

// ПРАВИЛЬНО: всё в одной транзакции с блокировкой
НачатьТранзакцию();
Попытка
    Блокировка.Заблокировать();
    Остаток = ПолучитьОстаток();
    Если Остаток >= Количество Тогда
        Списать(Количество);
    Иначе
        Отказ = Истина;
    КонецЕсли;
    ЗафиксироватьТранзакцию();
Исключение
    ОтменитьТранзакцию();
КонецПопытки;
```

### 13.2. Запрос без `РАЗРЕШЕННЫЕ`

```bsl
// ПЛОХО: запрос будет ждать чужие блокировки
ВЫБРАТЬ Ост.Номенклатура, Ост.КоличествоОстаток
ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки(...) КАК Ост

// ПРАВИЛЬНО
ВЫБРАТЬ РАЗРЕШЕННЫЕ Ост.Номенклатура, Ост.КоличествоОстаток
ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки(...) КАК Ост
```

### 13.3. Грубая блокировка всей таблицы

```bsl
// ПЛОХО: блокирует всё
Элемент = Блокировка.Добавить("РегистрНакопления.ТоварыНаСкладах");
// без УстановитьЗначение — блокирует ВЕСЬ регистр

// ПРАВИЛЬНО: по конкретным измерениям
Элемент = Блокировка.Добавить("РегистрНакопления.ТоварыНаСкладах");
Элемент.УстановитьЗначение("Склад", Склад);
Элемент.УстановитьЗначение("Номенклатура", Номенклатура);
```

### 13.4. Не очищать движения перед перепроведением

```bsl
// ПЛОХО: движения задвоятся при перепроведении
Движения.ТоварыНаСкладах.ДобавитьПриход();
// ...
Движения.Записать();

// ПРАВИЛЬНО: сначала очистить
Движения.ТоварыНаСкладах.Записывать = Истина;
Движения.ТоварыНаСкладах.Очистить();
// затем добавить
Движения.ТоварыНаСкладах.ДобавитьПриход();
Движения.Записать();
```

### 13.5. Не использовать МоментВремени()

```bsl
// ПЛОХО: устаревший момент на момент времени документа
Остатки = Запрос.Остатки(Дата, ...);
// если другой документ проведён позже, но с более ранней датой — конфликт

// ПРАВИЛЬНО: использовать МоментВремени()
Остатки = Запрос.Остатки(МоментВремени(), ...);
```
