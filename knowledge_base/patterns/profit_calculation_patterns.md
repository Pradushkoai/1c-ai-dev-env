# Паттерны расчёта прибыли и себестоимости в 1С

**Источник:** Практика решения задачи "Прибыль от продажи Лук" + [ITS #std729](https://its.1c.ru/db/v8std/content/729/hdoc) (оптимальные запросы), [ITS #std655](https://its.1c.ru/db/v8std/content/655/hdoc) (JOIN ограничения)  
**Связанные инструменты:** `generate_query`, `analyze_queries`, `dependency_graph.find_join_path`, `solve_check`

> Практические паттерны расчёта финансовых показателей: прибыли, себестоимости, рентабельности. Сфокусировано на задачах, где требуется связать продажи, закупки и справочники. Возникло из реальной задачи "Прибыль от продажи Лук", в которой выявлены пробелы в KB.

---

## 1. Базовая модель данных

Типовая схема для расчёта прибыли в торговле:

| Объект | Тип | Назначение |
|---|---|---|
| `РегистрНакопления.Продажи` (Обороты) | Регистр | Выручка от продаж (Приход/Расход) |
| `РегистрНакопления.Закупки` (Обороты) | Регистр | Закупки товаров |
| `РегистрНакопления.ТоварыНаСкладах` (Остатки) | Регистр | Текущие остатки |
| `РегистрСведений.ЦеныНоменклатуры` | Регистр | Актуальные цены |
| `РегистрСведений.Себестоимость` | Регистр | Себестоимость по партиям / ФИФО |
| `Справочник.Номенклатура` | Справочник | Товары |
| `Справочник.КлючиАналитикиУчетаНоменклатуры` | Справочник | Аналитика учета (НДС, валюты) |

## 2. Паттерн P1: Простая прибыль (выручка - закупки)

**Когда:** не нужна точная себестоимость по партиям, достаточно "в среднем по закупкам".

```bsl
// Прибыль = Выручка - Закупочная стоимость
ВЫБРАТЬ
    Прод.Номенклатура КАК Номенклатура,
    СУММА(Прод.Сумма) КАК Выручка,
    СУММА(Зак.Сумма) КАК ЗакупочнаяСтоимость,
    СУММА(Прод.Сумма) - СУММА(Зак.Сумма) КАК Прибыль,
    ВЫБОР
        КОГДА СУММА(Зак.Сумма) > 0
            ТОГДА (СУММА(Прод.Сумма) - СУММА(Зак.Сумма)) / СУММА(Зак.Сумма) * 100
        ИНАЧЕ 0
    КОНЕЦ КАК РентабельностьПроц
ИЗ
    РегистрНакопления.Продажи.Обороты(&Начало, &Конец, Период, ) КАК Прод
    ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.Закупки.Обороты(&Начало, &Конец, Период, ) КАК Зак
        ПО Прод.Номенклатура = Зак.Номенклатура
СГРУППИРОВАТЬ ПО
    Прод.Номенклатура
```

**Проблема этого подхода:** нет учёта остатков на начало периода. Если закуплено 100 шт, продано 50 шт — "прибыль" по этому запросу занижена в 2 раза.

## 3. Паттерн P2: Прибыль по проданным партиям (ФИФО/партионный учёт)

**Когда:** конфигурация использует партионный учёт (УПП, ERP, УТ 11).

```bsl
// Получить себестоимость проданных партий из специального регистра
ВЫБРАТЬ
    Прод.Номенклатура КАК Номенклатура,
    СУММА(Прод.Сумма) КАК Выручка,
    ЕСТЬNULL(Себ.Сумма, 0) КАК Себестоимость,
    СУММА(Прод.Сумма) - ЕСТЬNULL(Себ.Сумма, 0) КАК Прибыль,
    ВЫБОР
        КОГДА ЕСТЬNULL(Себ.Сумма, 0) > 0
            ТОГДА (СУММА(Прод.Сумма) - ЕСТЬNULL(Себ.Сумма, 0)) / ЕСТЬNULL(Себ.Сумма, 0) * 100
        ИНАЧЕ 0
    КОНЕЦ КАК РентабельностьПроц
ИЗ
    РегистрНакопления.Продажи.Обороты(&Начало, &Конец, Период, ) КАК Прод
    ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.СебестоимостьПродаж.Обороты(&Начало, &Конец, Период, ) КАК Себ
        ПО Прод.Номенклатура = Себ.Номенклатура
            И Прод.ДокументПродажи = Себ.ДокументПартии
СГРУППИРОВАТЬ ПО
    Прод.Номенклатура,
    Себ.Сумма
```

**Связь с `find_join_path`:** путь `Продажи → СебестоимостьПродаж` автоматически находится через `dependency_graph.find_join_path("РегистрНакопления.Продажи", "РегистрНакопления.СебестоимостьПродаж")`.

## 4. Паттерн P3: Средняя себестоимость (УТ 10, УПП)

**Когда:** используется средняя себестоимость, без партий.

```bsl
// 1. Остатки на начало (количество + сумма)
ВЫБРАТЬ
    Ост.Номенклатура КАК Номенклатура,
    Ост.КоличествоОстаток КАК Количество,
    Ост.СтоимостьОстаток КАК Сумма
ПОМЕСТИТЬ ВТ_ОстаткиНачало
ИЗ
    РегистрНакопления.ТоварыНаСкладах.Остатки(
        &НачалоПериода,
        Склад = &Склад
    ) КАК Ост
ИНДЕКСИРОВАТЬ ПО
    Номенклатура
;

// 2. Закупки за период
ВЫБРАТЬ
    Зак.Номенклатура КАК Номенклатура,
    СУММА(Зак.Количество) КАК Количество,
    СУММА(Зак.Сумма) КАК Сумма
ПОМЕСТИТЬ ВТ_Закупки
ИЗ
    РегистрНакопления.Закупки.Обороты(
        &НачалоПериода, &КонецПериода, Период,
        Склад = &Склад
    ) КАК Зак
СГРУППИРОВАТЬ ПО
    Зак.Номенклатура
ИНДЕКСИРОВАТЬ ПО
    Номенклатура
;

// 3. Средняя себестоимость = (Остаток + Закупки) / (Количество остатка + Количество закупки)
ВЫБРАТЬ
    Ном.Ссылка КАК Номенклатура,
    ЕСТЬNULL(Нач.Количество, 0) + ЕСТЬNULL(Зак.Количество, 0) КАК ВсегоКоличество,
    ЕСТЬNULL(Нач.Сумма, 0) + ЕСТЬNULL(Зак.Сумма, 0) КАК ВсегоСумма,
    ВЫБОР
        КОГДА ЕСТЬNULL(Нач.Количество, 0) + ЕСТЬNULL(Зак.Количество, 0) > 0
            ТОГДА (ЕСТЬNULL(Нач.Сумма, 0) + ЕСТЬNULL(Зак.Сумма, 0))
                  / (ЕСТЬNULL(Нач.Количество, 0) + ЕСТЬNULL(Зак.Количество, 0))
        ИНАЧЕ 0
    КОНЕЦ КАК СредняяСебестоимость
ПОМЕСТИТЬ ВТ_СредняяСебестоимость
ИЗ
    Справочник.Номенклатура КАК Ном
    ЛЕВОЕ СОЕДИНЕНИЕ ВТ_ОстаткиНачало КАК Нач ПО Ном.Ссылка = Нач.Номенклатура
    ЛЕВОЕ СОЕДИНЕНИЕ ВТ_Закупки КАК Зак ПО Ном.Ссылка = Зак.Номенклатура
ИНДЕКСИРОВАТЬ ПО
    Номенклатура
;

// 4. Продажи * средняя себестоимость
ВЫБРАТЬ
    Прод.Номенклатура КАК Номенклатура,
    СУММА(Прод.Количество) КАК КоличествоПродаж,
    СУММА(Прод.Сумма) КАК Выручка,
    СУММА(Прод.Количество) * ЕСТЬNULL(Себ.СредняяСебестоимость, 0) КАК СебестоимостьПродаж,
    СУММА(Прод.Сумма) - СУММА(Прод.Количество) * ЕСТЬNULL(Себ.СредняяСебестоимость, 0) КАК Прибыль
ИЗ
    РегистрНакопления.Продажи.Обороты(
        &НачалоПериода, &КонецПериода, Период,
        Склад = &Склад
    ) КАК Прод
    ЛЕВОЕ СОЕДИНЕНИЕ ВТ_СредняяСебестоимость КАК Себ
        ПО Прод.Номенклатура = Себ.Номенклатура
СГРУППИРОВАТЬ ПО
    Прод.Номенклатура,
    Себ.СредняяСебестоимость
```

## 5. Паттерн P4: Прибыль по конкретной продаже (аналитика сделки)

**Когда:** нужно посчитать прибыль от конкретной продажи, как в задаче "Прибыль от продажи Лук".

```bsl
// Найти документ продажи и посчитать прибыль от него
ВЫБРАТЬ
    Прод.Регистратор КАК ДокументПродажи,
    Прод.Номенклатура КАК Номенклатура,
    Прод.Количество КАК Количество,
    Прод.Сумма КАК Выручка,
    ЕСТЬNULL(Себ.Сумма, 0) КАК Себестоимость,
    Прод.Сумма - ЕСТЬNULL(Себ.Сумма, 0) КАК Прибыль,
    ВЫБОР
        КОГДА ЕСТЬNULL(Себ.Сумма, 0) > 0
            ТОГДА (Прод.Сумма - ЕСТЬNULL(Себ.Сумма, 0)) / Себ.Сумма * 100
        ИНАЧЕ 0
    КОНЕЦ КАК РентабельностьПроц
ИЗ
    РегистрНакопления.Продажи КАК Прод
    ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.СебестоимостьПродаж КАК Себ
        ПО Прод.Регистратор = Себ.ДокументПартии
            И Прод.Номенклатура = Себ.Номенклатура
ГДЕ
    Прод.Номенклатура = &НоменклатураЛук  // конкретный товар
    И Прод.Период МЕЖДУ &Начало И &Конец
УПОРЯДОЧИТЬ ПО
    Прод.Период
```

## 6. Паттерн P5: Прибыль с дополнительной аналитикой (НДС, валюты)

**Когда:** нужно добавить НДС, валюту или другие аналитические разрезы. В конфигурациях 1С есть специальный регистр `КлючиАналитикиУчетаНоменклатуры`, который связывает номенклатуру с НДС, ставками и т.д.

```bsl
// Связать продажи с ключами аналитики учёта (ставка НДС, валюта)
ВЫБРАТЬ
    Прод.Номенклатура КАК Номенклатура,
    Ключ.СтавкаНДС КАК СтавкаНДС,
    Ключ.ВалютаВзаиморасчетов КАК Валюта,
    СУММА(Прод.Сумма) КАК Выручка,
    СУММА(Прод.Сумма) / (1 + Ключ.СтавкаНДС.Ставка / 100) КАК ВыручкаБезНДС,
    СУММА(Прод.Сумма) - СУММА(Прод.Сумма) / (1 + Ключ.СтавкаНДС.Ставка / 100) КАК СуммаНДС,
    ЕСТЬNULL(СУММА(Себ.Сумма), 0) КАК Себестоимость,
    СУММА(Прод.Сумма) / (1 + Ключ.СтавкаНДС.Ставка / 100) - ЕСТЬNULL(СУММА(Себ.Сумма), 0) КАК Прибыль
ИЗ
    РегистрНакопления.Продажи.Обороты(&Начало, &Конец, Период, ) КАК Прод
    ВНУТРЕННЕЕ СОЕДИНЕНИЕ РегистрСведений.КлючиАналитикиУчетаНоменклатуры.СрезПоследних(
        &Конец, Номенклатура = Прод.Номенклатура
    ) КАК Ключ
        ПО Прод.Номенклатура = Ключ.Номенклатура
    ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.СебестоимостьПродаж.Обороты(
        &Начало, &Конец, Период,
    ) КАК Себ
        ПО Прод.Номенклатура = Себ.Номенклатура
СГРУППИРОВАТЬ ПО
    Прод.Номенклатура,
    Ключ.СтавкаНДС,
    Ключ.ВалютаВзаиморасчетов
```

**Авто-поиск пути через `find_join_path`:**

```python
path = graph.find_join_path(
    "РегистрНакопления.Продажи",
    "Справочник.Номенклатура"
)
# Возвращает: [
#   {"from": "РегистрНакопления.Продажи", "to": "РегистрСведений.КлючиАналитикиУчетаНоменклатуры",
#    "from_field": "Номенклатура", "to_field": "Номенклатура"},
#   {"from": "РегистрСведений.КлючиАналитикиУчетаНоменклатуры", "to": "Справочник.Номенклатура",
#    "from_field": "Номенклатура", "to_field": "Ссылка"}
# ]
```

## 7. Паттерн P6: Прибыль с группировкой по периоду (динамика)

```bsl
// Прибыль по месяцам
ВЫБРАТЬ
    НАЧАЛОПЕРИОДА(Прод.Период, МЕСЯЦ) КАК Период,
    Прод.Номенклатура КАК Номенклатура,
    СУММА(Прод.Количество) КАК Количество,
    СУММА(Прод.Сумма) КАК Выручка,
    ЕСТЬNULL(СУММА(Себ.Сумма), 0) КАК Себестоимость,
    СУММА(Прод.Сумма) - ЕСТЬNULL(СУММА(Себ.Сумма), 0) КАК Прибыль,
    ВЫБОР
        КОГДА ЕСТЬNULL(СУММА(Себ.Сумма), 0) > 0
            ТОГДА (СУММА(Прод.Сумма) - ЕСТЬNULL(СУММА(Себ.Сумма), 0)) / СУММА(Себ.Сумма) * 100
        ИНАЧЕ 0
    КОНЕЦ КАК РентабельностьПроц
ИЗ
    РегистрНакопления.Продажи КАК Прод
    ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.СебестоимостьПродаж КАК Себ
        ПО Прод.Регистратор = Себ.ДокументПартии
            И Прод.Номенклатура = Себ.Номенклатура
ГДЕ
    Прод.Период МЕЖДУ &НачалоГода И &КонецГода
СГРУППИРОВАТЬ ПО
    НАЧАЛОПЕРИОДА(Прод.Период, МЕСЯЦ),
    Прод.Номенклатура
УПОРЯДОЧИТЬ ПО
    Период, Номенклатура
```

**Ловушка #std658:** функция `НАЧАЛОПЕРИОДА` в WHERE блокирует индекс. Использовать диапазон `МЕЖДУ &НачалоГода И &КонецГода` в WHERE, а `НАЧАЛОПЕРИОДА` только в SELECT.

## 8. Паттерн P7: Прибыль с конвертацией валюты

```bsl
// Если продажи в разных валютах — конвертировать в одну (рубли)
ВЫБРАТЬ
    Прод.Номенклатура КАК Номенклатура,
    СУММА(Прод.Сумма * Курс.Курс / Курс.Кратность) КАК ВыручкаРуб,
    ЕСТЬNULL(СУММА(Себ.Сумма), 0) КАК СебестоимостьРуб,
    СУММА(Прод.Сумма * Курс.Курс / Курс.Кратность) - ЕСТЬNULL(СУММА(Себ.Сумма), 0) КАК ПрибыльРуб
ИЗ
    РегистрНакопления.Продажи КАК Прод
    ЛЕВОЕ СОЕДИНЕНИЕ РегистрСведений.КурсыВалют.СрезПоследних(
        Прод.Период,
        Валюта = Прод.Валюта
    ) КАК Курс
        ПО Прод.Валюта = Курс.Валюта
    ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.СебестоимостьПродаж КАК Себ
        ПО Прод.Регистратор = Себ.ДокументПартии
СГРУППИРОВАТЬ ПО
    Прод.Номенклатура
```

## 9. Типичные ошибки и их исправление

### 9.1. Не учтён возврат товаров

```bsl
// ПЛОХО: не учитывается, что покупатель мог вернуть товар
СУММА(Прод.Сумма) КАК Выручка  //gross

// ПРАВИЛЬНО: чистая выручка = продажи - возвраты
СУММА(ВЫБОР КОГДА Прод.ВидДвижения = ЗНАЧЕНИЕ(ВидДвиженияНакопления.Приход)
    ТОГДА -Прод.Сумма ИНАЧЕ Прод.Сумма КОНЕЦ) КАК ВыручкаЧистая
```

### 9.2. Скрытая потеря точности при делении

```bsl
// ПЛОХО: разрядность не определена (#std535)
СУММА(Прод.Сумма) / СУММА(Зак.Сумма) * 100

// ПРАВИЛЬНО: зафиксировать разрядность
ВЫРАЗИТЬ(СУММА(Прод.Сумма) / СУММА(Зак.Сумма) * 100 КАК ЧИСЛО(15, 2))
```

### 9.3. Не обработан NULL в делении

```bsl
// ПЛОХО: деление на NULL даёт NULL (теряем строку в отчёте)
СУММА(Прод.Сумма) / СУММА(Зак.Сумма) * 100

// ПРАВИЛЬНО: ВЫБОР с проверкой
ВЫБОР
    КОГДА ЕСТЬNULL(СУММА(Зак.Сумма), 0) > 0
        ТОГДА СУММА(Прод.Сумма) / СУММА(Зак.Сумма) * 100
    ИНАЧЕ 0
КОНЕЦ
```

### 9.4. GROUP BY не включает все поля

```bsl
// ОШИБКА: поле в SELECT, но не в GROUP BY
ВЫБРАТЬ Прод.Номенклатура, Прод.Склад, СУММА(Прод.Сумма)
СГРУППИРОВАТЬ ПО Прод.Номенклатура  // Склад не включён!

// ПРАВИЛЬНО
СГРУППИРОВАТЬ ПО Прод.Номенклатура, Прод.Склад
```

### 9.5. Двойной расчёт себестоимости

```bsl
// ОШИБКА: при JOIN с себестоимостью по двум условиям, можно задвоить
ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.СебестоимостьПродаж КАК Себ
    ПО Прод.Регистратор = Себ.ДокументПартии
    И Прод.Номенклатура = Себ.Номенклатура
// Если на один документ приходится несколько партий — задвоит выручку!

// ПРАВИЛЬНО: агрегат во вложенном подзапросе
ЛЕВОЕ СОЕДИНЕНИЕ (
    ВЫБРАТЬ
        Себ.ДокументПартии КАК ДокументПартии,
        Себ.Номенклатура КАК Номенклатура,
        СУММА(Себ.Сумма) КАК Сумма
    ИЗ РегистрНакопления.СебестоимостьПродаж КАК Себ
    СГРУППИРОВАТЬ ПО Себ.ДокументПартии, Себ.Номенклатура
) КАК Себ
    ПО Прод.Регистратор = Себ.ДокументПартии
    И Прод.Номенклатура = Себ.Номенклатура
```

## 10. Связанные правила и стандарты

- **BSL LS:** `FieldsFromJoinsWithoutIsNull`, `RefOveruse`, `SelectTopWithoutOrderBy`

## 11. Шаблоны для `generate_query`

### 11.1. Прибыль по номенклатуре за период

```bsl
ВЫБРАТЬ
    Прод.Номенклатура КАК Номенклатура,
    СУММА(Прод.Количество) КАК Количество,
    СУММА(Прод.Сумма) КАК Выручка,
    ЕСТЬNULL(СУММА(Себ.Сумма), 0) КАК Себестоимость,
    СУММА(Прод.Сумма) - ЕСТЬNULL(СУММА(Себ.Сумма), 0) КАК Прибыль,
    ВЫБОР
        КОГДА ЕСТЬNULL(СУММА(Себ.Сумма), 0) > 0
            ТОГДА ВЫРАЗИТЬ(
                (СУММА(Прод.Сумма) - ЕСТЬNULL(СУММА(Себ.Сумма), 0))
                / СУММА(Себ.Сумма) * 100 КАК ЧИСЛО(15, 2))
        ИНАЧЕ 0
    КОНЕЦ КАК РентабельностьПроц
ИЗ
    РегистрНакопления.{РегистрПродаж}.Обороты(&Начало, &Конец, Период, ) КАК Прод
    ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.{РегистрСебестоимости}.Обороты(
        &Начало, &Конец, Период,
    ) КАК Себ
        ПО Прод.Номенклатура = Себ.Номенклатура
СГРУППИРОВАТЬ ПО
    Прод.Номенклатура
УПОРЯДОЧИТЬ ПО
    Прибыль УБЫВ
```

### 11.2. Динамика прибыли по месяцам

```bsl
ВЫБРАТЬ
    НАЧАЛОПЕРИОДА(Прод.Период, МЕСЯЦ) КАК Месяц,
    СУММА(Прод.Сумма) КАК Выручка,
    ЕСТЬNULL(СУММА(Себ.Сумма), 0) КАК Себестоимость,
    СУММА(Прод.Сумма) - ЕСТЬNULL(СУММА(Себ.Сумма), 0) КАК Прибыль
ИЗ
    РегистрНакопления.{РегистрПродаж} КАК Прод
    ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.{РегистрСебестоимости} КАК Себ
        ПО Прод.Регистратор = Себ.ДокументПартии
ГДЕ
    Прод.Период МЕЖДУ &НачалоГода И &КонецГода
СГРУППИРОВАТЬ ПО
    НАЧАЛОПЕРИОДА(Прод.Период, МЕСЯЦ)
УПОРЯДОЧИТЬ ПО
    Месяц
```
