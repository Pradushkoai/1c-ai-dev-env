# Антипаттерны 1С разработки

## 🔴 Критичные

### 1. Бизнес-логика в модуле формы
**Плохо:**
```bsl
// Form/Module.bsl
&НаСервере
Процедура ВыполнитьРасчетНаСервере()
    Запрос = Новый Запрос;
    Запрос.Текст = "ВЫБРАТЬ ...";
    // расчёты
КонецПроцедуры
```

**Хорошо:**
```bsl
// Object/Module.bsl
Функция ВыполнитьРасчет() Экспорт
    // расчёты
КонецФункции

// Form/Module.bsl
&НаСервере
Процедура ВыполнитьРасчетНаСервере()
    Результат = РеквизитФормыВЗначение("Объект").ВыполнитьРасчет();
КонецПроцедуры
```

### 2. Запросы в цикле
**Плохо:**
```bsl
Для Каждого Элемент Из Список Цикл
    Запрос = Новый Запрос;
    Запрос.Текст = "ВЫБРАТЬ * ИЗ Справочник.Номенклатура ГДЕ Ссылка = &Ссылка";
    Запрос.УстановитьПараметр("Ссылка", Элемент);
    Результат = Запрос.Выполнить();
КонецЦикла;
```

**Хорошо:**
```bsl
Запрос = Новый Запрос;
Запрос.Текст = "ВЫБРАТЬ * ИЗ Справочник.Номенклатура ГДЕ Ссылка В (&Список)";
Запрос.УстановитьПараметр("Список", Список);
Результат = Запрос.Выполнить();
```

### 3. Отсутствие обработки ошибок
**Плохо:**
```bsl
Файл = Новый Файл(Путь);
Текст = Файл.Прочитать();
```

**Хорошо:**
```bsl
Попытка
    Файл = Новый Файл(Путь);
    Текст = Файл.Прочитать();
Исключение
    ЗаписьЖурналаРегистрации("Ошибка чтения файла", УровеньЖурналаРегистрации.Ошибка,,,
        ОписаниеОшибки());
    ВызватьИсключение "Не удалось прочитать файл: " + Путь;
КонецПопытки;
```

### 4. Прямой доступ к БД из формы
**Плохо:**
```bsl
&НаКлиенте
Процедура ПриИзменении(Элемент)
    Запрос = Новый Запрос; // ❌ Нельзя на клиенте!
КонецПроцедуры
```

**Хорошо:**
```bsl
&НаКлиенте
Процедура ПриИзменении(Элемент)
    Данные = ПолучитьДанныеНаСервере();
КонецПроцедуры

&НаСервере
Функция ПолучитьДанныеНаСервере()
    Запрос = Новый Запрос;
    // ...
КонецФункции
```

## 🟡 Важные

### 5. Неправильное именование
**Плохо:** `Процедура р1()`, `Перем а`
**Хорошо:** `Процедура ВыполнитьРасчет()`, `Перем СуммаИтога`

### 6. Отсутствие комментариев
**Плохо:**
```bsl
Функция Расч(А, Б)
    Возврат А * Б * 1.2;
КонецФункции
```

**Хорошо:**
```bsl
// Рассчитывает сумму с НДС 20%
//
// Параметры:
//  Сумма - Число - сумма без НДС
//  Ставка - Число - ставка НДС (0.20)
//
// Возвращаемое значение:
//  Число - сумма с НДС
Функция РассчитатьСуммуСНДС(Сумма, Ставка = 0.20) Экспорт
    Возврат Сумма * (1 + Ставка);
КонецФункции
```

### 7. Магические числа
**Плохо:**
```bsl
Сумма = Количество * 1000 * 1.2;
```

**Хорошо:**
```bsl
Перем КонстантаЦены, СтавкаНДС;
Сумма = Количество * КонстантаЦены * (1 + СтавкаНДС);
```

### 8. Сложные условия
**Плохо:**
```bsl
Если А > 0 И Б < 10 И В <> 5 И Г = Истина И Д > 100 Тогда
```

**Хорошо:**
```bsl
УсловиеВыполнено = (А > 0) И (Б < 10) И (В <> 5);
Если УсловиеВыполнено И (Г = Истина) И (Д > 100) Тогда
```

## 🟢 Стилистические

### 9. Отсутствие областей
**Плохо:** весь код в одной куче
**Хорошо:**
```bsl
#Область ПрограммныйИнтерфейс
// Экспортные процедуры
#КонецОбласти

#Область СлужебныйПрограммныйИнтерфейс
// Служебные экспортные
#КонецОбласти

#Область СлужебныеПроцедурыИФункции
// Внутренние
#КонецОбласти
```

### 10. Длинные процедуры
**Плохо:** процедура на 200 строк
**Хорошо:** разбить на 5 процедур по 40 строк

## Транзакции (источник: BSL LS + ITS #std651)

### 11. BeginTransaction без Try/Catch
**Плохо:**
```bsl
НачатьТранзакцию();
// ... операции ...
ЗафиксироватьТранзакцию(); // если исключение — транзакция не откатится
```
**Хорошо:**
```bsl
НачатьТранзакцию();
Попытка
    // ... операции ...
    ЗафиксироватьТранзакцию();
Исключение
    ОтменитьТранзакцию();
    ВызватьИсключение;
КонецПопытки;
```
**Связанные:** BSL LS: `BeginTransactionBeforeTryCatch`, `CommitTransactionOutsideTryCatch`

### 12. CommitTransaction вне Try
**Плохо:** `ЗафиксироватьТранзакцию()` после `КонецПопытки`
**Хорошо:** `ЗафиксироватьТранзакцию()` внутри `Попытка` перед `Исключение`

### 13. Несбалансированные транзакции
**Плохо:** `НачатьТранзакцию` без парного `Зафиксировать`/`Отменить`
**Связанные:** BSL LS: `PairingBrokenTransaction`

## Общие модули (источник: BSL LS + ITS)

### 14. Неправильное имя общего модуля
**Правила:**
- `ИмяМодуляКэшируемый` — для кэшируемых (Postfix)
- `ИмяМодуляКлиент` — для клиентских
- `ИмяМодуляКлиентСервер` — для клиент-серверных
- `ИмяМодуляВызовСервера` — для серверного вызова
- `ИмяМодуляГлобальный` — для глобальных
- `ИмяМодуляПолныеПрава` — для полных прав

**Связанные:** BSL LS: `CommonModuleNameCached`, `CommonModuleNameClient`, и т.д.

### 15. Переменные в общем модуле
**Плохо:** Объявление переменных на уровне модуля в общем модуле
**Связанные:** BSL LS: `CommonModuleVariables`

### 16. Общий модуль без API
**Плохо:** Общий модуль без ни одного экспортного метода
**Связанные:** BSL LS: `CommonModuleMissingAPI`

## Безопасность (источник: BSL LS + ITS)

### 17. Отключение безопасного режима
**Плохо:** `УстановитьБезопасныйРежим(Ложь)` без необходимости
**Связанные:** BSL LS: `DisableSafeMode`

### 18. Установка привилегированного режима
**Плохо:** `УстановитьПривилегированныйРежим(Истина)` без необходимости
**Связанные:** BSL LS: `SetPrivilegedMode`

### 19. Небезопасный поиск по коду
**Плохо:** `Справочники.Товары.НайтиПоКоду("00001")` без проверки результата
**Хорошо:** Проверять результат на пустоту
**Связанные:** BSL LS: `UnsafeFindByCode`

---

## Производительность (источник: BSL LS + ITS)

### 20. Кэшируемый общий модуль с публичными методами
**Плохо:** Кэшируемый общий модуль (ПовтИсп) с экспортными методами, изменяющими состояние
**Связанные:** BSL LS: `CachedPublic`

### 21. Чрезмерная проверка ОбменДанными.Загрузка
**Плохо:** Проверка `ОбменДанными.Загрузка` в каждом обработчике события
**Хорошо:** Только там где это действительно нужно
**Связанные:** BSL LS: `ExcessiveAutoTestCheck`

---

## Устаревшее (источник: BSL LS + ITS)

### 22. Устаревший ТекущаяДата()
**Плохо:** `ТекущаяДата()` — зависит от часового пояса сервера
**Хорошо:** `ТекущаяДатаСеанса()` — учитывает часовой пояс сеанса
**Связанные:** BSL LS: `DeprecatedCurrentDate`

### 23. Устаревший Найти()
**Плохо:** `Стр.Найти("подстрока")` — устаревший метод
**Хорошо:** `СтрНайти(Стр, "подстрока")` — современный метод
**Связанные:** BSL LS: `DeprecatedFind`

### 24. Устаревший Сообщить()
**Плохо:** `Сообщить("текст")` — без указания поля
**Хорошо:** `Сообщить("текст", СтатусСообщения.Внимание)` или использовать `ОбщегоНазначения.СообщитьПользователю()`
**Связанные:** BSL LS: `DeprecatedMessage`

## Дизайн кода (источник: BSL LS + ITS)

### 25. Сравнение с булевым
**Плохо:** `Если Флаг = Истина Тогда`
**Хорошо:** `Если Флаг Тогда`
**Связанные:** BSL LS: `CompareWithBoolean`

### 26. Удаление элемента при обходе коллекции
**Плохо:**
```bsl
Для Каждого Элемент Из Коллекция Цикл
    Если Условие Тогда
        Коллекция.Удалить(Элемент); // ошибка!
    КонецЕсли;
КонецЦикла;
```
**Хорошо:** Собрать удаляемые элементы в массив, потом удалить
**Связанные:** BSL LS: `DeletingCollectionItem`

### 27. Двойное отрицание
**Плохо:** `Если Не (Не Флаг) Тогда`
**Хорошо:** `Если Флаг Тогда`
**Связанные:** BSL LS: `DoubleNegatives`

### 28. Закомментированный код
**Плохо:** Оставлять закомментированный код в production
**Хорошо:** Удалять, использовать VCS (git) для истории
**Связанные:** BSL LS: `CommentedCode`

### 29. Дублирование строкового литерала
**Плохо:** Одна и та же строка встречается 3+ раза
**Хорошо:** Вынести в константу или переменную
**Связанные:** BSL LS: `DuplicateStringLiteral`

### 30. Дублирование вставки в коллекцию
**Плохо:** Повторяющийся код добавления элементов
**Хорошо:** Вынести в процедуру
**Связанные:** BSL LS: `DuplicatedInsertionIntoCollection`
