## #std437: Оформление текстов запросов

**ITS:** https://its.1c.ru/db/v8std/content/437/hdoc

### Суть
Текст запроса должен быть оформлен единообразно: ключевые слова на отдельных строках, поля выровнены, отступы через `|`.

### Правила
- Каждое ключевое слово (ВЫБРАТЬ, ИЗ, ГДЕ, СГРУППИРОВАТЬ ПО, УПОРЯДОЧИТЬ ПО) — на новой строке
- Поля в SELECT — каждое на отдельной строке
- Использовать `|` в начале каждой строки строкового литерала
- Алиасы через `КАК` — обязательны для вычисляемых полей

### Пример

```bsl
// ПРАВИЛЬНО
Запрос.Текст =
    "ВЫБРАТЬ
    |    Т.Ссылка КАК Ссылка,
    |    Т.Код КАК Код,
    |    Т.Наименование КАК Наименование
    |ИЗ
    |    Справочник.Номенклатура КАК Т
    |ГДЕ
    |    Т.ПометкаУдаления = ЛОЖЬ";

// ПЛОХО
Запрос.Текст = "ВЫБРАТЬ Т.Ссылка, Т.Код ИЗ Справочник.Номенклатура КАК Т";
```

### Связанные диагностики
- BSL LS: `AssignAliasFieldsInQuery`

## #std436: Многократное выполнение однотипных запросов

**ITS:** https://its.1c.ru/db/v8std/content/436/hdoc

### Суть
Избегать многократного выполнения однотипных запросов в цикле. Использовать один запрос с параметром-массивом.

### Правила
- Никогда не выполнять запрос в цикле (N+1 проблема)
- Использовать `В (&Массив)` для пакетной выборки
- Кэшировать результаты если данные не меняются

### Пример

```bsl
// ПЛОХО: N+1 запросов
Для Каждого Стр Из Таблицы Цикл
    Запрос = Новый Запрос("ВЫБРАТЬ ... ГДЕ Код = &Код");
    Запрос.УстановитьПараметр("Код", Стр.Код);
    Результат = Запрос.Выполнить();
КонецЦикла;

// ПРАВИЛЬНО: один запрос
Запрос = Новый Запрос("ВЫБРАТЬ ... ГДЕ Код В (&МассивКодов)");
Запрос.УстановитьПараметр("МассивКодов", Таблица.ВыгрузитьКолонку("Код"));
```

### Связанные диагностики
- BSL LS: `CreateQueryInCycle`
- Наши: Q010

## #std438: Проверка на пустой результат выполнения запроса

**ITS:** https://its.1c.ru/db/v8std/content/438/hdoc

### Суть
Проверку на пустой результат выполнять через `Пустой()`, а не через `Количество() > 0`.

### Правила
- Использовать `Результат.Пустой()` — не загружает данные
- НЕ использовать `Результат.Выгрузить().Количество() > 0` — загружает все строки
- НЕ использовать `Выборка.Следующий()` для проверки пустоты

### Пример

```bsl
// ПРАВИЛЬНО
Если Не Результат.Пустой() Тогда
    // обработка
КонецЕсли;

// ПЛОХО: загружает все данные
Если Результат.Выгрузить().Количество() > 0 Тогда
    // обработка
КонецЕсли;
```

## #std435: Ограничение на ПОЛНОЕ ВНЕШНЕЕ СОЕДИНЕНИЕ

**ITS:** https://its.1c.ru/db/v8std/content/435/hdoc

### Суть
Избегать `ПОЛНОЕ ВНЕШНЕЕ СОЕДИНЕНИЕ` (FULL OUTER JOIN) — медленно и сложно для понимания.

### Правила
- Не использовать ПОЛНОЕ СОЕДИНЕНИЕ без крайней необходимости
- Заменять на два LEFT JOIN через ОБЪЕДИНИТЬ ВСЕ
- Альтернатива: временные таблицы

### Пример

```bsl
// ПЛОХО
"... ПОЛНОЕ СОЕДИНЕНИЕ Б ПО А.Ключ = Б.Ключ"

// ПРАВИЛЬНО
"ВЫБРАТЬ ... ИЗ А ЛЕВОЕ СОЕДИНЕНИЕ Б ПО А.Ключ = Б.Ключ
|ОБЪЕДИНИТЬ ВСЕ
|ВЫБРАТЬ ... ИЗ Б ЛЕВОЕ СОЕДИНЕНИЕ А ПО Б.Ключ = А.Ключ
|ГДЕ А.Ключ ЕСТЬ NULL"
```

### Связанные диагностики
- BSL LS: `FullOuterJoinQuery`
- Наши: O003

## #std434: Использование ОБЪЕДИНИТЬ и ОБЪЕДИНИТЬ ВСЕ

**ITS:** https://its.1c.ru/db/v8std/content/434/hdoc

### Суть
Использовать `ОБЪЕДИНИТЬ ВСЕ` вместо `ОБЪЕДИНИТЬ` если не нужно удалять дубликаты.

### Правила
- `ОБЪЕДИНИТЬ` — удаляет дубликаты (дополнительная группировка, медленнее)
- `ОБЪЕДИНИТЬ ВСЕ` — не удаляет дубликаты (быстрее)
- По умолчанию использовать `ОБЪЕДИНИТЬ ВСЕ`
- `ОБЪЕДИНИТЬ` — только когда нужна уникальность строк

### Пример

```bsl
// ПРАВИЛЬНО: ОБЪЕДИНИТЬ ВСЕ — быстрее
"ВЫБРАТЬ ... ОБЪЕДИНИТЬ ВСЕ ВЫБРАТЬ ..."

// МЕДЛЕННЕЕ: ОБЪЕДИНИТЬ — удаляет дубликаты
"ВЫБРАТЬ ... ОБЪЕДИНИТЬ ВЫБРАТЬ ..."
```

### Связанные диагностики
- Наши: O008

## #std412: Упорядочивание результатов запроса

**ITS:** https://its.1c.ru/db/v8std/content/412/hdoc

### Суть
Упорядочивание результатов запроса должно использовать индексированные поля.

### Правила
- УПОРЯДОЧИТЬ ПО — использовать поля, по которым есть индексы
- При SELECT TOP — УПОРЯДОЧИТЬ ПО обязательно
- Не сортировать по вычисляемым полям без необходимости

### Пример

```bsl
// ПРАВИЛЬНО: сортировка по индексированному полю
"ВЫБРАТЬ ... УПОРЯДОЧИТЬ ПО Дата УБЫВ"

// ПЛОХО: SELECT TOP без ORDER BY
"ВЫБРАТЬ ПЕРВЫЕ 10 ... " // какие 10?
```

### Связанные диагностики
- BSL LS: `SelectTopWithoutOrderBy`
- Наши: O001

## #std535: Округление результатов арифметических операций в запросах

**ITS:** https://its.1c.ru/db/v8std/content/535/hdoc

### Суть
В запросах 1С арифметические операции могут давать неточные результаты из-за разрядности.

### Правила
- Использовать `ВЫРАЗИТЬ(... КАК ЧИСЛО(15,2))` для фиксации разрядности
- Не полагаться на автоматическое определение разрядности
- Проверять результат на тестовых данных

### Пример

```bsl
// ПРАВИЛЬНО
"ВЫБРАТЬ ВЫРАЗИТЬ(Т.Сумма / Т.Количество КАК ЧИСЛО(15,2)) КАК Цена"

// ПЛОХО: разрядность не определена
"ВЫБРАТЬ Т.Сумма / Т.Количество КАК Цена"
```

## #std726: Особенности использования ПОДОБНО

**ITS:** https://its.1c.ru/db/v8std/content/726/hdoc

### Суть
Оператор `ПОДОБНО` имеет особенности: регистрозависимость, спецсимволы, производительность.

### Правила
- `ПОДОБНО` — регистрозависимый (в отличие от =)
- `%` — любая последовательность символов
- `_` — один любой символ
- `[...]` — один символ из набора (только для SQL Server)
- `ПОДОБНО "%текст"` — full table scan (не использует индекс)
- `ПОДОБНО "текст%"` — использует индекс

### Пример

```bsl
// ПРАВИЛЬНО: использует индекс
"ГДЕ Т.Наименование ПОДОБНО "Товар%"" 

// ПЛОХО: full scan
"ГДЕ Т.Наименование ПОДОБНО "%Товар%""

// ПЛОХО: регистрозависимый поиск
"ГДЕ Т.Наименование ПОДОБНО "товар"" // не найдёт "Товар"
```

### Связанные диагностики
- BSL LS: `UsingLikeInQuery`, `IncorrectUseLikeInQuery`
- Наши: Q005

## #std758: Псевдонимы источников данных в запросах

**ITS:** https://its.1c.ru/db/v8std/content/758/hdoc

### Суть
Все источники данных в запросе должны иметь псевдонимы (алиасы) через `КАК`.

### Правила
- Обязательно указывать `КАК` для каждой таблицы
- Алиас должен быть коротким и осмысленным
- Не использовать зарезервированные слова как алиасы
- Обращаться к полям через алиас: `Т.Поле`, а не `Поле`

### Пример

```bsl
// ПРАВИЛЬНО
"ВЫБРАТЬ
|    Т.Ссылка,
|    Т.Наименование
|ИЗ
|    Справочник.Номенклатура КАК Т"

// ПЛОХО: нет алиаса
"ВЫБРАТЬ
|    Ссылка,
|    Наименование
|ИЗ
|    Справочник.Номенклатура"
```

## #std787: Вычисление количества записей в запросах

**ITS:** https://its.1c.ru/db/v8std/content/787/hdoc

### Суть
Для подсчёта количества записей использовать `КОЛИЧЕСТВО(*)`, а не `КОЛИЧЕСТВО(Поле)`.

### Правила
- `КОЛИЧЕСТВО(*)` — считает все строки (быстрее)
- `КОЛИЧЕСТВО(Поле)` — считает только непустые значения (медленнее)
- `КОЛИЧЕСТВО(РАЗЛИЧНЫЕ Поле)` — считает уникальные значения
- Для проверки существования — использовать `СУЩЕСТВУЕТ` или `Пустой()`

### Пример

```bsl
// ПРАВИЛЬНО: считает все строки
"ВЫБРАТЬ КОЛИЧЕСТВО(*) КАК ВсегоСтрок"

// МЕДЛЕННЕЕ: считает только непустые
"ВЫБРАТЬ КОЛИЧЕСТВО(Т.Ссылка) КАК ВсегоСтрок"

// ПЛОХО: для проверки существования
"ВЫБРАТЬ КОЛИЧЕСТВО(*) ИЗ ... ГДЕ ... // потом проверка > 0"

// ПРАВИЛЬНО: для проверки существования
Если Не Запрос.Выполнить().Пустой() Тогда
```
