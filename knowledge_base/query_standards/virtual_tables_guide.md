# Гайд: Виртуальные таблицы в запросах 1С

**Источник:** [ITS #std657](https://its.1c.ru/db/v8std/content/657/hdoc), [ITS #std733](https://its.1c.ru/db/v8std/content/733/hdoc), [ITS #std655](https://its.1c.ru/db/v8std/content/655/hdoc), [v8std.ru](https://v8std.ru)  
**Связанные инструменты:** `generate_query`, `analyze_queries`, `validate_query_static`, `solve_check`

> Подробный гайд по всем видам виртуальных таблиц в 1С: параметры, типичные ошибки, оптимизация. Подчиняется #std657 (обращения к виртуальным таблицам) и #std733 (эффективное обращение к "Остаткам").

---

## 1. Виртуальные таблицы — обзор

| Виртуальная таблица | Регистр | Назначение |
|---|---|---|
| `.Остатки` | РегистрНакопления (Остатки) | Остатки на момент времени |
| `.ОстаткиИОбороты` | РегистрНакопления (Остатки) | Остатки + обороты за период |
| `.Обороты` | РегистрНакопления (Обороты) | Обороты за период |
| `.ОборотыДтКт` | РегистрБухгалтерии | Обороты по счетам Дт/Кт |
| `.ОстаткиДтКт` | РегистрБухгалтерии | Остатки по счетам Дт/Кт |
| `.СрезПоследних` | РегистрСведений | Последние значения на момент |
| `.СрезПервых` | РегистрСведений | Первые значения на момент |

## 2. Правила #std657: обращения к виртуальным таблицам

### 2.1. ВСЕГДА передавать параметры

```bsl
// КАТЕГОРИЧЕСКИ ПЛОХО: без параметров — вычисляет по всем записям
ВЫБРАТЬ
    Ост.Номенклатура,
    Ост.КоличествоОстаток
ИЗ
    РегистрНакопления.ТоварыНаСкладах.Остатки КАК Ост

// ПРАВИЛЬНО: с периодом и условием
ВЫБРАТЬ
    Ост.Номенклатура КАК Номенклатура,
    Ост.КоличествоОстаток КАК КоличествоОстаток
ИЗ
    РегистрНакопления.ТоварыНаСкладах.Остатки(
        &КонецПериода,
        Склад = &Склад
            И Номенклатура В (&ГруппаНоменклатуры)
    ) КАК Ост
```

### 2.2. Параметры виртуальной таблицы `.Остатки`

Сигнатура: `Остатки(<Период>, <Условие>)`

| Параметр | Тип | Назначение |
|---|---|---|
| `Период` | Дата | Момент, на который считаются остатки. ОБЯЗАТЕЛЬНО |
| `Условие` | Строка | Фильтр в терминах измерений регистра |

```bsl
// Полная форма
РегистрНакопления.ТоварыНаСкладах.Остатки(
    &КонецДня,
    Склад = &Склад
        И Номенклатура В (&МассивНоменклатуры)
) КАК Ост
```

### 2.3. Параметры `.Обороты`

Сигнатура: `Обороты(<Начало>, <Конец>, <Периодичность>, <Условие>)`

```bsl
РегистрНакопления.Продажи.Обороты(
    &НачалоПериода,
    &КонецПериода,
    Период,  // или День, Неделя, Месяц, Квартал, Год
    Контрагент = &Контрагент
) КАК Об
```

### 2.4. Параметры `.СрезПоследних`

Сигнатура: `СрезПоследних(<Период>, <Условие>)`

```bsl
РегистрСведений.ЦеныНоменклатуры.СрезПоследних(
    &КонецДня,
    Номенклатура = &Номенклатура
        И ТипЦен = &ТипЦен
) КАК Цены
```

## 3. #std733: Эффективное обращение к "Остаткам"

### 3.1. Минимизировать выборку через условие

Чем больше полей в условии — тем меньше строк вычисляет виртуальная таблица:

```bsl
// ПЛОХО: вычисляет остатки по всей номенклатуре и складам
ВЫБРАТЬ ... ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки(&Дата)

// ПРАВИЛЬНО: фильтр в параметрах
ВЫБРАТЬ ... ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки(
    &Дата,
    Склад = &Склад И Номенклатура В (&МассивНоменклатуры)
)
```

### 3.2. Не использовать виртуальную таблицу в JOIN

**#std655:** Сначала во временную таблицу с индексом:

```bsl
// ПЛОХО: JOIN с виртуальной таблицей
ВЫБРАТЬ
    Ном.Ссылка,
    Ост.КоличествоОстаток
ИЗ
    Справочник.Номенклатура КАК Ном
    ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.ТоварыНаСкладах.Остатки(...) КАК Ост
        ПО Ном.Ссылка = Ост.Номенклатура

// ПРАВИЛЬНО
ВЫБРАТЬ
    Ост.Номенклатура КАК Номенклатура,
    Ост.КоличествоОстаток КАК Количество
ПОМЕСТИТЬ ВТ_Остатки
ИЗ
    РегистрНакопления.ТоварыНаСкладах.Остатки(
        &КонецДня,
        Склад = &Склад
    ) КАК Ост
ИНДЕКСИРОВАТЬ ПО
    Номенклатура
;

ВЫБРАТЬ
    Ном.Ссылка КАК Номенклатура,
    ЕСТЬNULL(Ост.Количество, 0) КАК Количество
ИЗ
    Справочник.Номенклатура КАК Ном
    ЛЕВОЕ СОЕДИНЕНИЕ ВТ_Остатки КАК Ост
        ПО Ном.Ссылка = Ост.Номенклатура
```

### 3.3. Передавать массив значений через `В (&Массив)`

```bsl
Запрос.УстановитьПараметр("МассивНоменклатуры", МассивНоменклатуры);

ВЫБРАТЬ
    Ост.Номенклатура КАК Номенклатура,
    Ост.КоличествоОстаток КАК Количество
ИЗ
    РегистрНакопления.ТоварыНаСкладах.Остатки(
        &КонецДня,
        Номенклатура В (&МассивНоменклатуры)
    ) КАК Ост
```

### 3.4. Использовать `ГруппыКонтрагентов` через таблицу

```bsl
// ПЛОХО: подставлять массив из 10000 элементов
Номенклатура В (&БольшойМассив)

// ПРАВИЛЬНО: передать через временную таблицу
ВЫБРАТЬ Поле ПОМЕСТИТЬ ВТ_Список ИЗ (...)
... Номенклатура В (ВЫБРАТЬ Поле ИЗ ВТ_Список)
```

## 4. Паттерны использования

### 4.1. Остатки + движения за период

```bsl
// Остатки на начало
ВЫБРАТЬ
    Нач.Номенклатура КАК Номенклатура,
    Нач.КоличествоОстаток КАК НачальныйОстаток
ПОМЕСТИТЬ ВТ_НачалоПериода
ИЗ
    РегистрНакопления.ТоварыНаСкладах.Остатки(&НачалоПериода, Склад = &Склад) КАК Нач
ИНДЕКСИРОВАТЬ ПО Номенклатура
;

// Обороты за период
ВЫБРАТЬ
    Об.Номенклатура КАК Номенклатура,
    СУММА(Об.КоличествоПриход) КАК Приход,
    СУММА(Об.КоличествоРасход) КАК Расход
ПОМЕСТИТЬ ВТ_Обороты
ИЗ
    РегистрНакопления.ТоварыНаСкладах.Обороты(
        &НачалоПериода, &КонецПериода, Период,
        Склад = &Склад
    ) КАК Об
СГРУППИРОВАТЬ ПО Об.Номенклатура
ИНДЕКСИРОВАТЬ ПО Номенклатура
;

// Итоговый отчёт
ВЫБРАТЬ
    Ном.Ссылка КАК Номенклатура,
    ЕСТЬNULL(Нач.НачальныйОстаток, 0) КАК НачальныйОстаток,
    ЕСТЬNULL(Об.Приход, 0) КАК Приход,
    ЕСТЬNULL(Об.Расход, 0) КАК Расход,
    ЕСТЬNULL(Нач.НачальныйОстаток, 0) + ЕСТЬNULL(Об.Приход, 0) - ЕСТЫNULL(Об.Расход, 0) КАК КонечныйОстаток
ИЗ
    Справочник.Номенклатура КАК Ном
    ЛЕВОЕ СОЕДИНЕНИЕ ВТ_НачалоПериода КАК Нач ПО Ном.Ссылка = Нач.Номенклатура
    ЛЕВОЕ СОЕДИНЕНИЕ ВТ_Обороты КАК Об ПО Ном.Ссылка = Об.Номенклатура
```

### 4.2. Срез последних цен

```bsl
// Цены на дату продажи
ВЫБРАТЬ
    Прод.Номенклатура КАК Номенклатура,
    Прод.Период КАК ДатаПродажи,
    Цены.Цена КАК Цена
ИЗ
    РегистрНакопления.Продажи КАК Прод
    ВНУТРЕННЕЕ СОЕДИНЕНИЕ РегистрСведений.ЦеныНоменклатуры.СрезПоследних(
        Прод.Период,  // ← динамическая дата из внешней таблицы
        Прод.Номенклатура = Номенклатура
            И ТипЦен = &ТипЦен
    ) КАК Цены
        ПО Прод.Номенклатура = Цены.Номенклатура
```

**Ловушка:** динамический период в виртуальной таблице — очень медленно для больших выборок. Лучше выгрузить продажи во временную таблицу, потом итерировать по периодам или использовать `ВЫБОР`:

```bsl
// АЛЬТЕРНАТИВА: для каждого периода отдельный запрос
Для Каждого Стр Из ТаблицыПериодов Цикл
    Запрос.Текст = "...СрезПоследних(&Дата, ...)";
КонецЦикла;
```

### 4.3. Обороты Дт/Кт (бухгалтерия)

```bsl
ВЫБРАТЬ
    Об.СчетДт КАК СчетДт,
    Об.СчетКт КАК СчетКт,
    Об.СубконтоДт1 КАК СубконтоДт1,
    Об.Сумма КАК Сумма
ИЗ
    РегистрБухгалтерии.Хозрасчетный.ОборотыДтКт(
        &Начало, &Конец, Период,
        СчетДт = &СчетДт И СчетКт = &СчетКт
    ) КАК Об
```

### 4.4. Получение остатков с группировкой по складам

```bsl
// Остатки по всем складам одновременно
ВЫБРАТЬ
    Ост.Склад КАК Склад,
    Ост.Номенклатура КАК Номенклатура,
    Ост.КоличествоОстаток КАК Количество
ИЗ
    РегистрНакопления.ТоварыНаСкладах.Остатки(
        &КонецДня,
        Склад В (&МассивСкладов)
    ) КАК Ост
УПОРЯДОЧИТЬ ПО
    Склад, Номенклатура
```

## 5. Ловушки виртуальных таблиц

### 5.1. Параметр-период игнорируется

```bsl
// ПЛОХО: вместо параметра — пустое значение
РегистрНакопления.ТоварыНаСкладах.Остатки(, Склад = &Склад)
// Считает остатки на текущий момент, не на нужную дату

// ПРАВИЛЬНО: всегда передавать дату
РегистрНакопления.ТоварыНаСкладах.Остатки(&КонецДня, Склад = &Склад)
```

### 5.2. Условие по полю ресурса (а не измерения)

```bsl
// НЕЛЬЗЯ: Сумма — ресурс, не измерение
РегистрНакопления.Продажи.Обороты(,,, Сумма > 1000)

// МОЖНО: фильтр в WHERE после виртуальной таблицы
ВЫБРАТЬ ... ИЗ РегистрНакопления.Продажи.Обороты(...) КАК Об
ГДЕ Об.Сумма > 1000
```

### 5.3. Не использовать `ОстаткиИОбороты` без необходимости

`ОстаткиИОбороты` — самая тяжёлая виртуальная таблица. Использовать только когда нужны одновременно остатки и обороты:

```bsl
// Избегать без необходимости:
РегистрНакопления.ТоварыНаСкладах.ОстаткиИОбороты(...)

// Лучше: два отдельных запроса — Остатки + Обороты
ВТ_Остатки + ВТ_Обороты + JOIN
```

### 5.4. Виртуальная таблица как источник для подзапроса в JOIN

```bsl
// ПЛОХО: вложенный подзапрос с виртуальной таблицей
ЛЕВОЕ СОЕДИНЕНИЕ (
    ВЫБРАТЬ ... ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки(...)
) КАК Ост

// ПРАВИЛЬНО: вынести в отдельную временную таблицу
ВЫБРАТЬ ... ПОМЕСТИТЬ ВТ_Остатки ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки(...)
ИНДЕКСИРОВАТЬ ПО Номенклатура;
// ... JOIN с ВТ_Остатки
```

## 6. Сравнение производительности

| Паттерн | Время (10 млн записей) | 备注 |
|---|---|---|
| Остатки без параметров | ~120 сек | Полное сканирование |
| Остатки с фильтром по складу | ~3 сек | Использует индекс по Склад |
| Остатки с фильтром по складу + массиву номенклатуры | ~0.3 сек | Двойная фильтрация |
| JOIN с виртуальной таблицей | ~45 сек | Скрытое декартово произведение |
| Временная таблица + индекс + JOIN | ~1.2 сек | Оптимально |

## 8. Шаблоны для `generate_query`

### 8.1. Остатки на дату

```bsl
ВЫБРАТЬ
    Ост.Номенклатура КАК Номенклатура,
    ЕСТЬNULL(Ост.КоличествоОстаток, 0) КАК Количество
ИЗ
    РегистрНакопления.{РегистрОстатков}.Остатки(
        &КонецПериода,
        {Условие}
    ) КАК Ост
```

### 8.2. Обороты за период с группировкой

```bsl
ВЫБРАТЬ
    Об.Номенклатура КАК Номенклатура,
    СУММА(Об.КоличествоПриход) КАК Приход,
    СУММА(Об.КоличествоРасход) КАК Расход,
    СУММА(Об.СуммаПриход) КАК СуммаПрихода
ИЗ
    РегистрНакопления.{Регистр}.Обороты(
        &Начало, &Конец, Период,
        {Условие}
    ) КАК Об
СГРУППИРОВАТЬ ПО
    Об.Номенклатура
```

### 8.3. Срез последних значений

```bsl
ВЫБРАТЬ
    Срез.Номенклатура КАК Номенклатура,
    Срез.Цена КАК Цена,
    Срез.Период КАК Период
ИЗ
    РегистрСведений.{РегистрСведений}.СрезПоследних(
        &Дата,
        {Условие}
    ) КАК Срез
```
