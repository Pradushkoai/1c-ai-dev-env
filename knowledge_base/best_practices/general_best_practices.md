# Best Practices 1С разработки

## 📐 Архитектура

### 1. Разделение ответственности
- **Модуль объекта** — бизнес-логика, обработчики событий (ПередЗаписью, ПриЗаписи)
- **Модуль менеджера** — фабричные методы (НайтиПоКоду, СоздатьДокумент)
- **Модуль формы** — UI-логика, обработчики элементов
- **Общий модуль** — переиспользуемая логика (с пометками Сервер/Клиент)

### 2. Клиент-серверное взаимодействие
- Тяжёлые операции — на сервере (&НаСервере)
- UI-операции — на клиенте (&НаКлиенте)
- Минимизируйте round-trips: одна серверная процедура вместо десяти

### 3. Области кода
```bsl
#Область ПрограммныйИнтерфейс          // Экспортные для внешних
#Область СлужебныйПрограммныйИнтерфейс // Экспортные для внутреннего использования
#Область СлужебныеПроцедурыИФункции    // Внутренние
#Область ОбработчикиСобытий            // Обработчики
#Область Инициализация                 // Инициализация
```

## 🔍 Поиск и запросы

### 4. Использование запросов
- Используйте параметры (`&Параметр`), не конкатенацию строк
- Группируйте операции в один запрос
- Используйте `В` вместо цикла с отдельными запросами
- Индексы: создавайте для часто запрашиваемых полей

### 5. Оптимизация запросов
```bsl
// Плохо: запрос в цикле
Для Каждого Элемент Из Список Цикл
    Запрос.УстановитьПараметр("Ссылка", Элемент);
    Результат = Запрос.Выполнить();
КонецЦикла;

// Хорошо: один запрос с В
Запрос.УстановитьПараметр("Список", Список);
Результат = Запрос.Выполнить();
```

## 📊 Работа с данными

### 6. Транзакции
```bsl
НачатьТранзакцию();
Попытка
    // операции
    ЗафиксироватьТранзакцию();
Исключение
    ОтменитьТранзакцию();
    ВызватьИсключение;
КонецПопытки;
```

### 7. Обработка ошибок
- Используйте `Попытка/Исключение` для внешних операций
- Логируйте ошибки через `ЗаписьЖурналаРегистрации`
- Предоставляйте пользователю понятные сообщения

### 8. Освобождение ресурсов
```bsl
Запрос = Неопределено;
Выборка = Неопределено;
// Явно освобождайте ссылки на большие объекты
```

## 🎨 UI/UX

### 9. Формы
- Используйте управляемые формы (Managed)
- Размещайте логику в модуле объекта, не в форме
- Используйте `РеквизитФормыВЗначение` / `ЗначениеВРеквизитФормы`
- Минимизируйте серверные вызовы

### 10. Отчёты
- Используйте СКД для всех отчётов
- Параметры — через СКД, не в коде
- Группировки — через СКД
- Условное оформление — через СКД

## 📝 Документирование

### 11. Комментарии
```bsl
// Краткое описание функции
//
// Параметры:
//  Параметр1 - Тип1 - описание
//  Параметр2 - Тип2 - описание
//
// Возвращаемое значение:
//  Тип - описание
```

### 12. Именование
- Процедуры/Функции: `ПолучитьДанныеПоКлиенту` (глагол + объект)
- Переменные: `СуммаИтога`, `СписокТоваров` (существительное)
- Константы: `КонстантаЦены` (с префиксом)
- Реквизиты: осмысленные имена без сокращений

## 🧪 Тестирование

### 13. Проверка заполнения
```bsl
Процедура ОбработкаПроверкиЗаполнения(Отказ, ПроверяемыеРеквизиты)
    Если ПустаяСтрока(Наименование) Тогда
        Сообщение = Новый СообщениеПользователю;
        Сообщение.Текст = "Укажите наименование";
        Сообщение.Поле = "Наименование";
        Сообщение.Сообщить();
        Отказ = Истина;
    КонецЕсли;
КонецПроцедуры
```

### 14. Валидация
- Проверяйте данные перед записью
- Используйте `ОбработкаПроверкиЗаполнения`
- Предоставляйте понятные сообщения об ошибках

## 🔒 Безопасность

### 15. Проверка прав
```bsl
Если Не ПравоДоступа("Чтение", Метаданные.Справочники.Номенклатура) Тогда
    ВызватьИсключение "Нет прав на чтение";
КонецЕсли;
```

### 16. RLS
- Используйте RLS для ограничения доступа по строкам
- Не реализуйте RLS в коде — только через метаданные

## ⚡ Производительность

### 17. Индексы
- Создавайте индексы для часто запрашиваемых полей
- Используйте `Indexing = Index` в метаданных

### 18. Кэширование
- Используйте кэш для часто используемых данных
- `ЗначенияВСтрокуXML` / `ЗначениеИзСтрокиXML` для сериализации

### 19. Пакетные операции
- Используйте пакетные запросы вместо цикла
- Группируйте операции

### 20. Ленивая загрузка
- Не загружайте все данные сразу

## Транзакции (источник: BSL LS + ITS)

### 21. Правильная структура транзакции
```bsl
НачатьТранзакцию();
Попытка
    // ... операции ...
    ЗафиксироватьТранзакцию();
Исключение
    ОтменитьТранзакцию();
    // Запись ошибки в журнал
    ВызватьИсключение; // проброс выше
КонецПопытки;
```

### 22. Минимум кода в транзакции
- Не делать запросы к БД внутри транзакции без необходимости
- Не вызывать интерактивные операции (Предупреждение, ОткрытьФорму)
- Не записывать другие объекты — только целевой

### 23. Транзакции и исключения
- `ЗафиксироватьТранзакцию()` — ВСЕГДА внутри `Попытка`
- `ОтменитьТранзакцию()` — ВСЕГДА внутри `Исключение`
- После `ОтменитьТранзакцию` — `ВызватьИсключение` для проброса

## Общие модули (источник: BSL LS + ITS)

### 24. Именование общих модулей
- `МодульКэшируемый` — повторное использование (ПовтИсп)
- `МодульКлиент` — только клиент
- `МодульКлиентСервер` — клиент и сервер
- `МодульВызовСервера` — серверный вызов
- `МодульГлобальный` — глобальный контекст
- `МодульПолныеПрава` — привилегированный режим

### 25. Структура общего модуля
```bsl
#Область ПрограммныйИнтерфейс
// Экспортные методы — API модуля
Функция МойМетод() Экспорт
    // ...
КонецФункции
#КонецОбласти

#Область СлужебныйПрограммныйИнтерфейс
// Экспортные методы — только для внутренних модулей
#КонецОбласти

#Область СлужебныеПроцедурыИФункции
// Внутренние методы — не экспорт
#КонецОбласти
```

### 26. Не использовать переменные уровня модуля
Общие модули — stateless. Переменные уровня модуля могут вызвать проблемы
при кэшировании (ПовтИсп).

## Безопасность (источник: BSL LS + ITS)

### 27. Безопасный режим
- Не отключать `УстановитьБезопасныйРежим(Ложь)` без крайней необходимости
- Если нужно — включать обратно сразу после операции

### 28. Привилегированный режим
- Использовать только в общих модулях с `ПолныеПрава`
- Минимизировать код в привилегированном режиме

### 29. Поиск по коду
- Всегда проверять результат `НайтиПоКоду` на пустоту
- Не полагаться на существование элемента

---

## Производительность (источник: BSL LS + ITS)

### 30. ТекущаяДатаСеанса вместо ТекущаяДата
- `ТекущаяДата()` — часовой пояс сервера (непредсказуемо)
- `ТекущаяДатаСеанса()` — часовой пояс сеанса (правильно)

### 31. СтрНайти вместо Найти
- `Стр.Найти()` — устаревший метод
- `СтрНайти(Стр, "подстрока")` — современный

### 32. Сообщить с указанием поля
```bsl
// Плохо
Сообщить("Заполните поле");

// Хорошо
СообщениеПользователю = Новый СообщениеПользователю;
СообщениеПользователю.Текст = "Заполните поле";
СообщениеПользователю.Поле = "Номенклатура";
СообщениеПользователю.УстановитьДанные(Форма);
СообщениеПользователю.Сообщить();
```

## Стандарты кода (источник: BSL LS + ITS)

### 33. Каноническое написание ключевых слов
- `Если` (не `есЛИ`, не `ЕСЛИ`)
- `Процедура` (не `ПРОЦЕДУРА`)
- `КонецЕсли` (не `Конец Если`)

### 34. Отсутствие кода до первой процедуры
Весь исполняемый код должен быть внутри процедур/функций.

### 35. Директивы компиляции
- `&НаСервере` — для серверных процедур
- `&НаКлиенте` — для клиентских
- `&НаСервереБезКонтекста` — для серверных без формы
- Не указывать избыточные директивы

### 36. Области кода
```bsl
#Область ПрограммныйИнтерфейс
#Область СлужебныйПрограммныйИнтерфейс
#Область СлужебныеПроцедурыИФункции
#Область Инициализация
```

### 37. Длина имени объекта метаданных
- Не более 80 символов
- Осмысленные имена
- Без префиксов разработчика (кроме расширений)

### 38. Зарезервированные имена параметров
Не использовать как имена параметров: `Форма`, `Элемент`, `ЭтаФорма`,
`ЭтотОбъект`, `Метаданные`, `ПраваДоступа`, и т.д.
- Используйте постраничную загрузку для больших списков
