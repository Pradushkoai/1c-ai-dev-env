# ADR-0008: Язык проекта — русский

**Дата:** 2026-07-04
**Статус:** Accepted

## Контекст

Проект 1c-ai-dev-env создавался как среда разработки для 1С:Предприятие —
российская платформа, весь контекст которой (документация 1С, стандарты
ITS, сообщество Инфостарт) на русском языке.

В кодовой базе ~80K LOC сложилось следующее распределение языков:
- 1303 комментария (`#`) на русском
- 751 docstring на русском
- 3594 строки с кириллицей (BSL templates, сообщения об ошибках, паттерны)
- 197 строк с латиницей (импорты, технические термины)
- README.md на русском, README.en.md — английский перевод (285 строк)

При этом в pyproject.toml указан `Development Status :: 4 - Beta` (без
указания языка), а в CONTRIBUTING.md нет явного правила о языке.

Задача 4.1 плана улучшения требовала принять решение:
- Вариант A: русскоязычный проект (комментарии на русском, README.en.md
  поддерживается волонтёрами)
- Вариант B: английский везде (для OSS-community)

## Рассмотренные варианты

### Вариант A: Русскоязычный проект

- **Pros:**
  - Соответствует домену (1С — российская платформа, вся документация на русском)
  - 99% существующего кода уже на русском — 0 migration cost
  - Целевая аудитория — русскоязычные 1С-разработчики
  - Сообщество 1С (Инфостарт, ITS) русскоязычное
  - BSL-код и стандарты 1С (STD 456 и др.) на русском — комментарии
    должны быть на том же языке для контекста
- **Cons:**
  - Ограничивает англоязычных контрибьюторов
  - README.en.md требует поддержки (сейчас 285 строк, может устаревать)

### Вариант B: Английский везде

- **Pros:**
  - Шире OSS-community
  - Инструменты (GitHub, IDE) лучше работают с английским
- **Cons:**
  - Migration: 1303 комментария + 751 docstring нужно перевести
  - Потеря контекста: BSL-код на русском, комментарии на английском — рассинхрон
  - Целевая аудитория (1С-разработчики) преимущественно русскоязычна
  - README.en.md уже есть — перевод существующего контента
  - Высокий риск regression при массовом переводе

## Решение

**Вариант A — русскоязычный проект.**

### Обоснование

1. **Домен определяет язык:** 1С:Предприятие — российская платформа.
   Все стандарты (ITS 456, 454, 455), документация, сообщество — русскоязычные.
   Комментарии к BSL-коду должны быть на русском для контекста.

2. **Существующий код:** 99% комментариев и docstrings уже на русском.
   Перевод 2054 строк — огромная работа с высоким риском regression.

3. **Целевая аудитория:** 1С-разработчики в РФ и СНГ. Английский для них
   — второй язык. README.en.md покрывает англоязычных пользователей.

4. **Паритет:** BSL-код (переменные, функции, области) на русском.
   Комментарии на английском создали бы рассинхрон: код `Процедура Тест()`
   с комментарием `# Test procedure` — нелогично.

5. **README.en.md существует:** англоязычные пользователи не отрезаны.
   Перевод поддерживается волонтёрами (явно указано в README).

### Правила

1. **Комментарии и docstrings в `src/`:** только русский.
2. **BSL-шаблоны (`templates/`):** только русский (выводятся пользователю).
3. **Сообщения об ошибках (`raise ValueError("...")`):** русский.
4. **README.md:** основной, русский.
5. **README.en.md:** английский перевод, поддерживается волонтёрами.
6. **CONTRIBUTING.md:** русский (целевая аудитория — русскоязычные контрибьюторы).
7. **ADR:** русский (архитектурные решения для команды).
8. **Commit messages:** русский (с английскими префиксами `feat:`, `fix:`, etc.).
9. **Имена переменных, функций, классов:** английский (Python convention).
10. **Импорты, технические термины:** английский (API, JSON, XML, etc.).

### Enforcement

- **AGENTS.md:** добавить правило "Комментарии и docstrings на русском"
- **CONTRIBUTING.md:** добавить раздел "Язык проекта"
- **Pre-commit hook (опционально):** проверка, что новые комментарии в `src/`
  содержат кириллицу или технические термины (не выполнять для экспериментального кода)

## Последствия

### Положительные

- ✅ 0 migration cost — код уже на русском
- ✅ Соответствие домену (1С)
- ✅ Контекст сохранён (BSL код + комментарии на одном языке)
- ✅ README.en.md покрывает англоязычных пользователей

### Отрицательные

- ⚠️ Англоязычные контрибьюторы должны использовать README.en.md
- ⚠️ README.en.md может отставать от README.md — нужен волонтёр-переводчик

### Future implications

- При появлении активных англоязычных контрибьюторов — пересмотреть
- При international expansion (если 1С выйдет на международный рынок) — пересмотреть
- README.en.md добавить в CI check (синхронизация с README.md)
