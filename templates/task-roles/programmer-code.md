## Чек-лист реализации

### Структура модуля (STD 454)
- [ ] `#Область ПрограммныйИнтерфейс` — экспортные процедуры/функции
- [ ] `#Область СлужебныйПрограммныйИнтерфейс` — служебные экспортные
- [ ] `#Область СлужебныеПроцедурыИФункции` — внутренние
- [ ] `#Область Инициализация` — обработчики событий

### Использование методов платформы (из плана Архитектора)
- [ ] Метод 1: `<method>` — использован в `<procedure>`
- [ ] Метод 2: `<method>` — использован в `<procedure>`
- [ ] Все методы из плана задействованы (не выдуманы новые)

### Стандарты 1С (STD)
- [ ] Запросы — через `УстановитьПараметр`, не конкатенация строк
- [ ] Серверные вызовы минимизированы
- [ ] Транзакции — только в `&НаСервере`
- [ ] Проверка прав через `ПравоДоступа` (если применимо)
- [ ] БСП-подходы: `ОбщегоНазначения.`, `ОбщегоНазначенияКлиентСервер.`

### Именование
- [ ] Имена переменных осмысленные (не `А`, `Б`, `Перем1`)
- [ ] Имена процедур/функций — глагол + существительное (СоздатьТовар, НайтиПоАртикулу)
- [ ] Имена модулей соответствуют стандартам 1С
