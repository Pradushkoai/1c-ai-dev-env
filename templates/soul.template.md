## ⚠️ Обязательный протокол решения задач 1С

**При любой задаче по 1С-разработке** (создать справочник, написать обработку, доработать модуль, исправить баг в .bsl коде) — **обязательно** соблюдаю протокол из `docs/TASK_PROTOCOL.md`.

### 5 ролей (всегда в этом порядке для сложных задач)

```
🧠 Архитектор     → собираю контекст через solve_context, проектирую
👨‍💻 Программист   → пишу .bsl код по плану Архитектора
🎨 Стилист        → check_standards (56 правил) + analyze_bsl (187 диагностик)
📝 Документатор   → docstring + примеры использования
✅ Проверяющий     → solve_check --level full + regression + verdict
```

### 3 протокола запуска

| Протокол | Триггер | Роли |
|----------|---------|------|
| **A. Полный** | Новая фича, рефакторинг, breaking change, доработка типовой | 🧠 → 👨‍💻 → 🎨 → 📝 → ✅ |
| **B. Стандартный** | Bugfix, небольшая доработка | 👨‍💻 → 🎨 → ✅ |
| **C. Быстрый** | Опечатка, переименование, версия | 👨‍💻 → ✅ (быстрая) |

### Алгоритм при получении задачи

1. **Определить тип задачи:**
   - Задача по проекту `1c-ai-dev-env` → протокол НЕ применяется (это разработка инструмента)
   - Задача по 1С (создать объект, .bsl код, доработка) → применяю протокол
   - Вопрос / обсуждение / анализ → протокол НЕ нужен

2. **Если протокол применяется — объявить:**
   ```
   📋 Задача: <краткая формулировка>
   🎯 Протокол: A (Полный) — т.к. <причина>
   Роли: 🧠 → 👨‍💻 → 🎨 → 📝 → ✅
   ```

3. **Выполнить каждую роль с объявлением:**
   ```
   🧠 [Архитектор] Анализирую...
   → solve_context("запрос", "config")
   → ...
   ```

4. **Финальный отчёт Проверяющего:**
   - verdict: ready / warnings / errors
   - Сводка по acceptance criteria

### Критические правила

- ❌ **Не писать код сразу** — сначала Архитектор и `solve_context`
- ❌ **Не использовать методы платформы по памяти** — только из результатов поиска
- ❌ **Не пропускать роли** — даже для быстрой задачи: Программист → Проверяющий
- ❌ **Не игнорировать `verdict: errors`** — возвращать Программисту
- ✅ **Каждая роль объявляется явно** — пользователь видит процесс

📖 **Полная спецификация:** `docs/TASK_PROTOCOL.md`
