## 3 принципа работы

### 1. Уточни ТЗ (не выдумывай)
Если задача неполная — задай уточняющие вопросы.
Не генерируй реквизиты, методы, формы «по умолчанию».
Лучше спросить, чем сделать неправильно.

### 2. Используй репозиторий как источник знаний
- `1c-ai search "запрос"` — BM25 поиск по 8141 методам платформы
- `1c-ai solve context "задача" --config X` — собрать контекст
- `1c-ai config list` — список загруженных конфигураций
- `1c-ai get_api_reference` (через MCP) — API-справочник конфигурации

Не пиши методы платформы по памяти — проверяй через поиск.

### 3. Проверь результат
- `1c-ai standards <file.bsl>` — 56 правил (0 errors обязательно)
- `1c-ai solve check <file.bsl> --level full` — полная проверка
- `1c-ai bsl analyze <file.bsl>` — 187 диагностик BSL LS

Если `verdict: errors` — вернись и исправь, не пушить с ошибками.

## Чек-лист решения задачи 1С

При любой задаче по 1С-разработке — **обязательно** пройди чек-лист из
`runtime/checklist.md` (или `templates/checklist.template.md`).

Чек-лист = проверяемые условия (не роли, не ритуал).
Если пункт ❌ — нельзя идти дальше.
